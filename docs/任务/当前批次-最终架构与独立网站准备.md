# 当前批次：最终架构与独立网站准备

## 目标

在今天开始建立最终多平台结构的基础，不提前实现尚未确认的平台 SDK。先完成独立网站生产部署准备，并为首批项目申请准备材料。

## 当前执行项

- [x] 初步确认 `packages/game-core` 为规则、AI、关卡和奖励的唯一运行时核心。
- [x] `apps/player-web` 承担独立站/H5玩家入口，`apps/admin` 承担管理入口，`apps/web` 仅保留兼容职责。
- [x] 初步确认平台生命周期、广告、分享、支付和账号只能通过适配接口进入。
- [x] 核对代码目录、构建脚本和适配边界，并建立源码/产物防回退门禁。
- [x] 确认独立网站生产构建、资源检查和自动回归门禁。
- [ ] 准备独立网站部署所需的域名、隐私页面、构建材料和演示入口。
- [ ] 准备首批平台/项目申请资料，先申请并等待平台条件。
- [x] 保留 Poki/CrazyGames 的接口骨架与本地 Mock，暂不实现真实 SDK。
- [x] 只记录 App 的未来外壳方向，不创建 App 发布包，不接入广告、IAP、账号或云存档。

## 阶段门槛

### 独立网站部署前

- `pnpm release:check` 通过。
- 生产构建能够生成独立网站部署产物。
- 独立网站关闭平台广告和对外 Admin 入口。
- 移动端、刷新、资源失败和核心对局路径完成检查。

### 平台 SDK 开发前

- 已取得平台项目/申请结果或官方接入规格。
- 已确认 SDK 版本、初始化、广告回调、生命周期、隐私和提交要求。
- 已确认平台构建产物和验收入口。

### App 启动前

- 至少一个 H5 渠道有真实用户行为数据。
- 已证明登录、云存档、广告或 IAP 有明确用户价值。
- 已确定移动端账号、隐私、支付、广告和商店发布责任。

## 暂不执行

- 不凭猜测开发真实 Poki/CrazyGames SDK。
- 不把带环境变量的 Web 构建称为已完成的平台包。
- 不创建空壳 Android/iOS 发布项目来制造完成假象。
- 不提前引入移动端第二套 UI 或第二套游戏规则。

## 最近验证

- 独立站构建：`pnpm build:standalone` 已切换到独立 `player-web`，生成36个路由结果。
- Poki/CrazyGames 构建变体由 `player-web` 生成；这只是平台构建边界，不代表真实SDK已接入。
- 资源检查：`pnpm check:assets` 已通过，23个真实资源引用完整；测试专用 `/__missing__/` 路径按故障注入处理，不计入生产资源。
- Web测试：`pnpm test:web` 已通过，16/16项通过。
- 平台包体审计：玩家包 Admin/API 为0，外部统计域名不存在；Admin包不包含公开玩家路由，严格审计均通过。
- 发布检查：核心测试59项通过、Web测试16项通过，平台边界、玩家边界、应用源码边界、资源、玩家构建和玩家产物审计全部通过；Admin构建和产物审计已加入门禁并单独验证通过。
