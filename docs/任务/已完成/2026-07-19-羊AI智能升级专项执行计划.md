# 羊 AI 智能升级专项执行计划

> 状态：已完成并归档。本文保存本期开发范围和验收门槛；长期产品原则见[羊 AI 机制与制作人验收指南](../../产品核心/09-羊AI机制与制作人验收指南.md)，运行证据见[羊 AI V2 24关验证](../../产品核心/验证记录/2026-07-19-羊AI-v2-24关验证.md)。

## 一、本期目标

本期不追求全局完美AI，而是把生产 Hard 从“短视战术防守”升级为“可信的主动战术对手”，并建立以后每次修改AI都能复用的24关验收体系。

完成后应达到：

- 春日保持可学习，后段不依赖明显送子；
- 夏秋能够稳定避吃、挡线和限制狼机动；
- 冬季能够预判完整捕食链、针对孤立狼形成局部合围，并减少优势局无意义往返；
- `winter-03` 的连续吃羊可以被归因成狼方强制路线、地图风险或AI劣着，不再只靠感觉争论；
- 制作人能在Admin看懂“AI为何这样走、是否降级、问题属于哪一类”。

## 二、明确不做

- 不在本期承诺求解24关完整理论胜负。
- 不直接训练AlphaZero类模型，不引入在线服务或生成式AI。
- 不偷偷动态调低或调高主线难度。
- 不为了提高羊胜率直接修改胜利目标8、基础规则或全部地图。

## 三、相关任务合并结论

全面检查任务清单、原剩余专项、需求池、24关资料和Admin后，本期AI相关确定性工作全部合并到 `T-030`：

| 原位置或问题 | 本期处理 |
|---|---|
| `winter-04 LONG_TAIL` | 第一批复现归因，第四批复验关闭 |
| 24关缺少完整羊决策质量 | 第一批建立矩阵，第六批进入门禁 |
| `winter-03` 连续送吃观察 | 第一批自动反例搜索；收到JSON后追加精确复盘，不等待JSON才开始 |
| Admin AI仍需制作人验证 | 第五批统一完成通俗结论、候选证据和问题包 |
| 24关人工试玩中的AI归因 | 第四批先自动筛查；用户只抽查高压关和报告体感 |
| Hard移动端性能与降级 | 第三批统一完成 |
| 关卡AI文案、档位和实际配置一致性 | 第四、五、六批统一检查 |
| AI故障恢复、合法行动、固定种子和基础回放 | 已有测试和入口，作为回归项，不重复开发 |

长期AI内容不在本任务维护，统一见[产品扩展](../../产品扩展/08-长期更新与无限内容架构.md)。本期只为稳定主线AI提供可扩展配置和搜索边界。

因此本期没有第二个隐藏的AI开发任务；AI代码、指标、24关、Admin、性能和门禁都由 `T-030` 单一管理。

## 四、执行批次

### 第一批：建立真实基线与反例库

**解决问题**：现有胜率不能说明羊每一步是否聪明。

开发内容：

1. 建立24关羊决策质量矩阵，统一固定种子和线上实际档位。
2. 每个羊回合记录：合法候选、被支配动作、下一完整狼回合最大可捕食数、狼机动变化、受困狼数、Hard节点、是否降级和行动解释。
3. 增加“连续可避免送子”和“最长连吃前责任动作”指标。
4. 建立典型局面集：避吃、堵落点、围孤狼、岩石封路、合理牺牲、7/8止损、优势收网和防循环。
5. 复现 `winter-04 LONG_TAIL` 的种子与棋谱，区分狼代理、地图、重复规则和羊AI责任。

产物：24关基线报告、关键棋谱、黄金局面测试集和可复现命令。

完成门槛：所有24关都有同一字段口径；任何 `chosenDominated`、Hard降级、长连吃和长尾都能定位到关卡、种子与动作。

### 第二批：Hard V2 候选与评分升级

**解决问题**：当前Hard只在即时最高分动作中搜索，无法选择短期略亏但能形成合围的计划。

开发内容：

1. 保留“有安全替代时不选被支配动作”的硬护栏。
2. 建立 `guided/foundation/tactical/strategic/expert` 五个生产画像，使24关曲线不再被三档内部算法限制。
3. 从“只搜索即时最高分”改为搜索安全候选或合理分差内的前若干候选。
4. 增加完整捕食链暴露、关键落点、孤狼目标、狼间接应、有效牺牲、目标8紧迫性和重复趋势。
5. 将“包围快照”扩展为“局部计划是否持续推进”，减少每步换目标。
6. 终局评分加入距离：更快困狼优于拖到上限，更晚被吃到8优于立即崩盘。

产物：五画像配置、Hard V2实现、单元测试、旧新AI同局面对照和解释字段。

完成门槛：黄金局面全部通过；Normal/Hard不选被支配动作；Hard在围堵、牺牲和链式防守局面显著优于旧版，且没有通过纯拖延虚增表现。

### 第三批：受限多回合搜索与性能

**解决问题**：更完整的候选仍需要稳定、多步且可在浏览器运行的推演。

开发内容：

1. 使用完整回合而非单个动作作为搜索语义，正确覆盖狼连续捕食和主动结束。
2. 增加动作排序、换位缓存和迭代加深；优先计算立即捕食、困狼和高风险候选。
3. 节点预算保持确定性；时间预算只作为生产安全兜底，不让同种子因设备速度产生不同走法。
4. 记录搜索完成深度、节点、降级原因和耗时分布。
5. 在桌面和移动视口构建中验证AI行动不冻结交互；必要时再决定是否移入Web Worker。

产物：搜索元数据、性能基线、降级测试和浏览器回归结果。

完成门槛：生产代表设备无明显卡顿；固定种子结果可复现；冬季批量运行不频繁降级；性能不足时安全回退而不是非法行动或随机超时。

### 第四批：24关重新校准

**解决问题**：AI增强会使旧胜率、难度和地图结论失效。

开发内容：

1. 按春夏秋冬重跑两种以上狼代理、固定种子和关键棋谱。
2. 按[24关AI难度曲线](../../产品核心/24关/24关学习曲线与重建设计.md)检查十二个两关阶段：前期是否突然过难、后期是否仍明显送子、优势羊是否会主动收网。
3. 优先处理 `winter-03`、`winter-04`、`winter-05`、`winter-06`，再检查两张夏季Hard关。
4. 先调AI，再判断地图/开局；只有AI已合理而关卡仍失衡时才调整生产地图。
5. 更新24关总览、单关风险和版本化验证记录，不复制运行数字到长期规范。

产物：新24关矩阵、代表狼胜/羊防守/异常棋谱、逐关风险和配置哈希。

完成门槛：无生产 `reject`；`winter-04` 长尾关闭；十二个阶段符合对应防守目标；冬季无连续两次可避免送子的代表局；每关至少有可理解狼胜路线，高难关也有真实羊防守压力。

### 第五批：Admin制作人验收闭环

**解决问题**：算法增强后，制作人仍需要看得懂和能复盘。

开发内容：

1. 单步诊断先显示通俗结论：安全、挡吃、合围、牺牲、送子或预算降级。
2. 展开证据再显示候选评分、搜索节点、最大捕食链和狼机动变化。
3. 批量验收突出决策质量、异常比例和建议动作，不把代理胜率称为玩家胜率。
4. “报告本局问题”导入后自动定位可疑羊步，并生成可下载复现包。
5. 增加正式画像与“即时防守基线”对照入口，方便制作人确认提升来自候选质量还是多回合规划。
6. 同时展示关卡AI画像与底层算法，避免把地图操作难度、玩家阶段和 `easy/normal/hard` 混为一个指标。

产物：Admin交互、通俗说明、导入复盘和使用说明。

完成门槛：制作人无需阅读代码即可回答“这关使用什么AI、它为何走这步、是否明显送子、下一步该改AI还是地图”。

### 第六批：发布门禁与收口

**解决问题**：避免以后AI或规则修改后悄悄退化。

开发内容：

1. 把快速羊决策护栏纳入常规发布门禁。
2. 把24关完整矩阵设为明确的慢速门禁或发布前命令，不能继续默认跳过且无人执行。
3. 固化随机种子、节点预算、配置哈希、算法版本和关键棋谱。
4. 执行核心、玩家、Admin、资源、构建和生产边界回归。
5. 更新任务状态；专项完成后归档验证证据并删除临时执行文档。

完成门槛：快速门禁和完整矩阵均通过；文档、Admin和生产配置一致；没有未解释P0/P1 AI问题。

## 五、验收指标

| 类别 | 必须记录 | 本期放行要求 |
|---|---|---|
| 动作质量 | 被支配动作、可避免送子、最大捕食链暴露 | Normal/Hard `chosenDominated=0`；冬季无连续可避免送子 |
| 主动防守 | 狼机动变化、受困狼、目标持续性、优势转化行动数 | Hard能在黄金局面形成并兑现局部合围 |
| 整局体验 | 胜方、终局、plies、首次捕食、最长连吃、重复与耗尽 | 无异常终局；长尾有解释；策略狼优于随机狼 |
| 难度梯度 | 春夏秋冬、Normal/Hard、旧新版本对照 | 前期仍可学习，冬季明显强于春季且不靠拖延 |
| 性能 | 节点、完成深度、耗时、降级率 | 固定种子可复现；生产不频繁降级；无可感知冻结 |
| 可解释性 | 行动标签、候选证据、问题归因 | 制作人能通过Admin完成复盘 |
| 玩家结果 | 重试、继续、通关尝试数和主观反馈 | 由上线前试玩和上线后数据验证，不用代理胜率替代 |

不预设一个统一狼胜率作为放行线。主线目标是玩家通过学习最终可赢；长期高阶内容只在产品扩展维护。

## 六、用户需要完成的事项

- **非阻塞反馈**：开发期间再遇 `winter-03` 或其他关卡连续送吃时，导出“报告本局问题”JSON；没有JSON也不阻塞自动反例搜索和Hard V2开发。
- **最终抽查**：Hard V2部署后试玩 `summer-02`、`summer-05` 和冬季6关；不要求重新人工跑全部自动矩阵。
- **每关只给一个判断**：太笨、合理、有压力但可赢、只会拖延或过难；出现具体疑问时再导出JSON。
- **产品确认**：确认主线防守和四季难度曲线是否满足首发。

其余24关基线、实现、模拟、棋谱、性能、Admin、门禁、归因、文档和生产复核全部由AI开发侧完成。

## 七、完成结果

- 五级画像已接入真实对局、重开恢复、问题报告、模拟和 Admin；24张关卡设计卡均显示对应画像。
- `sheep-ai-v2` 已实现严格劣着护栏、完整捕食链暴露、受困狼与机动评估、确定性节点预算和受限多回合搜索。
- Admin 已区分玩家难度、AI画像和底层兼容档位，可查看候选证据、搜索深度与降级原因，并自动定位导入问题包中的可疑羊步。
- 24关固定种子矩阵共运行720局，结果为 `15 pass / 9 review / 0 reject`；`chosenDominated=0`、可避免长链暴露 `0`、搜索降级 `0`，原 `winter-04 LONG_TAIL` 已关闭。
- 快速羊决策质量测试已进入 `pnpm release:check`；完整矩阵保留为 `pnpm test:candidate-acceptance` 与 `pnpm test:balance-matrix`。
- 剩余工作只有真实玩家体感，不再作为 AI 开发任务维护；抽查入口见[当前任务清单](../任务清单.md)。
