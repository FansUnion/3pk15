# T-043 文档任务治理整理

完成日期：2026-07-17

## 目标

建立清楚的任务生命周期：需求池 → 当前任务 → 已完成；同时将未来游戏内容与普通功能需求分开管理。

## 完成结果

- `任务清单.md` 只保留当前决定执行的玩家验证和发布验收。
- 新建 `需求池.md`，统一维护账号、悔棋、高级提示、AI难度、广告、分享、渠道和资源扩充等暂缓功能。
- `P-ACCOUNT.md` 保留为账号与云存档的需求池详细附件。
- 已完成的 Admin 生产工作台和地图候选验收器专项迁入“已完成”。
- `当前工作总览与执行索引.md` 简化为当前目标、事实、执行顺序和主要入口。
- `产品扩展` 只保留未来关卡、长期赛季、周期内容和独立挑战赛。
- 删除已被需求池吸收的功能型《辅助功能与商业化扩展方案》。
- 修复迁移产生的链接和旧任务状态引用。

## 文档边界

- `产品核心`：持续有效的规则、AI、24关、策略和验收标准，不因任务完成而迁移。
- `产品扩展`：未来关卡、赛季、挑战模式和长期游戏内容。
- `任务/需求池`：暂不执行的功能、平台、资源和商业化需求。
- `任务/任务清单`：现在执行或紧接着执行的任务。
- `任务/已完成`：关闭任务的必要证据和专项实施记录。
- `历史参考`：本次不整理。

## 后续规则

新需求不得同时出现在需求池和活动任务清单；任务完成后必须从活动清单移除。当前工作总览不再维护完成历史或第二套任务状态。
