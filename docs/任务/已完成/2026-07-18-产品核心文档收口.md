# 产品核心文档职责与收口审计

> 状态：2026-07-18 已完成文档编号、日期专项迁移、24关证据收口和链接检查。本文作为本次整理的决策记录，不再承担当前任务状态。

## 一、结论

`产品核心` 的主干逻辑已经形成，但目录仍混有“长期规范、日期专项、历史诊断、当前状态”四类内容。真正的问题不是文件数量本身，而是两份日期专项继续放在核心入口、两份长期标准没有编号，以及逐关设计卡重复保存会过期的模拟数字。

已落实的最终结构：

| 编号 | 长期核心文档 | 职责 |
|---|---|---|
| 01 | 核心产品定义 | 产品承诺、主线边界、首发质量门槛 |
| 02 | 关卡与平衡设计标准 | 地图设计、模拟、复盘和验收方法 |
| 03 | 玩家教学与策略系统 | 帮助、局内指导、策略标签和教学边界 |
| 04 | 奖励、皮肤与任务设计 | 奖励循环、收藏、任务和经济边界 |
| 05 | 产品体验与商业化原则 | 玩家旅程、商业化顺序、跨平台和反馈原则 |
| 06 | 视听资源与场景反馈验收标准 | 棋子、皮肤、棋盘、动画与声音的场景反馈放行标准 |

这六份文档分别回答不同问题，作为长期核心规范保留。原先两个未编号文档已编号为 05、06。

| `02-关卡与平衡设计标准.md` | 基本准确 | 缺少目标 8/9/10 边际价值指标、当前证据版本规则 | 补充方法，不复制本次具体数据 |

## 三、两个 0718 文档是否还要保留

### 0718 用户体验与商业化专项

不建议继续留在 `产品核心`。原因：

- 长期原则已经由 `05-产品体验与商业化原则.md` 承担。
- 当前问题和进度应该由第二批专项及任务清单承担。
- 其状态表已有不准确内容，会让读者误以为跨季节和 Admin 已收口。

处理结果：原文件已迁入 `任务/已完成`，明确标记为历史专项；当前状态回归任务清单。

### 0718 皮肤视觉诊断与修订记录

有历史证据价值，但不属于长期核心规范。长期标准由 `06-视听资源与场景反馈验收标准.md` 承担；具体资源事实由 SVG、音频和代码承担。原文件已迁入 `验证记录`，用户最终审美与试听仍由当前体验任务承接。

## 五、执行结果

1. 两个未编号长期标准已编号为 05、06，核心 README 当时改为六份长期规范；第二批专项完成后又按新的长期维护需要增加了第07份界面文案与交互标准。
2. 0718 用户体验专项已迁入 `任务/已完成`；长期原则由 05 承担，当前状态由任务清单承担。
3. 0718 皮肤诊断已迁入验证记录；长期视觉方法由 06 承担。
4. 24 张设计卡的旧矩阵、旧技术复选框和过时下一步已收口为版本化证据引用、当前自动状态与人工试玩重点。
5. 全仓旧文件名引用已检查；产品核心、第二批专项和本次迁移记录的 Markdown 相对链接全部有效。其他目录的历史失效链接不属于本次范围。

当时判断文案标准不单独建文档。第二批全流程审计进一步发现，玩家语言、Admin证据语言、动态数字口径、终局主次层级和无障碍状态横跨多个领域，继续分散维护会产生冲突，因此该判断已由[第08份长期标准](../../产品核心/08-界面文案与交互验收标准.md)替代。
