# 生产环境版本与功能一致性专项

> 本专项用于回答“生产是否部署最新版本、哪些能力真实生效、哪些只是本地完成”。功能长期标准仍由产品核心文档维护；本文在本批部署验证完成后归档。

## 一、2026-07-19 初始结论

生产环境已经部署提交 `3905e71`，不是旧版本：

- `/zh/settings` 已出现14类声音试听、单狼受困等上一批新增内容；
- 线上14个 WAV 与提交 `3905e71` 本地文件的 SHA-256 全部一致；
- 玩家站首页、设置、四季、玩法、关卡说明、对局入口、装扮、任务、隐私、robots、sitemap 和 llms 等17个公开入口均返回200；
- Admin 登录页返回200，根路径和受保护的 Admin、AI、关卡、皮肤页面返回307访问门跳转，符合保护设计。

因此用户感到“很多没生效”的主因是产品实现未达预期，不是 Vercel 没有部署。

## 二、发现与处理

| 级别 | 问题 | 代码事实 | 处理结果 |
|---|---|---|---|
| P1 | 多数声音过小 | 旧样本峰值跨度约 `-2.5dB` 至 `-20.1dB`，播放器还统一乘以0.7 | 已按场景归一化：核心动作约 `-2.5~-5dB`，轻交互约 `-6~-10dB`；播放增益调为0.85 |
| P1 | 首页同时显示“开始冒险”和“继续” | 静态第一关主按钮始终存在，继续入口再读取本地存档 | 已改成单一主入口，并明确区分开始、继续、下一关和最终关重玩 |
| P1 | “继续”含义不清 | `lastPlayedLevelId` 在进入对局时更新；最近关已通关时旧代码静默改为下一关 | 未通关显示“继续 · 最近进入关”；已通关且有下一关显示“下一关”；全通显示“再次挑战” |
| P2 | 四季关卡页互不关联 | 单季页只有返回四季总览，没有季节间导航 | 已在底部增加四季切换；当前季、可进入季和锁定季状态分开 |
| P2 | 无法证明线上资源是否对应当前提交 | 原来只能凭页面观感判断 | 新增 `pnpm audit:production:player` 和严格模式，检查17个入口与14个声音哈希 |

狼移动和羊移动不是同一声音：分别使用 `step.wav` 与 `sheep-step.wav`。旧版羊步峰值约 `-11.9dB`、狼步约 `-4.4dB`，播放后差异主要表现为羊步过轻；新版分别归一到约 `-5dB` 与 `-3.5dB`，仍保留材质和重量差异。

## 三、首页续玩契约

| 存档状态 | 主按钮 | 目标 |
|---|---|---|
| 从未进入关卡 | 开始冒险 | `spring-01` |
| 最近进入关尚未通关 | 继续 · 关卡名 | 最近实际进入的关卡 |
| 最近进入关已经通关且存在下一关 | 下一关 · 关卡名 | 正式顺序中的下一关 |
| 已通关最终关 | 再次挑战 · 冬日06 | `winter-06` |

玩家已有进度且最近关不是第一关时，主按钮下保留低优先级“从第一关开始”。它不能与续玩争夺主按钮层级。

## 四、自动检查边界

`pnpm audit:production:player` 用于报告17个公开入口、14个正式声音和3张高级棋盘；`pnpm audit:production:player:strict` 用于部署后门禁，任何入口失败或资源哈希落后都会失败。

自动检查可以证明路由、页面内容和静态资源版本，但不能证明手机外放听感、本地存档驱动行为、触控与音频授权体验，以及 Admin 登录后的业务操作感。这些继续通过真实浏览器反馈，出现问题后由AI负责复现和修复。
