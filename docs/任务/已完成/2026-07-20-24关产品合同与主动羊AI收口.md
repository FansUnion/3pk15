# 24关产品合同与主动羊 AI 收口

> 状态：代码、Admin、24关合同、720局 V5 候选门禁和960局 V5 玩家画像均已完成并归档。最多8局定向体验属于非阻塞真人体感确认；出现问题时按问题包重开修复，不再把它维护成未完成开发任务。

## 一、为什么需要本专项

此前已经完成规则一致性、地图重建、V3安全护栏、720局候选矩阵、192局玩家画像、教师AI和问题包回归。这些工作证明当前版本是可信基线，但主要回答“有没有结构错误、明显送子和已知漏洞”，没有完整回答：

- 羊是否像一个主动、有计划、可反制的娱乐对手；
- 五档差异是否不只是搜索深度和随机范围；
- 24关是否逐关具有明确玩家需求、对手意图和独特体验；
- 自动化能否先发现被动拖延、逐回合失忆、错误牺牲和优势不收网，而不是等待制作人试玩发现。

因此本专项不是推翻24关，而是把“玩家合同 -> 关卡合同 -> AI行为 -> 自动证据 -> 少量真人判断”连成闭环。

## 二、玩家最终应获得什么

1. 春季快速成功，但羊会合理回应，不排队送子。
2. 夏季开始遇到协同封路、孤狼围困和路线反制，必须使用两狼或三狼。
3. 秋季面对地形、牺牲、跨区和长链博弈，失败能归因到具体决策。
4. 冬季羊会持续针对目标、拆固定套路并主动收网；熟练玩家仍有可学习胜路。
5. 四季不是“同一种羊越来越谨慎”，每两关都能看出不同对手意图。
6. 玩家不用理解算法，但能从盘面说出“羊正在围哪只狼、封哪条路、我怎样反制”。

## 四、执行批次

### 第一批：产品模型与命名接入

- 保留五个稳定`aiProfile`代码ID，玩家/Admin名称改为教学陪练、警觉羊群、协同羊群、谋略羊群、老练羊群。
- 新增独立`aiStyle`与本关`opponentIntent`配置，不开发24套算法。
- 首发风格限定为封路、合围、分散诱导、战术交换、反固定猎手五类。
- Admin同时显示玩家操作难度、对手能力、行为风格、本关意图和玩家反制。

完成条件：24关均有三维配置；名称不再暗示羊只会被动防守；代码、Admin和设计卡合同一致。

### 第二批：主动意图与合理牺牲

- 为每次羊决策生成可解释意图：目标狼、目标通道、当前阶段、候选动作和换目标原因。
- 第一版优先使用从公开局面确定性推导的目标持续评分；若无法稳定保持计划，再引入独立`AiPlanState`。
- 若引入计划状态，必须随存档、问题包、回放和模拟一起序列化，重开必须清空，不能形成隐藏且不可复现的状态。
- 允许被更深搜索支持的战术交换，不能因下一回合多暴露一只羊就一律过滤；交换必须换来可测的出口封锁、接应切断或终局收益。
- 优势局提高收网进度，劣势局允许换线止损，重复局面不作为主动目标。

完成条件：代表局面能连续数步针对同一目标；牺牲有交换解释；优势不靠往返结束。

### 第三批：风格行为与24关绑定

- 封路型优先控制关键落点和通道。
- 合围型持续压缩同一只低机动狼。
- 分散诱导型主动拉开羊阵并迫使狼换线。
- 战术交换型在可证明收益时接受一只羊的代价。
- 反猎手型识别连续得分狼和重复通道，优先拆线。
- 每关只指定一个主要风格和一个辅助行为，不能同时堆满所有能力。

完成条件：盲看代表棋谱时能区分主要风格；相邻关不只是同一AI换地图。

### 第四批：自动验证升级

动作层继续检查合法、严格劣着、完整捕食链、预算和复现；新增：

| 指标 | 解决的问题 |
|---|---|
| 主动压力率 | 羊行动是否实际减少目标狼机动或控制关键点 |
| 目标持续性 | 连续羊回合是否围绕同一合理目标推进 |
| 困狼进度 | 优势是否逐步转成真实终局 |
| 牺牲交换收益 | 丢羊是否换来封锁、节奏或终局价值 |
| 反猎手成功率 | 是否拆解同一狼和同一通道的重复得分 |
| 无进展比例 | 是否只保子、横移和拖延 |
| 风格辨识度 | 各风格是否产生可观察差异 |

- 四类狼方画像由每关2种子扩大到至少10个固定种子，共至少960局。
- 结果按季节、两关阶段、能力和风格汇总，不把代理胜率冒充真人胜率。
- 任何红线输出责任动作、替代动作、意图变化和棋谱。

完成条件：确定性问题由自动化和教师先发现；曲线异常能归因到AI、地图、代理或证据不足。

### 第五批：24关复核与单变量调优

按[24关学习曲线与产品合同](../../产品核心/24关/24关学习曲线与产品合同.md)逐关检查。先修AI兑现，再判断地图；合格AI下仍无可读胜路、无新鲜感或天然偏置时，才单变量调整开局、岩石或参数。

完成条件：

- 24关无明显送子、纯拖延、逐回合失忆和固定猎手漏洞；
- 每关保留可理解狼胜棋谱与有效羊方施压棋谱；
- 春夏秋冬形成玩家成长和对手成长，而不是机械胜率阶梯；
- 所有`review`风险有代码归因和明确真人观察问题。

### 第六批：Admin与最多8局定向验收

- Admin用制作人语言回答：羊在针对谁、执行什么风格、这一手改变了什么、有没有更合理替代。
- 自动化按风险重新选择每季最多两关；未变化且证据稳定的关卡不重复试玩。
- 制作人只判断“有灵性、可反制、有压力但可赢、只会拖延、过难”，不负责发现算法底线错误。

完成条件：最多8局完成最终乐趣、压力和可读性判断，随后停止首发前假设驱动调优。

## 五、不做与停止条件

本期不直接训练神经网络，不追求完整全局求解，不做隐藏动态难度，也不把主线变成大师赛。

以下全部满足后结束本专项：

- 三维对手配置和24关合同一致；
- 主动意图、风格和新指标门禁通过；
- 至少960局四画像矩阵与代表棋谱通过；
- 无稳定复现的低级送子、固定套路、错误牺牲和优势不收口；
- 系统已经生成不超过8局的定向体验清单；制作人可在自然试玩中抽查，不阻塞自动侧收口；
- 后续调整只由真实完成、重试、继续、回访和新问题包触发。

这条停止线用于终止“制作人不断试玩、AI不断局部打补丁”的循环。
