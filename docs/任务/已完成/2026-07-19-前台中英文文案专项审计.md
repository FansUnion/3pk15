# 前台中英文文案专项审计

> 状态：已完成。2026-07-19 已按审计结果完成代码、24 关提示、自动门禁和双应用构建收口。本文保留问题与处理依据；长期表达原则只以[界面文案与交互验收标准](../../产品核心/07-界面文案与交互验收标准.md)为准。

## 完成结果

- C-01 至 C-29 已全部处理：规则事实、和局名称、羊移动描述、难度口径、装扮入口、任务异常、分享结果、无障碍名称和动态数量均已统一。
- 英文玩家端不再使用 `gap-eat / gap-rush / gap-cut` 或 `eats` 描述规则，正式用语为 `leap capture / capture / chain capture`。
- 24 关中英文三级提示统一从一个生成入口产生，并逐关检查盘面信号、行动目标、策略说明、重复、长度和内部术语。
- 新增玩家文案契约测试，覆盖可见字符串禁词、岩石单复数和 `24 × 2 × 3` 条动态提示；测试只检查玩家可见值，不误判内部对象键。
- 产品规则、地图、AI 和胜利目标未因文案收口而改变。

## 一、审计范围与结论

第一轮按信息源检查 `apps/player-web/i18n/messages.ts`、24 关中英文名称与说明、策略库、任务、皮肤、分享结果和公开 `llms.txt`。第二轮按玩家旅程重新检查首页、玩法帮助、季节入口、关卡说明、局内提示、连续捕食、终局、奖励、图鉴、任务、设置与隐私。第三轮检查硬编码文本、读屏标签、单复数、异常错误和动态变量拼接后的完整句子。

当前结论：中文主流程基本能完成操作，但仍有 **4 项规则或产品事实误导、7 项高频术语/表达障碍、18 项一致性、异常状态与自然度问题**。英语问题更集中，自造的 `gap-eat / gap-rush / gap-cut` 不应作为正式规则术语。建议先完成 P0/P1，再统一 24 关英语说明和低频页面。

## 二、P0：会让玩家形成错误理解

| ID | 当前问题 | 代码事实与影响 | 推荐修改 |
|---|---|---|---|
| C-01 | 中英文帮助仍称“秋季密岩、冬季空盘硬仗” | 当前秋季为少量岩石形成斜门、通道和路线岛；冬季为空盘与稀疏雪岩混合。旧文案会让玩家预期与盘面冲突。 | 中：`秋季训练通道与时机，冬季考验站位与三狼配合。` 英：`Autumn tests routes and timing; winter tests spacing and teamwork.` |
| C-02 | 首页称春季 `gentle flock / 简易羊群` | 春季正式配置为 `normal`，不是 Easy AI。玩家可能把教学开局误认为 AI 档位承诺。 | 中：`基础走位 · 逐步学习`；英：`Learn movement · build confidence`。难度另用“玩家挑战难度”表达。 |
| C-03 | `Stalemate / 僵局` 同时指三次重复和行动耗尽 | 在常见棋类语境中 stalemate 常指无合法行动；本游戏狼无合法行动却是羊胜，会造成规则冲突。 | 统一结果名为中 `和局`、英 `Draw`；副文案继续分别解释“三次重复”或“达到行动上限”。 |
| C-04 | 帮助用“不能往第 1 行退 / cannot retreat toward row 1”解释羊移动 | 玩家棋盘没有显示行号，也不一定知道第 1 行在哪一侧。规则代码实际是羊可横移或向狼方前进，不能回到初始边。 | 中：`羊可横向移动或朝狼方前进，不能退回开局所在的一侧。` 英：`Sheep move sideways or toward the wolves, never back toward their starting edge.` |

## 三、P1：高频且不够通俗

| ID | 当前问题 | 用户障碍 | 推荐修改 |
|---|---|---|---|
| C-05 | 英语混用 `gap-eat`、`gap-rush`、`gap-cut` | 都是项目自造词，英语玩家无法从既有棋类经验理解，而且同一规则用了三个名字。 | 正式术语统一为 `leap capture`；首次解释：`wolf — empty point — sheep in one straight line; the wolf lands on the sheep.` 后续只用 `capture` 或 `chain capture`。 |
| C-06 | 英语混用 `eat/eats` 与 `capture` | `eat 8` 偏口语，计数、按钮和规则页又使用 capture，降低规则稳定性。 | 英语全部使用 `capture / captures`。中文规则动作保留易懂的“隔空吃”，成绩与目标使用“捕获”。 |
| C-07 | 中文策略提示大量使用“建立接触、机动、兑现、覆盖” | 对设计人员准确，但普通玩家不容易转成下一步操作。 | 首次教学改成动作语言，例如“让一只狼靠近羊群，另外两只守住两侧”“这只狼走完后仍要有退路”。抽象策略名可留在进阶提示。 |
| C-08 | `Difficulty {n}/5 / 难度 {n}/5` 未说明口径 | 玩家可能理解为羊 AI 档位，但它实际表示地图与操作挑战。 | 中：`挑战难度 {n}/5`；英：`Challenge {n}/5`。Admin 继续单独展示羊 AI 强度。 |
| C-09 | 新手指导允许“点红圈羊或中间空点” | 虽然交互兼容两种点法，但同时教两种入口增加首课认知负担。 | 首课只教“点红圈中的羊”；中间空点保留为可用快捷操作但不主动说明。 |
| C-10 | `报告本局问题` 完成提示直接要求发送 JSON | 对普通玩家是技术术语，也没有说明发到哪里；对项目所有者又确实需要该文件。 | 中：`问题记录已下载。反馈时请附上这个文件。` 英：`Game report downloaded. Attach this file when sending feedback.` Admin 文档再说明 JSON。 |
| C-11 | 中文 `图鉴` 页面实际承担预览、兑换和装备 | “图鉴”容易被理解为只看收藏，不能预期可兑换和换装。 | 导航建议改为中 `装扮`、英继续 `Skins`；页面内可保留“已收集图鉴”作为子概念。 |

## 四、P2：双语一致性和自然度

| ID | 当前问题 | 推荐修改 |
|---|---|---|
| C-12 | 英文分享图无论界面语言都写 `{n} actions`，中文结果卡中英混排 | 根据 locale 输出 `次行动 / actions`；与分享文字使用同一格式函数，避免两处维护。 |
| C-13 | 英文隐私页向用户暴露 `localStorage` 和 `WHOIS`，并写 `when published` | 正文改为“stored only in this browser”；联系方式使用当前真实公示渠道。技术名可放折叠详情，不面向普通玩家。 |
| C-14 | 中文 FAQ 使用“一期”，英语使用 `this version` | “一期”是内部项目语言。改为“当前版本暂无账号和云存档”。 |
| C-15 | 秋季入口写 `Same AI tier / 同档 AI`，但玩家不需要先理解档位 | 改为体验职责：中 `路线判断 · 把握时机`；英 `Read routes · time the attack`。 |
| C-16 | 24 关英语说明存在 `representative funnel`、`clean chain`、`own both ends` 等设计稿口吻 | 保留地图特色，但改为直接行动，例如 `Use one wolf to draw the flock out while the other two guard the exits.` |
| C-17 | 中文在“捕食目标、捕获、隔空吃、吃子线”之间切换 | 建议词典：规则动作=`隔空吃`，连续动作=`连吃`，目标/计数=`捕获`，策略路线=`捕食路线`；玩家端不使用“吃子线、兑现捕食”。 |
| C-18 | 中文页面硬编码英文眉题 `Collection`、`Fangrush field guide` | 将眉题纳入消息树：中 `装扮收藏 / 猎场指南`，英 `Collection / Fangrush field guide`。品牌名 Fangrush 本身可保留。 |
| C-19 | 棋盘无障碍名称始终是英文 `Fangrush board / Rock terrain / Sheep / Wolf` | `BoardSvg` 接收本地化无障碍标签；中文读作“棋盘、岩石、第几行第几列、羊、狼”，英语保持自然表达。装饰性预览可隐藏重复朗读。 |
| C-20 | 英文动态岩石数量会生成 `1 rocks` | 不继续用一个 `{n} rocks` 模板兼容所有数字；增加单复数格式，`1 rock / 2 rocks / no rocks`。中文统一为“无岩石 / 1 枚岩石 / n 枚岩石”。 |
| C-21 | 任务领取异常会直接显示 `unknown quest / already claimed / incomplete` | core 返回稳定错误码，玩家端映射为本地化可行动提示，例如“任务状态已更新，请刷新后重试”，不得直接显示内部英文。 |
| C-22 | 季节棋盘写“通关对应季节解锁 / Clear its season to unlock”，与代码不符 | 代码在季节入口解锁时就赠送对应棋盘，并非打完整个季节后才给。改为中 `解锁对应季节后获得`，英 `Unlocked when this season opens`。 |
| C-23 | 任务页顶部只显示一个无标签的通用碎片数字 | 增加币种名称或可读图标标签：中 `通用碎片 {n}`，英 `{n} universal shards`，避免玩家不知道数字含义。 |
| C-24 | 首页主图 `alt` 固定为英文，中文读屏仍读英语 | 主图替代文本随 locale 输出；若相邻正文已经完整描述同一内容，则改为空 `alt` 避免重复朗读。 |
| C-25 | 三层关卡提示由多个字段临时拼接，中英文并非逐关等价内容 | 为每层限定职责和长度：观察=一个盘面信号，目标=本关要做到的动作，策略=下一步计划。英文不能用通用策略目标替代中文逐关目标；24关生成后逐句验收。 |
| C-26 | 奖励目标显示 `Next skin: ... costs 50 / 下一件皮肤：...需要 50` | 动态数字缺币种。改为英 `50 universal shards`，中 `需要 50 通用碎片`；从统一货币格式函数生成。 |
| C-27 | 双倍状态显示 `Double drop {t} / 双倍掉落 {t}` | `{t}` 是剩余时间，但文案没有说明。改为英 `Double shard drops · {t} left`，中 `碎片双倍 · 剩余 {t}`。 |
| C-28 | 结果详情可能生成 `first capture none / 首次捕食 无` | 无捕获局应输出完整短语：英 `no captures`，中 `本局未捕获`；有首吃时再显示“首次捕获发生在第 n 次行动”。 |
| C-29 | 玩家提示、关卡说明和策略库可各自表达同一策略，缺少渲染后契约 | 增加生成结果测试：禁止双重前缀、连续重复句、内部字段名和超长提示；测试中英文 24×3 条完整提示，而不是只测消息树字段存在。 |

## 五、逐关英语优先修改清单

| 关卡 | 当前高风险表达 | 推荐方向 |
|---|---|---|
| spring-01 | `try one gap-eat` | `make your first leap capture` |
| spring-02 | `clean a gap-eat` | `move around it and create a capture` |
| summer-01 | `gap-rush line` | `breakthrough route` |
| summer-02 | `gap-cut in` | `enter through an open line` |
| summer-03 | `representative funnel` | `an asymmetric funnel`，并直接说明三狼分工 |
| autumn-03 | `open cross exposes long chains` | 说明“先看最终落点，再决定是否继续连吃” |
| winter-01 | `tear the surround` | `break the flock's formation` |

其余 16 关未发现规则级错误，但应在执行时按同一词典做自然度复核，避免只替换关键词后留下生硬句子。

## 六、推荐执行批次

1. **规则事实收口**：修正四季描述、羊移动方向、和局名称和难度口径；补对应中英文测试。
2. **核心术语收口**：移除所有玩家可见 `gap-*` 与英语 `eats`，建立中英术语常量或测试黑名单。
3. **全流程自然化**：调整首页、帮助、局内、终局、报告、装扮、任务、隐私和分享卡；覆盖正常、失败、重复点击和数据刷新等异常状态。
4. **24 关英文重写**：保持地图事实和教学目标不变，只把设计语言改成玩家可执行的动作语言；同步关卡卡片契约。
5. **验收**：运行文案契约、24×3 动态提示快照、玩家测试和生产构建，并在手机、平板、桌面各检查一次中英文长文本、单复数和换行。

## 七、本轮不建议做的事

- 不改变游戏规则、胜利目标、AI 或地图来配合旧文案；应让文案服从当前代码事实。
- 不把所有策略术语删除。策略名可以保留，但首次提示必须先给可执行动作。
- 不在代码、单关卡和规范文档各维护一份术语表。长期术语标准只写入 `07-界面文案与交互验收标准.md`，本专项完成后移入已完成记录。

## 八、已确认的产品用语

- 英语正式规则名采用 `leap capture`，中文继续采用“隔空吃”。
- 玩家结果统一使用“和局 / Draw”，不再使用“僵局 / Stalemate”。
- 玩家导航将“图鉴”改为“装扮”。
- 季节棋盘按真实逻辑表述为“解锁季节后获得”，不再写成“通关整个季节”。
- 上述用语已经进入实现与长期规范；后续只根据真实玩家理解反馈调整自然度，不再恢复旧术语。
