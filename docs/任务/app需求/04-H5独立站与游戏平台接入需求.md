# Fangrush H5 独立站、Poki 与 CrazyGames 接入需求

> 三个 H5 构建共享玩家产品和核心游戏，但它们不是同一个发行包。独立站是自有渠道；Poki/CG 是受平台政策约束的门户渠道。

## 1. 三端策略结论

| 项目 | 独立站 | Poki | CrazyGames |
|---|---|---|---|
| 主要作用 | 品牌入口、自有账号与云存档、Web 长期入口 | 平台流量与广告验证 | Basic/Full 流量与广告验证 |
| 登录 | Fangrush Google，后续 Apple | Poki User | CrazyGames User |
| 云存档 | Fangrush Cloud | Poki Cloud Gamesaves | CrazyGames Data |
| 自有后端 | 必须 | 默认不接；平台授权后可用 Poki token 映射 | 可通过 User module 链接，但首选 Data |
| 广告 | 首版关闭 | 只用 Poki SDK | 只用 CG SDK |
| IAP | 首版关闭 | 首版关闭 | 未获平台邀请前关闭 |
| 包形态 | 可有 Next 服务端能力 | 可上传的静态游戏目录/ZIP | 可上传的静态游戏目录/ZIP |
| 外部请求 | 自主管理 | 默认受限，需批准 | 按平台允许列表与用途 |

准确结论：两个门户不是绝对禁止自有服务端，而是不允许把自有外部登录当作平台主登录。若使用自有后端，必须先用平台身份完成授权和映射，并遵守外部请求审批。对当前游戏，平台云已经足够，首发没有必要增加自有云复杂度。

## 2. 公共 H5 运行时

三个构建必须共用：

- `packages/game-core`、24 关、AI、奖励和 `SaveGame`。
- 玩家 UI、棋盘、输入、音频和本地化。
- `IdentityAdapter`、`StorageAdapter`、`AdsAdapter`、`LifecycleAdapter` 产品语义。
- 广告暂停/恢复、奖励幂等、存档迁移和错误降级。
- 设备/尺寸适配和核心自动测试。

三个构建必须分开：

- 平台 SDK、账号入口、云存储 provider、广告 provider。
- 外部请求 allowlist、环境变量和 CSP。
- 平台元数据、封面/缩略图、上传包和验收工具。
- 统计、分享 URL、静音和平台生命周期实现。

## 3. 当前工程差距

仓库已有 `build:standalone`、`build:poki`、`build:crazygames` 和 `GamePlatform/AdsContract/StoragePort` 雏形，但尚不能视为真实平台接入完成：

- 当前目标构建仍执行普通 `next build`，没有证明产物是门户可直接上传的静态目录。
- `NEXT_PUBLIC_ADS_PROVIDER=portal_sdk` 目前没有真实 provider，实际会落到 unavailable。
- `GamePlatform` 支持 lifecycle port，但 `getPlatform()` 尚未注入 Poki/CG 实现。
- 存档默认直接使用浏览器 localStorage，尚未接 Poki Cloud Gamesaves 或 CG Data。
- 平台账号、token 验证、静音、Inspector/Portal 测试尚无完成证据。

另一工具接 H5 广告时必须在现有契约上实现 adapter，并补齐上述证据，不能在页面里散落 `window.PokiSDK` 或 `window.CrazyGames.SDK` 调用。

## 4. 静态门户包要求

### 4.1 推荐方案

优先让同一玩家运行时支持“独立站服务端构建”和“门户静态导出”双模式：

- 独立站保留 Next 服务端/动态能力。
- Poki/CG 目标启用静态导出或等价静态打包，产出含 `index.html` 的目录。
- 构建期关闭 middleware、API route、服务端认证和不兼容静态导出的动态能力。
- 动态关卡路由预生成；所有运行资源可由相对路径加载。

`player-runtime` 按总体架构先抽为共享包。若抽取后 Next 双模式仍无法稳定产生门户静态包，再新增轻量 `apps/portal-web` Vite 壳消费同一 runtime；不复制 React 页面和游戏 store。

### 4.2 静态产物门禁

- 无 Node/Next 服务端依赖、Admin、`.env`、source map、密钥和内部报告。
- 无独立站 Google/Apple 登录、Fangrush refresh cookie 或自有广告代码。
- 入口、资源、动态路由和音频使用平台包内可访问路径。
- 断开本地开发服务器后仍可由静态服务器完整游玩。
- 包体、文件数、外部请求和 CSP 通过目标平台检查。
- 构建报告记录 target、revision、SDK provider 和能力开关。

## 5. 独立站 H5 需求

### 5.1 基础产品

- 保持无需登录可玩和本地存档。
- 保持英文/中文、SEO、分享链接和品牌页面。
- 首版无广告、无 IAP，不加载 Poki/CG/AdMob SDK。
- 浏览器存储被禁用、配额不足或损坏时友好降级，不白屏。

### 5.2 账号与云

- 按账号文档接 Google、Fangrush 会话、云存档、合并、删除和导出。
- 登录入口不进入 portal 构建；不能只靠 CSS 隐藏，必须由构建能力关闭代码路径。
- API、数据库和 provider 故障不阻止本地游玩。
- 提供云同步状态、重新登录和冲突恢复。

### 5.3 站点与运维

- 正式域名、HTTPS、OAuth origin/redirect、API 域名和 CORS 配置冻结。
- 隐私政策、条款、支持、账号删除页面公开可访问。
- 部署健康检查、错误监控、数据库备份和恢复演练。
- API 与站点可独立发布，但版本兼容有明确窗口。
- Cookie/同意提示根据真实分析、广告和地区适用性配置，不做空洞弹窗。

### 5.4 独立站验收

1. 无登录完整游玩和刷新恢复。
2. Google 登录、跨浏览器恢复、冲突、登出和删除通过。
3. 禁止第三方 cookie/存储、弱网、API 500 时本地路径可用。
4. 独立站产物不含门户 SDK；门户产物不含独立站 auth。

## 6. Poki 接入需求

### 6.1 准入与商务

- 获得 Poki for Developers 访问并确认实际合同、排他范围、分成、更新和终止条件。
- 以提交时官方政策和签署合同为准，不把本地摘要当合同事实。
- 完成开发者/收款/税务资料和游戏权利确认。

### 6.2 SDK 初始化与生命周期

- 尽早加载并初始化 Poki SDK；初始化失败也应继续加载游戏。
- 游戏可玩后只调用一次 `gameLoadingFinished()`。
- 玩家进入/恢复真实操作时调用 `gameplayStart()`；菜单、暂停、结算、失败时调用 `gameplayStop()`。
- 事件顺序幂等并通过 Poki Inspector 日志。
- 遵守平台静音、输入和父页面滚动边界。

### 6.3 Poki 广告

- 只使用 `commercialBreak` 和 `rewardedBreak`，不混入其他广告网络。
- `commercialBreak` 只在自然停顿机会请求；不实现自己的曝光频率计时器，是否展示由 Poki 决定。
- `rewardedBreak` 必须由用户主动操作触发，奖励清晰且只需一条广告。
- 广告开始时暂停输入/AI/音频；完成、跳过、无填充、错误时恢复。
- 只有 rewarded promise 明确成功才发奖励；普通继续路径始终存在。
- AdBlock 下核心游戏完整可玩，不用遮罩强迫关闭。

### 6.4 Poki 账号和云存档

- User Accounts 能力可用时，`getUser()` 查询状态，`login()` 只由用户点击触发。
- 登录成功可能刷新页面；加载后重新查询用户并从稳定存档恢复。
- 登录用户的 Cloud Gamesaves 由 SDK 透明同步；存档保持在平台限制内。
- 仅本地诊断/临时键按平台规则使用忽略前缀，避免上传无意义数据。
- 若平台账号不可用或用户拒绝，隐藏登录入口并使用本地存档。
- 只有明确需要跨渠道且获得 API key/外部请求批准后，才把 Poki 短时 token 发到 Fangrush 服务端验证；不保存短时 token。

### 6.5 Poki 产品与包要求

- 支持桌面、手机和平板，控制提示随设备变化。
- 处理 iframe 缩放、16:9 可玩区域、超宽/窄屏和父页滚动。
- 按政策移除不允许的外链、第三方品牌启动页和外部广告。
- 私密模式/localStorage 不可用时不崩溃。
- 文本较多时扩展本地化；首发至少英语完整，中文包不影响英文布局。
- 提供并最终确认静态/动画缩略图、说明和真实玩法素材。

### 6.6 Poki 验收

1. Inspector 初始化、loading、gameplay、commercial、rewarded 事件全部通过。
2. 广告成功/失败/取消、AdBlock、静音和输入暂停通过。
3. User Accounts/Cloud Gamesaves 若启用，游客、登录、刷新和换设备通过。
4. 包体、外部请求、相对路径、缩放、桌面/移动和平板通过。
5. 正式包不含 Mock、外部登录、外部广告和 Admin。

## 7. CrazyGames 接入需求

### 7.1 发布阶段

- Basic Launch 主要验证平均游玩时长、进入玩法转化和留存，广告可能关闭。
- Full Launch 才完成完整 SDK、广告和适用的平台功能。
- 需求和验收要区分 Basic/Full，不能因为 Basic 无广告把 adapter 误判为故障。

### 7.2 SDK 与生命周期

- 加载当期 HTML5 SDK 并等待 `init()`；平台外环境安全降级。
- 资源加载开始/结束调用 loading 事件。
- 玩家开始/恢复真实游玩调用 `gameplayStart()`；菜单、暂停、切关和结算调用 `gameplayStop()`。
- 遵守平台 `muteAudio` 及其变化，优先于游戏内音量设置。
- Full 阶段按平台要求上报游戏完成进度。

### 7.3 CrazyGames 广告

- 只用 CG `midgame` 和 `rewarded` 视频广告；首版不做 Banner。
- `midgame` 只在死亡/失败、关卡完成等自然断点请求。
- `rewarded` 由玩家主动触发，奖励只在 `adFinished` 发放。
- `adStarted` 暂停输入/AI/音频；`adFinished` 和 `adError` 都恢复。
- 显式处理 Basic 禁用、unfilled、AdBlock、cooldown 和其他错误。
- 不自行保证一定曝光；平台常规冷却由 SDK 决定。
- AdBlock 下核心游戏完整可玩。

### 7.4 CrazyGames 账号与 Data

- 不显示外部 Fangrush 登录；需要账号时使用 CrazyGames User。
- 推荐 Data module 作为 CG 构建唯一进度存储 adapter：游客写平台代理的 localStorage，登录后平台自动迁移/同步。
- 提交时启用对应 Progress Save 选项，否则 Data module 可能不可用。
- SDK 初始化时先加载 Data，再初始化玩家 save store；禁止先用普通 localStorage 写一套、随后无序覆盖 Data。
- 单用户数据不超过 1MB；考虑平台 1 秒 debounce、极端情况下更长延迟，关键节点尽早写入。
- 已发布本地存档迁移到 Data 时做一次性键迁移并标记完成。
- 若使用 Fangrush 后端，只通过 CG User token/账户链接流程映射，处理游客登录、登出和账号切换。

### 7.5 CrazyGames IAP

- 当前按平台开放/邀请能力处理；未明确获得权限前标记不适用。
- 不把 Google Play Billing、StoreKit 或独立站支付 UI 放入 CG 包。
- 未来若获开放，单独补平台商品、登录、Xsolla/支付、退款和权益设计，不复用 App 商店凭证。

### 7.6 CrazyGames 包、质量与素材

- 重新核对提交时的初始下载、总包、文件数和外部资源限制；当前本地基线为初始不超过 50MB、总包不超过 250MB、文件数不超过 1500。
- 支持桌面和移动；桌面横屏、手机触控、iframe 多尺寸可玩。
- 新玩家尽快进入真实玩法，不用长文本和多层菜单阻挡。
- 英语 UI 完整，内容满足平台年龄/质量要求。
- 三类封面和横竖预览视频使用真实游戏画面；仓库已有候选，仍需人工确认和上传。

### 7.7 CrazyGames 验收

1. Basic 包在广告禁用时完整可玩并有可观测漏斗。
2. Full 包 SDK、loading、gameplay、mute、progress 事件通过 Portal 测试。
3. midgame/rewarded 全回调、cooldown、AdBlock 和无填充通过。
4. Data 游客、登录、自动迁移、登出、换设备和 1MB 边界通过。
5. 包体、文件数、相对路径、外部请求、桌面/移动尺寸通过。
6. 正式包无 Mock、外部登录、外部广告和 Admin。

## 8. 三端优先级

| 阶段 | 独立站 | Poki | CrazyGames |
|---|---|---|---|
| H0 | 稳定本地版 | 准入、静态包 | Basic 静态包 |
| H1 | Google 登录 + Fangrush Cloud | SDK 生命周期 + 广告 | Basic 数据验证；Full SDK 生命周期 + 广告 |
| H2 | 删除/导出、监控和备份 | User Accounts + Cloud Gamesaves（能力开放后） | Data module + User（Full/能力开放后） |
| H3 | Apple Web 登录候选 | Poki token 映射自有后端（有跨端价值时） | CG token 映射自有后端（有跨端价值时） |

门户广告工作可以与独立站账号云存档并行；共同前置是静态玩家运行时、平台能力接口和一致的广告生命周期语义。
