# 任务系统与商业化联动专项计划

> 任务ID：`T-028`。状态：代码、自动化、Admin模型和本地浏览器验证已完成，等待用户确认任务提醒强度与奖励体感。本专项只维护本期诊断、批次、验收与完成状态；长期不变原则见[奖励、经济与商业化设计](../产品核心/04-奖励经济与商业化设计.md#231-任务系统的长期产品合同)。

## 一、专项要解决什么

当前任务系统可以自动累计并手动领取，但它还没有作为商业系统的一部分接受完整设计。若只增加红点和“领取全部”，会让更多玩家领取奖励，却可能更快耗尽皮肤目标、降低广告价值，并暴露未领取奖励在周期刷新时消失的问题。

本专项同时解决四件事：

1. 玩家无需接取任务，也不会因为没看见入口而丢奖励。
2. 任务能被自然发现，但不打断棋盘和下一关。
3. 任务、关卡、广告、重复通关和未来复盘消耗进入同一经济模型。
4. 任务支持学习、回访和收藏，不演变成签到压力、广告作业或低价值刷关。

## 二、实施前诊断事实

### 2.1 实施前任务奖励

| 周期 | 任务 | 当前奖励 |
|---|---|---:|
| 每日 | 今日对局1次 | 3通用碎片 |
| 每日 | 今日通关1关 | 5通用碎片 |
| 每周 | 本周通关3关 | 15通用碎片 |
| 每周 | 本周获得20通用碎片 | 10通用碎片 |

玩家不需要接取。开始对局、通关和获得碎片会自动累计；达标后必须进入任务页手动领取。任务页把“可领取”排在“进行中”和“已领取”之前。

### 2.2 实施前发现路径

- 首页有“任务”卡片，但没有可领取数量。
- 局内顶部菜单中的任务入口会显示“若干项可领取”，但玩家必须先打开菜单。
- 页脚有任务链接。
- 任务完成时没有即时但非打断式反馈，结算页也不显示本局完成了什么任务。
- 任务页没有“领取全部”。

因此当前体验很可能是：玩家先正常玩，某次偶然进入任务页，才发现已经完成多项任务。自动累计是正确的，但完成反馈和领取闭环偏弱。

### 2.3 实施前经济风险

任务理论周上限是：

```text
(每日3 + 每日5) × 7 + 每周15 + 每周10 = 81通用碎片/周
```

对比当前经济：

- 24关首次通关共240通用碎片；
- 三套角色主题成本为80、160、240，累计480；
- 首次通关激励广告每次通常增加10，24次最多再增加240；
- 当前经济模拟只比较无广告、半数广告和全部广告，没有把任务收入加入路径。

81碎片已经超过第一套霜狼的80成本。任务如果被显著增强发现性，真实碎片供给会增加，三档收藏可能比现有模型更早耗尽。

### 2.4 实施前任务与广告的隐性联动

“本周获得20通用碎片”使用的是通用碎片获得指标。当前广告追加通过同一通用碎片发放入口，也会推进该任务。因此任务文案没有要求广告，但看广告可以更快完成任务。

这不符合“所有任务必须无广告可完成、广告不是玩家任务”的长期原则。需要把“游戏内获得碎片”和“广告/任务/补偿发放”区分为产品来源，任务只统计允许的来源。

### 2.5 实施前周期风险

每日或每周键变化时，旧周期进度和领取状态会重置。若任务已经完成但玩家没有看到或没有点击领取，奖励不会自动补发。对一个提醒较弱的系统，这相当于因入口发现性不足惩罚玩家。

## 三、现行实施结果

| 范围 | 现行结果 |
|---|---|
| 任务预算 | 每日对局2、每日通关2、每周通关8、每周对局碎片4；满额40/周 |
| 广告解耦 | “本周通过对局获得20碎片”只统计关卡与复玩碎片，广告、任务奖励和补偿不推进任务 |
| 周期保护 | 跨日/跨周前自动补发已完成未领取奖励；重复结算幂等；设备时间回退不会重开旧周期 |
| 发现性 | 首页任务卡和局内菜单显示可领取数量；结算页在自然停顿点显示任务奖励入口 |
| 领取 | 支持单项领取和“全部领取”；领取后显示余额、下一皮肤和差额 |
| Admin | 展示28天休闲/规律/活跃假设模型、三类广告选择、碎片来源和40/55/81周预算对比；明确不是线上玩家统计 |
| 自动证据 | 核心任务/存档41条定向测试、玩家端39条、Admin 6条通过；完整发布门禁见验证记录 |

联合模型的关键结果：

- 规律玩家28天、不看广告：获得328碎片，完成前两套，距离极光盟誓还差152；
- 规律玩家28天、约一半首通广告：获得428碎片，距离极光盟誓还差52；
- 同配置55/周任务预算：距离极光只差8，目标过于接近耗尽；
- 同配置旧81/周预算：三套全部完成并剩余80，明显削弱后续任务和广告价值。

因此40/周作为当前首发预算进入代码；它不是永久不变的数值，未来只能依据真实回访、领取、兑换和广告后继续数据调整。

## 四、现行玩家体验

```text
玩家正常游戏，无需接任务
  -> 进度自动累计
  -> 对局结束后轻量显示“完成1项任务”
  -> 首页与局内菜单显示可领取数量
  -> 任务页按“可领取 / 进行中 / 已领取”排序
  -> 一键领取全部
  -> 显示获得碎片、当前余额、下一皮肤和剩余差额
  -> 周期刷新时自动保护已完成未领取奖励
```

### 3.1 提醒层级

- 对局进行中不弹任务提示。
- 结算页只显示一行完成摘要，不抢“下一关”或“重试”的主按钮。
- 首页任务卡和局内菜单使用数量角标，不在所有页面持续闪烁。
- 第一次完成任务时解释“进度自动累计，奖励在任务页领取”；后续不重复教学。
- 不使用“还剩几小时，立即领取”制造焦虑；系统负责保护完成奖励。

### 3.2 领取方式

- 保留手动领取，让玩家感知任务价值和碎片来源。
- 增加“领取全部”，不要求逐项点四次。
- 单项领取继续保留，方便理解具体来源。
- 领取成功后展示收藏目标推进，而不是只显示余额增加。
- 周期刷新前完成但未领取的任务自动发放；未完成任务正常清空，不跨周期累积。

### 3.3 任务内容结构

第一阶段继续保持少量、确定、自然完成的任务：

| 类型 | 目的 | 合适示例 | 禁止示例 |
|---|---|---|---|
| 参与 | 失败也有进度，降低挫败 | 游玩1局 | 连续登录7天，否则清零 |
| 成长 | 奖励理解和成功 | 通关1关、本周通关3关 | 连胜、限定某张未解锁地图 |
| 学习 | 引导真实策略 | 完成一次连吃、使用提示后通关 | 完全依赖随机局面的苛刻条件 |
| 收藏 | 让奖励有去向 | 查看装扮、装备已拥有主题 | 必须购买或看广告 |

首轮不急于增加任务数量。先修经济和体验，再决定是否用学习任务替换高度重叠的“游玩1局 / 通关1局”。

## 五、任务与商业化怎样联动

### 4.1 联动原则

任务、广告和IAP不是三种互相抢夺的按钮，而是服务不同玩家：

| 玩家 | 任务作用 | 广告作用 | 未来IAP作用 |
|---|---|---|---|
| 轻度玩家 | 正常游玩也有小幅进度 | 可以完全不看 | 无要求 |
| 回访玩家 | 每日/每周形成近期目标 | 自愿缩短收藏等待 | 可直接购买喜欢的高质量内容 |
| 收藏玩家 | 看见余额和下一目标 | 在接近目标时主动加速 | 购买明确主题、合集或去广告 |
| 策略玩家 | 未来通过学习/复盘任务获得价值 | 可换取高级复盘等即时服务 | 可购买高级服务权益，但不买胜负能力 |

### 4.2 第一阶段供给关系

第一阶段必须同时模拟以下来源：

```text
关卡首次通关
+ 重复通关
+ 每日/每周任务
+ 激励广告
+ 补偿或活动
- 皮肤和棋盘兑换
- 未来高级复盘等功能消耗
= 玩家余额与目标耗尽时间
```

任务应提供稳定但有限的基础补充，广告提供自愿加速，IAP未来提供直接透明的价值。不能刻意让任务奖励停在皮肤价格前一点，再用广告补齐；广告价值应来自玩家自己已经认可的收藏目标。

### 4.3 首轮经济候选

当前81/周不能直接视为最终合理。执行阶段先比较三组方案，不先拍脑袋改正式值：

| 方案 | 周任务理论上限 | 作用 |
|---|---:|---|
| 现状 | 81 | 作为风险基线 |
| 收敛 | 30–40 | 约为入门皮肤的38%–50%，保持回访价值且不取代关卡 |
| 中间 | 50–60 | 检查更强回访激励是否会提前耗尽收藏 |

比较至少覆盖休闲、规律、活跃三类**假设行为模型**，以及不看广告、部分观看、积极观看三类选择。它们是制作人调整参数时使用的情景模拟，不是当前线上玩家数据。最终值先由模型排除明显失衡，再由未来真实数据校准。

### 4.4 广告边界

- 不创建“观看1次广告”任务。
- 广告碎片不计入“通过游戏获得碎片”任务。
- 完成任务后不立即弹广告；先领取并理解奖励，再由结算或收藏目标提供独立的自愿入口。
- 广告失败、取消和无填充不影响任务进度、已完成状态和基础奖励。
- 玩家已经解锁所有当前收藏目标时，任务仍可正常完成，但应降低无用途碎片继续膨胀的风险；必要时隐藏广告入口，而不是制造新任务消耗。

### 4.5 与未来复盘、悔棋和IAP的关系

- 基础棋谱回放永久免费，不纳入任务付费。
- 高级复盘未来可消耗碎片或广告，但不设置“使用高级复盘”每日任务，避免制造功能消耗。
- 悔棋可有每日免费额度和额外广告机会，但不通过任务返还悔棋次数形成循环刷取。
- App IAP可直接解锁高质量外观、去广告或高级服务权益；日常任务继续面向所有玩家，不因付费提高战力或任务奖励倍率。
- 只有当内容更新和留存稳定后，才评估赛季任务或付费通行证；24关首发不需要战令体系。

## 六、容易被忽略的长期风险

### 5.1 任务变成第二份关卡列表

任务不能要求玩家重复完成已经厌倦的关卡。任务数量和复杂度必须低于核心关卡，玩家打开游戏的主要理由仍是策略挑战，而不是清单打勾。

### 5.2 货币通胀和收藏耗尽

任务发现性、广告观看率或未来活动任一提高，都会增加碎片供给。每次新增来源必须同时重跑目标耗尽模拟；不能等玩家大量积压货币后才临时提高皮肤价格。

### 5.3 时区、离线和改时间

当前周期按设备本地时间计算。首发本地存档可以接受，但必须测试跨日、跨周、离线恢复和设备时间变化，避免重复领取或误清奖励。未来账号云存档需要由服务端时间和幂等记录接管，不能直接合并多个设备的每日桶。

### 5.4 奖励来源不透明

通关、任务、广告、补偿和未来购买必须有来源标签。否则既无法解释玩家余额，也无法判断广告是否真的带来额外行为。

### 5.5 任务内容不适配玩家阶段

任务必须根据已解锁章节和已有能力筛选。新手不能收到高级连吃、冬季地图或尚未理解规则的任务；资深玩家也不能长期只看到“游玩1局”。

### 5.6 任务奖励反向伤害皮肤价值

奖励过少会让任务像杂务，过多会让收藏失去稀缺感。判断标准不是“领取率越高越好”，而是任务是否提高回访和目标理解，同时保持皮肤值得追逐。

### 5.7 数据隐私和平台差异

第一阶段本地事件足以做功能验证；真实分析接入必须遵守各渠道隐私和同意要求。独立站无广告，H5平台广告由平台能力决定，未来App再处理账号、商店和原生广告，不为了统一报表把平台隐私逻辑塞进核心规则。

### 第一批：联合经济基线

AI负责：

- 建立包含关卡、重复通关、任务、广告和三档皮肤的统一经济模拟。
- 模拟1、7、14、28天，以及休闲、规律、活跃玩家。
- 对比任务周上限81、30–40、50–60三组候选。
- 输出首次兑换、全部目标耗尽、余额积压和广告失去目标的时间。
- 明确当前任务奖励是否需要调整；没有数据前不直接写正式数值。

完成条件：能回答“任务增加后，玩家多久兑换每套皮肤、广告何时仍有价值、何时出现无用途余额”。

### 第二批：任务来源与周期合同

AI负责：

- 将任务进度使用的游戏碎片与广告、任务奖励、补偿来源分开。
- 保证所有任务完全不看广告也能完成。
- 设计并实现已完成未领取奖励的周期切换保护。
- 验证每日、每周、离线、刷新、改时间和旧存档迁移。
- 保持任务奖励领取幂等，不能重复发放。

完成条件：广告不会隐性推进任务；玩家不会因没看见入口丢失已经完成的奖励。

### 第三批：玩家发现与领取体验

AI负责：

- 任务完成反馈只在对局结束出现。
- 首页和局内菜单显示可领取数量。
- 任务页增加“领取全部”和明确的周期说明。
- 领取后显示碎片余额、下一皮肤目标和差额。
- 保持下一关/重试是结算主操作，并检查手机、平板、桌面。

完成条件：玩家无需预先理解任务，也能知道自己完成了什么、在哪里领取、奖励有什么用。

### 第四批：Admin经济模拟与制作人解释

AI负责：

- Admin经济页同时展示关卡、任务、广告和兑换来源。
- 提供三类假设玩家活跃模型、三类广告选择和多个时间范围的对比，并醒目标注“模型，不是真实玩家统计”。
- 用制作人语言解释供给、消耗、首次兑换和目标耗尽，不暴露函数名。
- 标出广告计入任务、未领取丢失、货币积压等风险是否已经关闭。

当前边界：玩家端没有把会话、回访、任务领取、广告和兑换事件上传到共享数据库或分析服务，Admin也没有生产玩家数据源。因此本批只能提供可复现经济模型，不能显示DAU、留存、真实活跃分层或真实广告转化。

完成条件：制作人不读代码也能判断模型中的任务奖励是否过高、广告是否仍有理由、皮肤目标是否可持续，同时不会把模拟结果误认为线上玩家行为。

真实玩家活跃度属于平台上线后的数据任务，需要先确定事件字典、隐私同意、匿名标识、数据保留、渠道数据源和Admin权限，再接入网站分析、H5平台报表或自有服务端。该工作不在`T-028`中假装完成，也不为首发模型提前引入数据库。

### 第五批：自动门禁与文档收口

AI负责：

- 增加进度来源、手动领取、领取全部、周期保护、幂等和旧存档测试。
- 增加联合经济里程碑，防止后续修改任务或皮肤时悄悄破坏广告与收藏路径。
- 覆盖广告成功、失败、取消、无填充时任务状态不变化。
- 运行两端生产构建与发布门禁。
- 完成后更新核心文档、任务状态和版本化验证记录，并将本专项迁入已完成。

完成条件：代码、测试、Admin、玩家文案和长期产品合同表达同一套规则。

### 第六批：用户体验确认

用户只需确认：

- 任务完成提醒是否明显但不烦；
- “领取全部”是否顺手；
- 奖励是否有价值但不过量；
- 任务是否像自然附加目标，而不是强迫每天上线的作业。

出现问题时由AI复现、归因和修复，不要求用户分析代码或经济公式。

## 八、验收标准

| 维度 | 放行标准 |
|---|---|
| 可理解 | 玩家知道无需接取、自动累计、在哪里领取、奖励有什么用 |
| 不打扰 | 对局中无弹窗；结算任务提示不抢下一关或重试 |
| 公平 | 所有任务不看广告可完成，不出售战力，不要求未解锁内容 |
| 奖励保护 | 已完成未领取奖励不会因跨日/跨周消失，且不能重复发放 |
| 经济 | 任务、广告、关卡和消费进入同一模拟；收藏目标不会被任务过早耗尽 |
| 商业 | 广告有独立、可理解的自愿价值，不通过任务隐性强迫 |
| 可持续 | 新任务和新奖励来源必须通过联合经济门禁，长期不靠涨价消化通胀 |
| 可管理 | Admin能解释来源、消耗、目标耗尽和风险，不要求制作人阅读代码 |

## 九、明确不做

- 本专项不接Poki/CrazyGames真实SDK。
- 不增加广告任务、连续登录、断签惩罚、战令或付费任务倍率。
- 不增加第三种任务货币。
- 不立即增加大量策略任务或随机任务池。
- 不用任务奖励补偿坏地图、过难AI或失败解释不足。
- 不在完成联合经济模拟前直接把81改成另一个未经验证的数字。
