# 需求分析与建议方案（待确认）

> 实施状态（2026-07-18）：交互与预览批次已完成。局内底栏收口为 Hint / Help / More；重新开始、声音、语言和退出已分层；胜利结算增加下一关名称、教学点和难度。皮肤卡可切换大号真实棋盘预览，棋盘缩略图改为迷你实战构图。霜狼/Night Watch 的最终高级美术仍需用户审美验收，未把“资源可加载”误标为美术通过。

## 一、深层意图

这里不是一个皮肤问题和几个菜单问题，而是“界面能否帮助玩家作决定”：皮肤页要回答为什么值得积累碎片；对局页要回答现在做什么；菜单要让高频动作容易、危险动作克制、低频设置可找到。

## 二、皮肤价值判断

霜狼和 Night Watch 已做过技术修订，但“能加载、可识别”不等于“有高级感”。当前兑换价格主要表达稀缺，实际差异仍偏向配色和局部轮廓。若默认皮肤更清晰完整，玩家不会因价格更高就认可兑换价值。

高级皮肤至少应满足：实战尺寸可识别、主题一眼可见、狼羊属于同一套装、相对默认有明确升级、长期使用不影响判断。价格不能替代质量。

### 可选方向

**A. 精修现有套装**：保留主题，重做轮廓、材质、高光和套装一致性。成本较低，但上限受现有造型约束。

**B. 建立标杆主题套装（推荐）**：每套覆盖狼、羊、棋盘适配和克制的反馈。霜狼可强化锐利冰晶轮廓与冷雾捕食反馈；夜巡可强化月光边缘、深色皮毛和星光反馈。差异至少覆盖轮廓、材质、反馈中的两类，不只是换色。

**C. 发布期以棋盘主题为主奖励**：若角色美术短期达不到标准，先让季节棋盘承担收藏价值，角色只保留默认和少量高质量套装。棋盘面积大、主题更易感知，但必须保证棋子对比度。

推荐 B 为目标、C 为上线兜底，不因已有碎片系统强推质量不足的角色皮肤。

## 三、皮肤页预览

当前棋盘卡片确实把完整背景压缩进很小的固定区域。Moonlit Field 因明暗反差强仍可辨认，其他季节背景会丢失纹理和层次。问题首先在预览方式，不一定是原资源本身。

推荐两层预览：

1. 列表卡使用专门缩略图、局部特征或代表性色板，不直接压缩整张背景。
2. 选中后显示较大的真实迷你棋盘，包含棋盘线、岩石、狼和羊。

角色卡也应显示真实游戏尺寸的狼羊组合。解锁状态和碎片差额要清楚，但不能盖过视觉主体。吸引力来自主题差异、实战效果可信和目标清楚，不是更大的价格数字。

## 四、对局信息

### 下一关

进行中不建议持续显示下一关信息，它与当前决策无关并分散注意。胜利结算时展示下一关名称、一句挑战钩子、新机制或难度变化，以及“下一场狩猎”主按钮。失败结算只强调重试、复盘和可选提示。

### 难度

难度适合放在关卡选择、关卡介绍和胜利后的下一关预告。进行中只常驻当前目标、进度、回合压力和特殊规则；玩家主动打开关卡信息时可查看难度。

难度不能只有“简单/困难”，应说明压力来源，如路线宽松、岩石封锁、羊 AI 更谨慎、回合更紧。

## 五、局内反馈

'Green = step · Red ring on sheep = gap-eat' 像开发图例，要求玩家同时理解颜色、英文动作和内部术语。

推荐按状态显示：

- 未选狼：“选择一只狼。”
- 已选狼：“点绿色空位移动；点红圈中的羊捕食。”
- 无捕食目标时不显示红圈说明。
- 连吃时：“还有可捕食目标。继续，或结束连吃。”

空位标记、羊目标环和图标也应有形状差异，颜色只作强化。

## 六、底部菜单与设置

当前五项等宽平铺把不同频率和风险的功能放在同一级：Hint 是核心辅助，Help 是支持，Reset 是破坏性动作，Mute 是设置，Exit 是导航。这会增加误触，也不突出主要任务。

推荐固定保留三项：

1. 'Hint'：卡住时的主要辅助。
2. 'Help'：本关简明规则和图例。
3. 'More'：重新开始、声音、语言、完整设置、退出。

'Reset' 改为“重新开始”，放入 More 并二次确认；'Exit' 写清“退出本局/返回关卡”。声音用图标显示状态，可放 More，宽屏也可作为独立图标。

语言是全局偏好，应在站点页头/设置页和局内 More 中均可找到，并立即作用于当前页。当前设置页主要维护静音、继续、帮助和隐私，语言主要依赖站点页头/页脚；沉浸对局缺少自然入口，应补齐。

## 七、执行批次

### 第一批：交互与信息收口

- 改写局内反馈和状态规则。
- 底部栏收口为 Hint、Help、More。
- 完成重新开始确认、退出语义、声音和语言入口。
- 调整胜利结算的下一关预告和难度表达。

### 第二批：预览系统

- 为棋盘建立专用缩略图/色板。
- 增加选中后的真实迷你棋盘预览。
- 统一套装在列表与实战中的预览尺度。

### 第三批：标杆皮肤

- 选择霜狼或 Night Watch 中一套作为首个生产级标杆。
- 按既有视觉标准重做，并在默认、季节、月夜棋盘验证。
- 标杆通过后再决定另一套精修、重制或暂时下线。

## 八、验收标准

- 实战尺寸下能立即区分默认与高级套装，并说出主题差异。
- 未解锁皮肤的预览足以支持兑换决定。
- 对局只保留当前决策信息，下一关在胜利后出现。
- Hint/Help 易找；重新开始不误触；语言、声音、退出可在 More 找到。
- 操作提示无需内部术语，也不只依赖红绿。

## 九、推荐决策

建议确认“先收口交互，再升级预览，最后做一套标杆高级皮肤”。推荐主题套装 B，棋盘奖励 C 作为上线兜底；不要在预览不清和高级感不足时继续提高兑换价格或增加同质量皮肤。

## 十、以点带面的补充诊断

皮肤缩略图和五项菜单暴露的是同一种产品问题：实现按资源或函数组织，玩家却按目标、频率、风险和价值理解产品。还应一并检查：

1. **技术通过不等于产品通过**：资源检查只能证明文件存在、路径正确，不能证明高级感、兑换意愿和实战可读性。
2. **奖励成本与价值倒挂**：碎片数量定义了解锁门槛，却没有证明奖励值得等待；若预览弱，进度越长挫败越强。
3. **状态层级不统一**：已装备、已拥有、可兑换、还差碎片、当前预览需要明确主次，不能让每种状态争夺注意。
4. **设置入口碎片化**：语言在站点导航、静音在局内、其他设置在设置页，缺少一致的“全局偏好”模型。
5. **危险动作缺少保护**：重新开始和退出会丢失当前进度，应与普通帮助按钮在视觉和确认流程上区分。
6. **结果页职责过多的风险**：下一关、重玩、分享、奖励和广告不能全部成为主按钮，应按“继续游戏优先，商业化可见但不劫持”排序。
7. **缺少真实视口验收**：资源原图、皮肤卡片、实际棋盘是三种尺度，必须分别验收，不能用看原 SVG 代替实战判断。

这批改造应顺带建立“页面任务清单”：每个页面只有一个主要用户目标、一个主操作，其他功能按辅助、设置、危险操作分层。
