# Fangrush 需求池

> 本文维护已经提出但尚未启动的功能。需求按实际启动时机排序；迁入当前任务清单后，不在这里并行维护进度。

## 阶段 A：H5 平台接入

启动条件：生产体验和提交素材稳定，用户取得平台账号/权限并决定开始技术接入。

### Poki / CrazyGames SDK 与联调

- 范围：真实 SDK、加载与生命周期事件、暂停/恢复、平台静音、广告回调、Inspector/Preview/Portal 联调、平台构建上传、审核修订和发布。
- 价值：满足平台技术要求，取得真实分发和广告收入能力。
- 风险：两平台接口和审核节奏不同；密钥、协议、收款和提交权限必须由用户管理。
- 完成结果：两个平台分别有可审计适配器、平台构建、回归记录和审核状态。

### H5 广告与商业漏斗优化

- 范围：插屏/激励机会点、奖励价值、频控、广告完成率、广告后继续率和收入实验。
- 启动条件：至少一个平台上线并获得足够真实流量。
- 边界：广告不阻挡下一关，不在思考或连吃中打断，不以收入增长掩盖首局完成和留存下降。

## 阶段 B：Google Play / App Store

启动条件：H5或独立站证明游戏具有可接受的完成率、时长、重试、回访和商业潜力，用户决定启动移动商店版本。

### 移动应用与商店发布

- 范围：Capacitor移动壳、原生生命周期、离线与升级、签名、商店构建、崩溃监控和版本回滚。
- 商店事项：隐私披露、数据安全、年龄分级、审核说明、商店素材和基础运营。
- 商业能力：AdMob等原生广告；去广告或付费皮肤 IAP；恢复购买、交易幂等和审核沙盒。
- 边界：继续复用同一 `game-core`，不为 App 复制规则、AI或24关配置。

### 账号与云存档（非上架硬前置）

- 价值：跨设备同步、恢复进度和未来统一运营。
- 成本：后端、认证、隐私、安全、迁移和长期运维。
- 启动条件：出现明确跨设备需求，或多个渠道确实需要统一身份与进度。
- 详细方案：[P-ACCOUNT](独立站专属需求/P-ACCOUNT.md)。

## 阶段 C：真实玩家证据驱动的增强

### 专业美术、角色动画与原创音频

- 价值：提升第一印象、角色辨识、商店点击和品牌长期价值。
- 范围：品牌级角色重制、完整状态帧、环境音、主题BGM和高级演出。
- 启动条件：玩家或平台反馈证明当前资源限制转化、理解或留存。

### 高级无障碍

- 范围：完整读屏、全键盘棋盘操作、高对比主题、细粒度音量和系统化可访问性测试。
- 启动条件：真实用户需要、目标平台要求，或团队决定提高覆盖人群。

### 奖励与长期留存深化

- 范围：皮肤主题故事、奖励曲线实验、重复游玩收益、收藏展示和长期目标。
- 启动条件：数据证明奖励理解、回访动力或收藏价值是主要问题。

### 分享增强与传播实验

- 当前基础：结果卡、局面图和系统分享已经具备。
- 后续范围：完整棋谱链接、挑战好友、分享模板实验和平台传播活动。
- 启动条件：基础分享产生可观察使用量或传播价值。

### 悔棋

- 价值：降低误操作和长局单步失误造成的挫败。
- 风险：必须完整恢复羊AI随机状态、连吃起点、重复计数和奖励边界，也可能削弱决策重量。
- 启动条件：玩家反馈或数据证明误操作是主要失败与流失原因。

### 高级提示与路线演示

- 价值：帮助多次失败但愿意继续的玩家跨过策略理解门槛。
- 风险：不能把模拟策略包装成必胜答案，也不能用提示掩盖坏地图。
- 启动条件：当前三级指导仍无法解决可复现的策略理解问题。
