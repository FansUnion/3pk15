# 第一阶段商业化与碎片激励执行计划

> 状态：自动化与代码侧已完成，等待真实设备、菜单体感和三套皮肤审美确认；尚未接入真实广告 SDK。本文只维护当前阶段的验收证据和剩余用户判断；长期原则以[奖励、经济与商业化设计](../产品核心/04-奖励经济与商业化设计.md)为准。

## 一、当前结论

- 独立站继续无广告，用于品牌、演示、SEO和平台申请。
- 第一阶段 H5 的主要变现候选是胜利结算后的自愿激励广告。
- 当前代码已经具备广告能力开关、Mock、成功/取消/失败/无填充/冷却、暂停恢复、一次结算一次奖励和即时通用碎片发放。
- 当前没有 Poki/CrazyGames 真实 SDK，因此现阶段广告收入为零；真实接入继续由需求池管理。
- 通用碎片已经形成霜狼 80、夜巡狼群 160、极光盟誓 240 三档目标；免费主线提供 240，首次通关广告全看提供 480。
- 新存档默认跟随春夏秋冬关卡使用对应棋盘；装扮页可切换为固定棋盘，旧存档迁移保留主动选择。
- Admin试玩返回关卡台已使用 Admin 专用链接，不再生成`/zh/admin/levels`。
- 局内底部只保留与当前局面相关的提示操作；低频帮助和成长入口集中在顶部菜单，避免长期占据棋盘操作区。
- 长期悔棋、100关奖励扩展、IAP商品、低频插屏和更多皮肤先只维护在长期商业化合同；本期不因为这些长期能力直接扩大代码范围。

## 二、本期目标

在不接真实 SDK、不改变独立站无广告原则的前提下，把经济和界面准备到以下状态：

1. 玩家不看广告仍能完成 24 关和四季基础收藏。
2. 有收藏兴趣的玩家在第一阶段 H5 中愿意主动观看一部分广告。
3. 广告按钮说明即时奖励、目标和差额，而不是只说“看广告”。
4. 没有目标、没有广告能力或没有填充时，不展示无意义入口。
5. 接入真实 SDK 后，只替换平台适配器，不重新设计奖励、存档和结算流程。
6. 春夏秋冬默认呈现对应季节棋盘，玩家仍可自主选择高级或固定棋盘。
7. Admin试玩返回、局内菜单和成长入口在商业化前形成稳定导航合同。

## 三、两轮诊断结论

### 第一轮：代码与经济事实

- 诊断时24关首次通关提供240通用碎片、两个角色主题总成本仅130；现已调整为三档累计480。
- 一次首次通关广告增加10，相当于把本关通用碎片收益从10提高到20；奖励比例并不低，问题在于目标太少、旧价格太低。
- 广告可在重复通关继续获得至少3碎片，但没有剩余商品时，这只是无用途余额。
- 四季标准棋盘和高级棋盘原本只使用全局`boardId`；现已增加棋盘模式和每季选择。
- Admin返回404已由专用链接边界测试覆盖，不再依赖玩家端本地化导航。

### 第二轮：玩家体验与商业结果

- 第一阶段不应把广告奖励简单提高到20或降低免费奖励；这会加速经济耗尽，不能形成长期收入。
- 需要同时建立价格梯度、第三个旗舰目标、目标差额表达和季节棋盘自动呈现，广告才有可见价值。
- 局内进行时应尽量专注棋盘；皮肤、任务和商业目标应在首页、结算和菜单中可发现，但不抢占移动与连吃操作。
- 当前“更多”菜单内部结构基本合理，问题是底栏三个同权文字入口不符合实际使用频率。帮助并非每局高频操作，成长入口又隐藏较深。
- 第一阶段导航应从“功能都塞进更多”升级为“棋盘专注 + 上下文提示 + 清楚菜单 + 结算成长目标”。

## 四、执行批次

### 第一批：经济目标审计与候选方案

AI负责：

- 生成 24 关无广告、部分看广告、每次可用都观看三条经济路径；列出每关后的通用碎片、季节碎片、可兑换目标和剩余差额。
- 统计当前两个角色主题在三条路径下的解锁关卡，验证广告是否导致过早清空全部目标。
- 用`80 / 160 / 240`作为首轮定价候选：无广告24关完成前两套，部分观看推进旗舰目标，首次通关广告全看可完成三套。
- 确认霜狼、夜巡狼群分别是否达到80和160档的视觉价值；设计或制作一套达到240档要求的旗舰狼羊主题候选。
- 为新增候选建立视觉质量、棋盘辨识和实装价值要求；资源未达标前不进入经济目录。

完成条件：能明确回答“玩家第几关第一次想看广告、广告加速多少、什么时候失去观看理由”。

### 第二批：奖励报价与目标追踪

AI负责：

- 将结算激励表达调整为“即时奖励 + 当前目标 + 观看后差额”。
- 目标按最近可兑换的通用碎片外观选择；全部解锁后隐藏广告入口。
- 保持下一关为主要按钮，广告为清楚但次要的自愿操作。
- 在皮肤页显示来源、余额、成本和差额；避免玩家只积累数字却不知道用途。
- 保证广告成功发奖幂等，取消、失败、无填充和冷却不影响基础奖励、声音、输入和继续游戏。

完成条件：玩家在点击前不需要进入详情页，也能知道为什么值得看。

### 第三批：季节棋盘与装扮合同

AI负责：

- 将默认棋盘模式改为“跟随关卡季节”，并为旧存档提供明确迁移。
- 在装扮页提供“跟随季节 / 固定棋盘”选择；长期数据结构支持每季标准或高级棋盘选择。
- 确保春夏秋冬关卡默认产生明确视觉变化，高级棋盘兑换后能在对应季节自然使用。
- 更新首页、结算和装扮页的说明，让玩家知道棋盘为何变化、在哪里预览和切换。
- 回归棋子、岩石、危险提示和合法行动在所有棋盘上的辨识度。

完成条件：从未进入装扮页的新玩家也能在四季感受到不同棋盘；主动选择固定棋盘的玩家不会被系统覆盖。

### 第四批：Admin返回与局内菜单收口

AI负责：

- 修复Admin试玩中所有共享玩家链接的本地化前缀问题；Admin路由只生成`/admin/*`，不使用`/zh/admin/*`兼容路径。
- 增加回归测试，覆盖顶部返回、结果详情返回、更多菜单退出、上一关和下一关。
- 将局内常驻底栏从三个同权文字入口重新评估为“上下文提示 + 菜单”的轻量结构；帮助进入菜单，连续失败时再主动提示。
- 保留更多菜单的分组，但提高装扮、任务和可领取状态的可发现性；商业入口只在结算和成长场景表达，不在棋盘决策中制造干扰。
- 在手机、短屏、平板和桌面检查菜单、连续捕食操作条、提示和棋盘高度不会互相挤压。

推荐第一阶段结构：

```text
顶部：返回 / 关卡名 / 菜单
棋盘附近：仅在需要时出现“提示”
菜单：本局、进度与装扮、帮助与设置、报告、退出
结算：奖励、下一关、当前收藏目标、可选广告
```

完成条件：高频操作更近，低频操作仍容易找到，皮肤与任务不依赖玩家猜测“更多”里有什么。

### 第五批：Mock 商业闭环与自动化

AI负责：

- 覆盖无广告、成功、取消、失败、无填充、冷却和重复回调。
- 覆盖首次通关、重复掉落、任务奖励、广告奖励和皮肤兑换的组合。
- 增加经济里程碑门禁：无广告路径始终可完成；广告路径只加速外观，不修改关卡、AI和胜负。
- 在 Admin 经济页提供三条路径对比、目标耗尽点和广告奖励模拟；页面使用制作人可理解的语言。
- 定义未来平台事件：广告机会、开始、完成、失败原因、奖励到账、下一关、皮肤查看、兑换和装备。

完成条件：没有真实 SDK 也能完整演练第一阶段经济和异常流程。

### 第六批：真实设备与制作人确认

用户负责：

- 判断现有和新增候选皮肤是否真的值得收藏；不好看的资源不能靠提高价格变得有价值。
- 在手机、平板和桌面确认结算页中“下一关”和广告报价的层级是否自然。
- 判断广告奖励是否容易理解：知道获得多少、推进什么目标、观看后还差多少。

AI负责收到反馈后的复现、调整和回归，不要求用户检查代码。

### 第七批：平台接受后接真实 SDK

该批当前不执行，仍保留在[需求池](需求池.md)：

- Poki/CrazyGames 平台适配器；
- 加载与游戏生命周期；
- 广告回调、填充、冷却和平台静音；
- Inspector/Preview/Portal 联调；
- 平台真实事件、收入和留存分析。

完成条件：平台真实广告失败不阻断游戏，实际收入、观看率和广告后继续率可联合判断。

## 五、明确不做

- 不在独立站启用广告。
- 不在选择、移动、捕食、连吃或羊 AI 行动中播放广告。
- 不降低基础奖励、不提高旧皮肤价格来强迫观看。
- 不把广告写成每日/每周任务。
- 不出售削弱羊 AI、复活、额外狼、必胜提示或主线通行权。
- 不在没有真实流量前承诺收入金额或广告单价。
- 不同时接多套 App 广告网络或提前开发 App IAP。

## 七、任务停止线

前三批完成后，若仍没有足够质量的收藏目标，不接真实广告 SDK；因为即使 SDK 正常，玩家也没有观看理由。若收藏目标和 Mock 闭环成立，则冻结经济合同，等待平台接受后只做适配、联调和真实数据优化。

第一阶段商业化成功不是“每个人都看广告”，而是：愿意收藏的玩家能理解并主动选择，普通玩家不被阻断，广告后继续游戏的人数和收入都能被真实记录。

长期能力的衔接边界见[奖励、经济与商业化设计](../产品核心/04-奖励经济与商业化设计.md)第2节：每日悔棋、100关奖励弧段、自然断点插屏、单个主题/季节包/当前合集/去广告IAP和完整主题皮肤均先做产品合同，不在本期假装已经上线。
