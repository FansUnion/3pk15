# 24关 AI 难度曲线全面复核专项

> 状态：开发侧前五批已完成，等待制作人最多 8 局定向体验。本文前半部分保留升级前 V2 诊断，最终实施结果见第十节；长期稳定原则仍由[羊 AI 机制与制作人验收指南](../产品核心/09-羊AI机制与制作人验收指南.md)与[24关学习曲线与重建设计](../产品核心/24关/24关学习曲线与重建设计.md)维护。

## 一、最终判断

当前 24 关 AI 曲线是**方向科学、映射已接入、质量证明不足**，不能解释成“冬季已经达到玩家理解中的专家水平”。

- 五种生产画像确实按 `guided → foundation → tactical → strategic → expert` 接入真实关卡，线上报告中的 `winter-06` 也确实使用 `expert`，不是错档或默认值覆盖。
- “所有画像不选严格劣着、夏季开始看完整狼回合、冬季强调多回合合围”的设计方向正确。
- 当前 Expert 只是两层、1800 节点的受限启发式搜索，不是全局求解，也没有达到文档要求的全部智能维度。
- 当前固定种子门禁主要能发现直接送子、单回合长连吃、长尾和首次接触风险；它没有检查同一只狼跨多个回合持续捕食、临近第 8 只时的紧迫防守、目标持续性和真实玩家套路。
- 因此旧结果 `15 pass / 9 review` 只能表示“15 关未命中旧自动红线”，不能表示 15 关已经完成产品质量验收。九个 `review` 也不是九张坏地图，而是存在需要棋谱或真人判断的风险。

设定可行性结论：**条件可行**。规则、24关结构和五画像可以继续使用，但在本专项完成前，冬季“专家防守”和完整 24 关学习曲线只能视为待校准目标。

## 二、用户问题包的精确复盘

证据文件：`fangrush-report-winter-06-2026-07-20T01-50-24.json`。

| 项目 | 真实结果 |
|---|---|
| 关卡与画像 | `winter-06`、`expert`、`sheep-ai-v2` |
| 固定种子 | 初始 `82196`，42 个羊回合后为 `82238` |
| 终局 | 狼胜，`targetEaten`，85 plies，捕食 8 只 |
| 捕食归属 | `wolf-2` 捕食 6 只，`wolf-3` 捕食 2 只，`wolf-1` 捕食 0 只 |
| 捕食 plies | 23、33、49、57、67、75、79、85 |
| 当前代码复现 | 42/42 个羊决策完全复现；所有 Expert 决策完成目标两层搜索，无降级 |
| 严格劣着 | 0 次 |
| 有即时零暴露替代却选择暴露 | 1 次，报告动作 54、局面 ply 53，`sheep-11 → 4,2` |

### “一只狼能不能吃羊”的准确回答

规则允许任何一只狼独立移动和捕食，不要求三只狼共同参与一次捕食；三狼的价值是控制路线、限制羊移动并为实际捕食者制造落点。因此“一只狼连续承担多次捕食”本身不是规则错误。

这局也不是一只狼完成全部捕食，而是两只狼分别完成 6 次和 2 次。不过玩家质疑仍然有效：75% 的捕食集中在同一只狼，最后四次均由 `wolf-2` 完成，说明现有 Expert 没有充分惩罚跨回合的固定猎手路线。第三只狼即使可能提供了站位压力，终章仍没有呈现文档所承诺的“必须通过三狼综合协作才能突破”的清晰体验。

### 第 54 步怎样归因

当前分析器把该步标为 `sacrifice` 而不是 `blunder`：它认为羊虽然暴露一只，但换得的机动、包围或后续局面总分更高。该决定完整完成了两层搜索，因此不是预算降级造成。

这不能直接证明牺牲一定错误；要证明错误，需要比较更深反事实分支。但它证明当前验收存在盲区：

1. “非支配动作”不等于对终局目标正确。
2. 当前模型只显式计算下一整个狼回合最多吃几只，未衡量同一猎手在后续多个回合的持续收割能力。
3. 当前评分对“已经被吃 4–7 只”的终局紧迫度没有专门加权，同样的牺牲在开局和第 7 只前可能被近似评价。
4. 旧门禁要求存在一个同时不增加连吃、狼机动和总分损失的替代动作才记为“可避免暴露”，因此这一步没有触发旧红线。

结论：本局不能仅凭结果宣布 AI 犯了理论错误，但足以确认**Expert 的产品行为和验收覆盖没有达到终章要求**。

## 三、学习曲线定义与真实实现对账

| 设计要求 | 当前实现 | 判断 |
|---|---|---|
| 24关共享“不选严格劣着”底线 | `analyzeSheepActions` 过滤被支配动作 | 已实现，但底线定义偏窄 |
| 春季宽容、后期逐步收紧 | 五画像候选宽度、随机带和搜索预算不同 | 已实现粗粒度曲线 |
| 每两关“引入—练习” | 文档和地图职责存在；同季通常共用同一画像 | 仅内容层实现，AI行为没有逐对校准参数 |
| 夏季看完整狼回合 | tactical 搜索深度 1，可展开完整狼回合 | 基本实现 |
| 秋季长链、地形、牺牲 | 评分包含单回合捕食链、机动、包围；strategic 两层搜索 | 部分实现，地形和牺牲是启发式近似 |
| 冬季多回合目标、主动合围、优势转化 | expert 两层搜索、候选更宽、无随机并列容差 | 部分实现；无目标记忆、终局紧迫度和跨回合猎手指标 |
| 冬季 05/06 为最强综合考试 | 均映射 expert，地图不同 | 配置已接入，玩家体验未证明；winter-06 已有反例风险 |
| 难度逐关可解释 | `difficulty` 很快封顶，同季画像相同 | 未充分实现，后台标签不能代表细粒度曲线 |

### 科学性结论

原设计采用“共同智力底线 + 逐步扩大规划和主动性”，比“春 Easy、夏 Normal、冬 Hard”更科学，应保留。问题不在五画像概念，而在将 12 个学习阶段压缩成五套固定参数后，没有再给每两关建立可测行为目标。

推荐最终结构不是 24 套 AI，也不是单纯增加搜索深度，而是：

```text
共享规则与安全底线
  + 五种稳定生产画像
  + 每两关少量透明调优参数
  + 关卡专属反例棋谱与验收指标
```

每关调优只允许使用候选宽度、搜索预算、终局紧迫度、猎手压制、目标持续性等明确参数；不能暗中作弊、读取玩家未来操作或为了胜率让前期故意送子。

## 四、现有自动证据为什么没有发现

### 1. 快速门禁覆盖太浅

`ai-profile-quality.test.ts` 每关只在固定的第一个羊决策检查合法、确定、非支配和至少一层搜索。它证明配置和基础算法能工作，不检查中后盘智能。

### 2. 三类狼代理缺少资深玩家套路

现有 `random`、`mixed`、`chain-aware` 能发现基础胜路和单回合连吃，但没有明确执行：

- 固定一只狼作为猎手，另外两只只控制路线；
- 连续数回合复用相同捕食通道；
- 在已吃 5–7 只时优先追求最快终结；
- 诱导 AI 做“评分好看但终局失守”的牺牲。

因此自动代理很少赢，不代表资深玩家不能稳定利用同一套路。

更根本的问题是，当前 `mixed` 和 `chain-aware` 狼代理也调用羊 AI 使用的 `evaluateScore` 选择局面。双方共享同一启发式时，会共同忽略跨回合固定猎手等风险，形成“同一模型验证自己”的闭环。新增玩家代理必须有独立目标、状态记忆和策略实现，不能只调整随机比例。

### 3. 指标只看单回合链，不看跨回合收割

当前记录 `maxCaptureChain`，但本问题包的最长单回合链并不异常，真正问题是 23 到 85 ply 间同一只狼反复得分。需要新增：

- 捕食集中度：最多捕食狼占全部捕食比例；
- 连续猎手次数：相邻若干次捕食是否由同一只狼完成；
- 捕食间隔与末段速度：第 5 到第 8 只用了多少 plies；
- 跨两到三轮猎手风险：羊走后，同一狼是否仍保有可重复建立的捕食路线；
- 三狼参与度：非捕食狼是否实际压缩了合法行动，而非闲置。

### 4. `pass` 名称容易被误读

本次完整矩阵重跑仍为 `15 pass / 9 review / 0 reject`，与 2026-07-19 记录一致。这里的 `pass` 是候选风险门禁结果，不是生产验收结果。后续界面和文档统一改名：

- `auto-clear`：未命中自动红线；
- `needs-review`：命中长尾或首次接触风险；
- `auto-reject`：命中明确结构或动作红线；
- `player-approved`：只有制作人/玩家完成体验确认后才能使用。

## 五、需要补齐的 AI 智能维度

| 维度 | 当前覆盖 | 本期目标 |
|---|---|---|
| 合法与一致 | 完整 | 保持 |
| 即时安全 | 有非支配护栏 | 收紧“安全替代”的定义，避免总分掩盖明显送吃 |
| 单回合连吃 | 已计算完整狼回合 | 保持并增加责任动作回溯 |
| 跨回合猎手压制 | 缺失 | 识别同一狼的持续接触、落点和再次捕食路线 |
| 终局紧迫度 | 缺失 | 已吃 5–7 只时显著提高止损和断线优先级 |
| 主动合围 | 只有局面包围分 | 增加对同一目标狼的计划持续性和实际机动压缩 |
| 合理牺牲 | 只看综合启发式 | 要求牺牲换来可测封锁、节奏或终局收益 |
| 地形理解 | 间接反映在合法行动 | 增加通道、关键落点和岩石间距风险指标 |
| 反循环与优势转化 | 只有重复惩罚 | 优势局必须持续减少目标狼机动或完成封锁 |
| 可解释性 | Admin 有评分分解 | 转成玩家语言：阻断谁、保护谁、为何牺牲、搜索是否完成 |

## 六、执行计划

本期采用六批执行。前五批由开发侧完成；只有全部自动门禁通过后，才进入不超过8局的制作人定向体验。

### 第一批：测试语义、问题包和反例库

1. 将本次 `winter-06` 问题包复制为版本化回归样本，不依赖下载目录。
2. 建立问题包复盘器，自动输出捕食归属、责任羊步、安全替代、搜索深度、终局阶段和复现结果。
3. 建立反例数据格式，覆盖局面、期望行为、禁止行为、证据来源、算法版本和配置指纹。
4. 把现有快速 AI 测试明确命名和注释为“首决策配置烟雾门禁”；旧 `pass/review/reject` 只表示自动风险，不再表示产品验收。
5. 为关键测试补齐“证明什么、不能证明什么、代理含义、可复现输入”的代码注释。

完成条件：当前问题可由一条命令复现并输出责任局面；所有入口不再让维护者误读自动 `pass`。

### 第二批：四级玩家画像代理

新增四种彼此独立的狼方画像，不以羊 AI 的 `evaluateScore` 作为共同决策核心：

| 画像 | 本期实现重点 | 用途 |
|---|---|---|
| 新手 | 直接捕食优先，其余接近随机 | 检查地图是否无需理解也能赢 |
| 普通 | 短连吃、退路、基础双狼协作 | 模拟完成春季后的玩家 |
| 熟练 | 夹击、换线、岩石、落点控制 | 检查夏秋学习是否有收益 |
| 资深 | 固定猎手、三狼分工、诱导、终局竞速 | 寻找冬季可重复利用套路 |

保留现有 `random/mixed/chain-aware` 作为历史诊断基线，但不再称为完整玩家画像。先用策略单元测试证明每个画像在特定局面确实执行目标策略，再运行完整关卡。

完成条件：同一测试局面中四画像表现出可解释差异；资深画像能复现或主动搜索本次固定猎手路线。

### 第三批：离线教师 AI 与动作裁判

1. 建立高于生产预算的离线教师搜索，支持更宽候选、迭代加深、换位缓存和明确 `unknown`。
2. 对每个羊候选输出短期后悔值、未来两到四轮捕食风险、困狼进度和终局风险。
3. 教师 AI 不直接接入玩家实时对局，只负责反例生成、动作比较和回归裁判。
4. 对“牺牲”要求输出交换收益；无法证明换得封锁、节奏或终局收益时标为可疑。
5. 用不同实现或目标函数打破生产 AI 与模拟代理共享评分的验证闭环。

完成条件：本次第54步能得到“合理牺牲、明显错误或预算内未知”的可解释结论，而不是只得到一个综合总分。

### 第四批：生产 AI 防守质量升级

1. 在评分中加入随 `eatenSheep` 增长的终局紧迫度。
2. 增加同一猎手跨回合威胁和关键落点控制，避免只看下一回合最长链。
3. 为牺牲增加交换条件：只有明显改善困狼进度、阻断终局或形成可验证节奏时才允许。
4. 增加目标持续性，使战略/专家画像能连续围绕同一只弱狼推进。
5. 保持确定节点预算；先优化动作排序、缓存和启发式，再评估是否需要提高深度，避免移动端卡顿。
6. 每次调整必须同时跑反例库和四画像矩阵，避免修复冬季却破坏春季教学宽容。

完成条件：冬季不再稳定出现“送一次、同一狼反复吃、优势不收口”；生产设备仍稳定完成目标搜索。

### 第五批：24关曲线、全量对照与 Admin

为 12 个两关阶段建立行为合同，不增加 12 套算法：

| 阶段 | 必须表现 | 主要验证 |
|---|---|---|
| 春01–02 | 合法、自保、保留教学空间 | 新手首吃和无严格劣着 |
| 春03–04 | 避开直接送吃、堵一次落点 | 短链责任动作 |
| 春05–06 | 限制单狼、开始识别双狼 | 单猎手不能无代价重复得分 |
| 夏01–02 | 看完整狼回合、管理通道 | 完整链和退路 |
| 夏03–04 | 识别孤狼、持续局部封锁 | 两轮目标持续性 |
| 夏05–06 | 战术综合与合理牺牲 | 牺牲收益可解释 |
| 秋01–02 | 关键点和区域防守 | 岩石/通道控制 |
| 秋03–04 | 跨区和长链终点 | 两轮捕食风险 |
| 秋05–06 | 优势转化、减少拖延 | 合围进度与反循环 |
| 冬01–02 | 高压站位、阻断固定套路 | serial-hunter 对抗 |
| 冬03–04 | 多回合目标、主动困狼 | 资深代理和真实棋谱 |
| 冬05–06 | 最强公平综合考试 | 无重复收割漏洞且仍有可理解狼胜路 |

同时执行：

1. 固定当前版本作为基线，对升级前后使用相同关卡、种子和玩家画像。
2. 现有720局历史矩阵继续保留；四画像生成独立新矩阵，不混算历史数字。
3. 增加捕食集中度、连续猎手、末段捕食速度、三狼参与度、教师后悔值和计划持续性。
4. 先修 AI，再判断地图；合格 AI 下仍无可理解狼胜路时才调整开局或岩石。
5. Admin 关卡页显示两关阶段、必须表现、自动状态和玩家验收状态。
6. Admin AI 页导入问题包后直接回答“谁吃了几只、哪步有替代、是否重复猎手、教师结论是什么”。
7. 更新逐关设计卡的 AI 行为目标、风险、验证日期和证据链接。

完成条件：四画像随季节呈可解释的相对曲线；每关异常都能归因到 AI、地图、代理或未确认体验，而不是只剩胜率数字。

### 第六批：最多8局制作人定向验收

自动化从每季选择两张信息量最高的代表关，总量不超过8局。建议初始候选为每季一个转折关和一个季末考试关，最终按自动风险选择，不固定要求重复玩未变化关卡。

- 完整对局只判断：太笨、合理、有压力但可赢、只会拖延、过难。
- 单步争议提供局面对比，不要求重打一整关。
- 未改动且证据未变化的关卡不重复验收。
- 8局通过后结束首发前假设驱动调优，其余校准等待真实玩家数据。

完成条件：制作人只确认乐趣、压力和可理解性，不再承担发现确定性 AI 缺陷的责任。

## 七、放行标准

以下条件全部满足，才能称“24关 AI 学习曲线按预期实现”：

- 所有正式画像不选择严格被支配动作。
- 夏季起无可避免责任的完整狼回合送链；冬季无可稳定复现的跨回合单猎手漏洞。
- 已吃 5–7 只时，Expert 对安全替代和终局阻断表现出明确更高优先级。
- strategic/expert 在优势局持续压缩同一目标狼，而不是只保羊或往返。
- 每两关行为合同有固定种子证据，地图、开局和 AI 不在同一关同时无解释跳升。
- `winter-05/06` 至少各保留一条可理解狼胜棋谱和一条真实羊防守棋谱；胜局不能主要依赖明显送子。
- 移动端 Expert 在节点预算内稳定完成目标深度，无频繁降级和页面卡顿。
- 自动状态与玩家验收状态分离；自动通过不得自动写成产品验收通过。

捕食集中度不应简单设成“一只狼最多吃 N 只”的规则。它是风险信号：若高集中同时伴随可避免安全替代、重复相同通道或非捕食狼无实际压制，应判 AI/地图待审；若另外两狼持续限制路线且羊无更优防守，则可视为三狼协作的合理兑现。
