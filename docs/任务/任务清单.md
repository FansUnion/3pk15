## 当前目标

近期聚焦三条主线：

1. **第二批体验专项**：先解决 Admin 关卡信息难懂、跨季节通关承接、结算信息层级和全流程页面体验问题；胜利条件先评估，不直接改规则。
2. **生产环境用户体验验收**：AI 完成可自动检查和可复现问题修复，用户验证真实设备、皮肤审美和24关体验。
3. **平台基础材料准备**：准备 Poki 准入表单和 CrazyGames Basic 首次提交所需的文案、截图、封面、视频及主体信息清单；不接真实 SDK，不做平台联调。

## 当前明确待办

### AI 可直接完成

- 完成 T-025 的 Admin 关卡信息和指标口径改造。
- 完成 T-026 的规则证据审计、跨季节承接修复、结算精简和全流程自动页面检查。
- 按[平台基础材料清单](../商业成功/上架事项/基础材料清单.md)补齐截图场景与文件命名、封面规格、预览视频脚本、候选素材和权利缺口。

### 必须由用户完成

- 使用真实手机、平板和桌面验证触控、阅读、音频和整体感受。
- 复核改造后的 Admin 关卡信息是否足够通俗，确认“吃 8 羊”的主观胜利感受。
- 逐步试玩24关；当前正式人工验收仍为 `0/24`。
- 提供表单所需的真实姓名/邮箱/国家/主体信息，确认最终封面、视频、英文文案和素材权利。
- 登录平台、接受协议并点击最终 Submit。

## 验收口径

- 自动化通过不能代替真实设备体验和24关人工结论。
- 人工反馈不要求一次完成；每次提供页面、设备、操作、预期和实际表现即可，由 AI 负责复现、归因和回归。
- 平台“基础材料完成”不等于已接 SDK、已通过 Inspector、已提交审核或已上架。
