当前不以平台上架为近期硬目标，采用并行推进：一边准备独立站和H5部署，一边由用户抽空进行项目质量验证，同时完成最终代码结构和多平台架构准备。平台申请、真实SDK和App继续保留启动条件，避免提前开发造成返工。

## 剩余工作与责任边界

### 我可以直接完成

1. 实现Poki与CrazyGames真实SDK适配器，并保持Standalone安全降级。
2. 接入平台加载、首次输入、Gameplay Start/Stop、暂停恢复、关卡进度和平台静音事件。
3. 接入平台插屏/激励广告回调，复用现有输入与音频暂停、失败恢复和奖励幂等边界。
4. 完成平台分享链接、平台构建脚本、Admin/外链剔除、请求白名单和包体统计。
5. 建立Poki/CrazyGames本地SDK Mock与自动测试，覆盖初始化失败、无填充、取消、AdBlock、冷却和重复回调。
6. 生产英文名称、短/长描述、操作说明、隐私与审核说明初稿，并整理公共提交清单。
7. 生成并验证方形、横版、竖版封面候选，以及平台截图和预览视频候选；最终视觉选择由用户确认。
8. 持续运行 `pnpm release:check`，修复我能复现的P0/P1，并维护代码、测试和任务状态。

### 收到人工反馈后由我完成

1. 根据24关试玩JSON区分规则、地图/开局、羊AI、界面/教学、奖励和技术问题。
2. 对有证据的单关执行单变量调优，重跑固定种子矩阵、候选门禁与关键棋谱，并更新对应关卡设计卡。
3. 修复真实设备发现的布局、触控、音频、生命周期或恢复问题，再执行回归门禁。
4. 根据平台Inspector或Portal反馈修订SDK、包体、素材和提交说明。

### 必须由用户或平台完成

1. 用户实际试玩24关并填写结构化结论；当前正式人工确认仍为0/24。
2. 至少使用真实手机/平板/桌面设备完成关键路径和主观体验确认。
3. 注册并提交Poki准入、创建CrazyGames项目，接受平台协议并管理账号、密钥和团队权限。
4. 在Poki Inspector、CrazyGames Preview/Portal中执行平台侧检查并提交审核。
5. 确认最终封面、视频和商店文案是否符合品牌预期；处理平台要求的版权或主体证明。

### 当前不应开发

- 悔棋、高级提示、账号/云存档、专业资源重制、长期奖励深化、App壳、原生广告和IAP仍在需求池，启动条件尚未满足。
- Poki/CrazyGames真实SDK暂不开发；先维护适配边界、Mock、构建变体和平台材料。

## T-005与T-016重点

- `T-005`工程准备状态（2026-07-17）：已完成。Admin关卡台默认按 `reject → 长尾 → review → pass` 排序，可筛选人工验收状态；单关试玩自动记录对局指标，并结构化记录设备、玩家经验、策略/教学/棋盘/奖励理解、问题归因、严重性和复现步骤。
- 正式试玩通过只统计“当前关卡版本一致 + 至少一局完整终局 + 10项证据完整 + 人工结论为通过”的关卡；旧记录继续兼容，但资料不完整时不计入24关通过数。
- 人工执行顺序：先复盘 `spring-03`、`winter-05` 两张长尾关，再完成其余11张 `review`，最后抽查11张 `pass`；发现问题先按规则、地图/开局、羊AI、界面/教学、奖励或技术异常归因。
- `T-005`记录关卡、设备、玩家经验、尝试次数、结果、策略理解、奖励理解和可复现问题。
- `T-016`自动门禁准备（2026-07-17）：已覆盖损坏/不可用存储降级、活动局面配置不匹配与终局快照拒绝、首次奖励幂等和24关配置迁移；新增静态资源引用检查。首页、章节、皮肤、任务、设置和对局入口已在375/390/414/1280四种宽度完成24组冒烟，无横向溢出或控制台错误。

## 管理规则

1. 当前不再增加影响核心规则、AI和24关结构的大功能；发现必要问题时先记录证据再调整范围。
2. 同一任务只在本文维护状态；产品规范不写第二套进度。
3. 新想法先进入需求池，具备价值、触发条件和验收标准后才迁入本文。
4. 平台政策会变化，`T-017`执行时以官方文档和平台检查器为准。
5. 当前总览和任务清单只维护未完成事项；完成事实移至已完成目录或由代码、测试和Git记录证明。
6. 平台上架不是当前硬阻塞；用户可以并行进行独立站/H5部署和项目质量验证。
