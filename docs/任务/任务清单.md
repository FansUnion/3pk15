# Fangrush 当前任务清单

> 本文只维护现在执行的工作。已完成事实由代码、测试和 Git 记录证明；尚未启动的功能见 [需求池](需求池.md)。

## 当前目标

近期只聚焦两条主线：

1. **生产环境用户体验验收**：先由 AI 完成可自动检查和可复现问题修复，再由用户验证真实设备、Admin 易用性、皮肤审美和24关体验。
2. **平台基础材料准备**：准备 Poki 准入表单和 CrazyGames Basic 首次提交所需的文案、截图、封面、视频及主体信息清单；不接真实 SDK，不做平台联调。

## 当前明确待办

### AI 可直接完成

- 检查 `fangrush.com` 与 `admin.fangrush.com` 的公开状态、路由、资源和部署结果。
- 在桌面、平板、手机视口执行自动页面检查，修复确定性的布局、溢出、资源和交互问题。
- 持续运行发布门禁；代码变化后保持玩家包不包含 Admin、密钥和非目标平台能力。
- 补齐平台基础材料的文案、规格、文件命名、候选截图和待确认项。
- 收到试玩反馈后完成归因、修复、固定种子复测和关卡资料更新。

### 必须由用户完成

- 使用真实手机、平板和桌面验证触控、阅读、音频和整体感受。
- 实际使用 Admin 的关卡、AI、皮肤、经济和检查页面，反馈难懂或低效的地方。
- 逐步试玩24关；当前正式人工验收仍为 `0/24`。
- 提供表单所需的真实姓名/邮箱/国家/主体信息，确认最终封面、视频、英文文案和素材权利。
- 登录平台、接受协议并点击最终 Submit。

## 当前不做

- Poki/CrazyGames 真实 SDK、广告、生命周期、Inspector/Portal 联调和正式平台接入已移入 [需求池](需求池.md)，待线上体验稳定且用户决定启动后再迁回任务清单。
- 悔棋、高级提示、账号/云存档、专业资源重制、App、原生广告和 IAP 继续保留在需求池。

## 验收口径

- 自动化通过不能代替真实设备体验和24关人工结论。
- 人工反馈不要求一次完成；每次提供页面、设备、操作、预期和实际表现即可，由 AI 负责复现、归因和回归。
- 平台“基础材料完成”不等于已接 SDK、已通过 Inspector、已提交审核或已上架。
