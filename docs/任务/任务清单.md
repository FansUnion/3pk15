## 状态定义

- `未开始`：需求已确认，尚未实施。
- `进行中`：正在实现或验证，且当前可继续推进。
- `部分完成`：已有代码或方案，但未达到验收条件。
- `依赖中`：需求正确，但必须等待前置任务完成。
- `暂缓`：保留需求，当前阶段不得开发。
- `不纳入首发`：需求可能有价值，但不是当前首发成功条件，不得阻塞上线。
- `待淘汰确认`：当前证据认为不合格，修订后必须重新走候选验收；不能继续标记为生产通过。
- `已完成`：验收条件已由代码、测试或实际产品路径确认。

已验收的基础能力和关闭任务见 [已完成记录](已完成/README.md)，不在本清单重复展示。

## 当前结论：24 关不是 24/24 合格

候选验收器给出的当前状态必须作为任务事实维护：

- `pass`：11 关。只表示没有命中自动红线，仍需玩家试玩。
- `review`：13 关。当前主要命中策略敏感，需确认好策略是否能被玩家理解和执行。
- `reject`：0 关。`winter-01`、`winter-03`、`winter-05` 已通过单变量羊阵修订退出 reject，但仍需玩家试玩。

因此当前可表述为：**24/24 配置可运行，11/24 自动门禁通过，13/24 待复盘或玩家确认，0/24 reject，0/24 玩家试玩确认。**

## 更新规则

1. 开始任务前，在此处将状态改为 `进行中`，并保留依赖关系。
2. 完成任务时，必须记录代码位置、验证方式、数据变化和未解决风险；满足验收条件后才能改为 `已完成`。
3. 新需求先挂到对应阶段；若改变核心规则、AI、布局或平衡目标，先更新核心需求基线，再创建任务。
4. 已过时的方案不删除，改为“参考”并链接到替代任务或现行文档。
