## 当前目标

独立站和 Admin 已完成 Vercel 部署；当前由用户集中体验并提出一批结构化问题，随后进行代码、UI 和关卡问题收口，最后再进入 H5 平台联调和提交。长期功能继续留在需求池。

## 剩余工作与责任边界

### 我可以直接完成

1. 完成双H5平台可在本地准备的合规矩阵、构建证据、文案、素材候选和Mock测试。
2. 持续运行 `pnpm release:check`，修复可复现的P0/P1并维护当前状态。
3. 收到部署或试玩反馈后，完成代码修复、固定种子回归和关卡资料更新。

### 收到人工反馈后由我完成

1. 根据24关试玩JSON区分规则、地图/开局、羊AI、界面/教学、奖励和技术问题。
2. 对有证据的单关执行单变量调优，重跑固定种子矩阵、候选门禁与关键棋谱，并更新对应关卡设计卡。
3. 修复真实设备发现的布局、触控、音频、生命周期或恢复问题，再执行回归门禁。
4. 根据平台Inspector或Portal反馈修订SDK、包体、素材和提交说明。

### 必须由用户或平台完成

1. 用户实际试玩24关并填写结构化结论；当前正式人工确认仍为0/24。
2. 至少使用真实手机/平板/桌面设备完成关键路径和主观体验确认。
3. 注册并提交Poki准入、创建CrazyGames项目，接受平台协议并管理账号、密钥和团队权限。
4. 在Poki Inspector、CrazyGames Preview/Portal中执行平台侧检查并提交审核。
5. 确认最终封面、视频和商店文案是否符合品牌预期；处理平台要求的版权或主体证明。

### 当前不应开发

- 悔棋、高级提示、账号/云存档、专业资源重制、长期奖励深化、App壳、原生广告和IAP仍在需求池，启动条件尚未满足。
- Poki/CrazyGames真实SDK在取得平台权限和现行接入规格后开发；当前只完成本地适配边界、Mock、构建变体和材料。

## 验收待办

- 正式试玩通过只统计“当前关卡版本一致 + 至少一局完整终局 + 10项证据完整 + 人工结论为通过”的关卡；旧记录继续兼容，但资料不完整时不计入24关通过数。
- 人工执行顺序：先复盘 `spring-03`、`winter-05` 两张长尾关，再完成其余11张 `review`，最后抽查11张 `pass`；发现问题先按规则、地图/开局、羊AI、界面/教学、奖励或技术异常归因。
- 用户在正式域名上完成手机、平板和桌面关键路径，提交结构化问题；AI负责修复、回归和更新关卡结论。

## 管理规则

1. 当前不再增加影响核心规则、AI和24关结构的大功能；发现必要问题时先记录证据再调整范围。
2. 同一任务只在本文维护状态；产品规范不写第二套进度。
3. 新想法先进入需求池，具备价值、触发条件和验收标准后才迁入本文。
4. 平台政策会变化，`T-017`执行时以官方文档和平台检查器为准。
5. 当前总览和任务清单只维护未完成事项；完成事实移至已完成目录或由代码、测试和Git记录证明。
6. 平台上架不是当前硬阻塞；用户可以并行进行独立站/H5部署和项目质量验证。
