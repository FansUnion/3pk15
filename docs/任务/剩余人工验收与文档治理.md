# 剩余人工验收与文档治理

> 本文只维护不能由当前自动开发直接关闭的人工体验和文档治理。羊 AI V2 的开发、自动验证、Admin和发布门禁已经完成，证据见[24关验证记录](../产品核心/验证记录/2026-07-19-羊AI-v2-24关验证.md)；这里仅保留玩家体感抽查。

## 一、用户最终验收

- 手机、平板、桌面完整旅程与弹层、触控、连吃操作。
- 14类声音的真实设备听感。
- 新版岩石的儿童辨识，以及霜狼、霜羊、Night Watch 羊的审美。
- 24关学习顺序、策略可读性、难度跳跃和目标8体感。
- 羊 AI 高风险关按以下顺序抽查：`spring-02`、`summer-01` 看是否只会拖延；`summer-03`、`summer-06`、`autumn-06` 看首次接触路线是否可读；`winter-02/03/04/06` 看专家防守是否有压力但仍可赢。
- 抽查时每关只需给出“太笨、合理、有压力但可赢、只会拖延、过难”之一；遇到明显送子或往返，再通过“报告本局问题”导出 JSON。

这些不是代码“未开发”，而是不能由自动化代替的产品验收。用户只需反馈现象和感受，复现、归因、修复、棋谱与回归由AI开发侧完成。

## 二、文档治理

- 全量 Markdown 链接检查发现约40处失效相对链接，集中在商业平台双语资料、普通资源文档和游戏创意旧入口。
- 本次迁移的产品核心与任务链接正常。
- 后续只清理现行高价值文档：能映射到现行文档则修正，职责已经消失的旧入口删除；不恢复过时文档体系，不全面翻新历史原始材料。

## 三、独立事项

- H5基础材料仍是当前并行任务；真实SDK和平台联调仍在需求池。
- 独立站SEO是长期规范，不属于AI或人工体验专项。
- 账号、悔棋、高级提示和App继续按需求池启动条件管理；所有长期游戏内容只在[产品扩展](../产品扩展/README.md)维护。

## 四、关闭条件

1. 用户人工验收项均有“通过”或具体反馈。
2. AI收到的反馈均已复现、归因和回归。
3. 现行高价值文档不存在失效入口。
4. 完成后删除本文件；长期标准继续由产品核心文档维护。
