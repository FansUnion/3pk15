# 全面诊断与解决方案

## 1. 皮肤与游戏资源

### 诊断

- 现有资源的角色识别度不足，玩家在正常观看距离下可能无法立即分辨狼和羊。
- 部分资源更像装饰图形，不像完整的角色主题。
- 皮肤名称、预览、碎片成本、解锁方式和实际激励之间的联系不够强。
- 夜间或特殊主题如果过早获得，缺少稀缺感；如果长期看不到，也缺少目标提示。

### 解决方案

- 先建立角色识别标准：轮廓、头部特征、体态、颜色对比和最小显示尺寸。
- 每套主题必须同时定义狼、羊、棋盘背景、状态色和音效气质。
- 后台增加资源验收字段：识别度、主题一致性、缩小可读性、明暗对比、移动端表现。
- 前台展示“主题预览 + 解锁目标 + 当前进度 + 装备后效果”，让皮肤成为明确的长期目标。
- 专业资源生产暂不强行塞入当前开发批次，但所有接口和展示容器必须支持后续替换。

## 2. 关卡设计与平衡

### 诊断

- 第一关同时承担教学和留存，若缺少岩石或关键策略元素，玩家可能没有体验到游戏完整价值。
- 只看狼羊数量无法判断公平性，必须考虑初始位置、可行动空间、首回合优势、岩石阻挡和强制路线。
- 固定布局有利于教学和复盘，但连续关卡全部固定会降低重复游玩变化。
- 目前需要建立“地图为什么这样设计”的可追溯记录。

### 解决方案

- 第一关采用“低认知负担但包含核心差异”的设计：不堆规则，但至少让玩家理解选择、移动、吃子和阻挡中的核心体验。
- 每关记录：目标策略、教学点、狼方优势、羊方逃生空间、岩石作用、预计回合数、失败原因和测试结果。
- 使用固定布局承担教学，使用多布局或受控变体承担长期重玩；变体必须经过规则引擎验证。
- 建立关卡测试矩阵：新手人工测试、熟练玩家测试、AI 模拟、极端走法测试和无效操作测试。
- 不以“能通关”作为唯一标准，同时关注首次理解时间、平均对局时长、重试率和异常挫败点。

## 3. 新手引导与留存

### 诊断

- 新玩家需要同时理解角色、回合、可走位置、吃子、连吃和胜负条件。
- 直接把正式第一关承担全部教学，会造成信息密度和关卡平衡之间的冲突。
- 单独训练章节会增加入口成本，但能降低正式关卡的教学负担。

### 解决方案

- 采用“短引导 + 第一关实战”的组合：先用几步可跳过的交互提示讲清基本操作，再进入正式第一关。
- 引导只解释当前动作，不一次讲完全部规则。
- 每个提示必须有跳过、继续、重新查看和无障碍可读状态。
- 第一关负责让玩家完成一次有意义的策略选择，而不是负责解释所有系统。
- 首次胜利后立即展示奖励、下一关和再玩一次，减少玩家停在结果页的概率。

## 4. 棋局交互、动画和音效

### 诊断

- 玩家需要在短时间内知道“我选中了谁、能走哪里、这一步是否成功、现在轮到谁”。
- 仅有颜色或轻微动画时，移动端和低注意力场景容易丢失反馈。
- 音效如果没有静音状态、重复播放控制和关键事件分级，会造成打扰。

### 解决方案

- 将反馈分为选择、可走、落子、吃子、连吃、AI 思考、胜利、失败和无效操作九类。
- 关键反馈至少具备两种通道：视觉加声音；静音时视觉仍必须完整。
- 动画优先表达空间关系和结果，控制时长，避免操作被长动画锁住。
- 明确 reduced-motion 策略：减少位移和闪烁，但保留状态差异和结果反馈。
- 所有音效由统一的静音开关控制，并防止同一事件短时间重复叠加。

## 5. 页面和导航体验

### 诊断

- 首页、章节、关卡、皮肤、设置和棋局之间需要形成稳定的回路。
- 用户必须始终知道自己在哪里、下一步做什么、返回后会回到哪里。
- 桌面端需要充分利用宽度，移动端需要控制信息密度和按钮拥挤。

### 解决方案

- 统一页面头部、主要行动按钮、返回路径和页脚导航。
- 明确区分“返回关卡列表”“退出到首页”“重试当前关卡”“继续上次游戏”。
- 所有重要页面都提供至少一个明确的下一步行动。
- 验证 375、390、414、1024、1280、1440 和 1920 像素视口。
- 检查键盘焦点、按钮名称、图片替代文本、颜色对比、错误状态和刷新恢复。

## 6. 商业化

### 诊断

- 过早加入广告会破坏新手学习和第一局情绪。
- 撤回、双倍奖励、碎片和皮肤可以形成商业化入口，但必须防止变成强制付费或广告通关。
- 广告展示频率不能只凭感觉，需要按用户阶段和行为数据控制。

### 解决方案

- 第一阶段只设计激励型广告，不在新手首局中断流程。
- 广告只能提供可选收益，例如额外碎片、一次额外撤回或胜利奖励翻倍。
- 每局广告上限、每日上限和冷却时间由产品规则控制，不由第三方 SDK 自己决定核心体验。
- SDK 负责广告加载和回调；游戏负责展示时机、资格判断、奖励发放和失败兜底。
- 广告未加载、用户拒绝或播放失败时，基本玩法和普通奖励仍然可用。
- 在有真实用户量后再根据完成率、留存、广告收益和投诉率调整频率。
