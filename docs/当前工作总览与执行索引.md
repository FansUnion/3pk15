# Fangrush 当前工作总览

> 当前状态以 [任务清单](任务/任务清单.md) 为准。本文只说明目标、边界和入口。

完整执行顺序和每批任务的解决目标见[执行计划：地图、AI、体验与发布准备](任务/执行计划-地图AI体验与发布准备.md)。

## 当前阶段

玩家端与 Admin 已拆为两个独立应用并部署到 Vercel：

- 玩家端：`https://fangrush.com`
- Admin：`https://admin.fangrush.com/admin`
- Admin 功能统一位于 `/admin/*`；玩家端不展示 Admin 入口。

代码侧发布基线已建立：玩家端和 Admin 生产构建、应用边界、资源、测试与发布门禁均通过。第二批全流程体验和第三批 AI、终局解释、问题复现与声音可靠性专项已经完成。第四批试玩确认旧自动矩阵只使用单吃狼代理，且现有标准缺少岩石拓扑、地图组合差异和羊方优势转化证据；当前先重建地图与AI验收体系，再完成24关定稿，同时继续平台基础材料。

## 三条近期主线

### 1. 生产环境体验验收

第二、三批确定性问题已经修复并通过自动回归。用户继续在真实手机、平板和桌面体验玩家端与 Admin；局面、AI、终局或声音问题优先通过玩家端“报告本局问题”导出 JSON，Admin 可直接导入和回放。

### 2. 地图、AI与24关验收

T-025 验收体系与 T-005 三批地图重建已经完成。24关生产自动验收为 `24/24`，诊断标签为 `8 pass / 16 review / 0 reject`；非空岩石布局与完整开局均无相同、镜像或旋转重复。玩家按学习顺序体验并反馈问题，不再阻塞生产使用。

### 3. 平台基础材料

近期只准备 Poki early access 和 CrazyGames Basic 首次提交需要的资料，包括字段口径、文案、截图、封面、视频和主体信息清单。真实 SDK、广告、生命周期、Inspector/Portal 联调不是当前任务，统一保留在需求池。
