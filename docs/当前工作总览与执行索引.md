# Fangrush 当前工作总览

> 当前状态以 [任务清单](任务/任务清单.md) 为准。本文只说明目标、边界和入口。

## 当前阶段

玩家端与 Admin 已拆为两个独立应用并部署到 Vercel：

- 玩家端：`https://fangrush.com`
- Admin：`https://admin.fangrush.com/admin`
- Admin 功能统一位于 `/admin/*`；玩家端不展示 Admin 入口。

代码侧发布基线已建立：玩家端和 Admin 生产构建、应用边界、资源、测试与发布门禁均通过。第二批全流程体验和第三批 AI、终局解释、问题复现与声音可靠性专项已经完成。下一阶段不扩张核心规则和大型功能，聚焦真实体验验收、24关人工试玩和平台基础表单材料。

## 三条近期主线

### 1. 生产环境体验验收

第二、三批确定性问题已经修复并通过自动回归。用户继续在真实手机、平板和桌面体验玩家端与 Admin；局面、AI、终局或声音问题优先通过玩家端“报告本局问题”导出 JSON，Admin 可直接导入和回放。

### 2. 24关人工试玩

自动矩阵和24张关卡资料卡已经具备，但正式人工结论仍为 `0/24`。用户可分批试玩并反馈理解度、难度和放行意见；AI负责复现、归因、固定种子复测和更新单关证据。

### 3. 平台基础材料

近期只准备 Poki early access 和 CrazyGames Basic 首次提交需要的资料，包括字段口径、文案、截图、封面、视频和主体信息清单。真实 SDK、广告、生命周期、Inspector/Portal 联调不是当前任务，统一保留在需求池。

## 当前边界

- 24关自动模拟和资料卡已具备，但人工验收仍为 `0/24`。
- 平台本地构建边界和合规矩阵已具备，但不表示平台审核或真实 SDK 已完成。
- 已完成事项不继续堆积在任务清单；长期功能不进入当前执行队列。
