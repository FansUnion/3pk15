# 帮助中心、局内指导与策略教学方案

> 实现状态（2026-07-17）：帮助中心与三级局内指导已完成；提示使用和连续失败按关卡记录。悔棋、广告、自动落点与完整路线演示未实施，继续暂缓。

## 1. 工作定位

本项工作同时服务四个目标：

1. 让新玩家快速理解规则并完成有效操作。
2. 让玩家逐步发现狼方的核心策略，而不是依赖随机试错。
3. 用可选的提示和悔棋降低挫败感，提升留存和复玩。
4. 将玩家策略、提示使用和胜率数据反向用于关卡设计与难度平衡。

核心原则：帮助功能服务玩法，不替代玩法；广告提供额外资源，不阻断基础体验。

## 2. 帮助中心结构

入口：全站统一的帮助图标，首页、章节页、关卡页和游戏页均可访问；游戏中打开帮助时保留当前关卡上下文。

内容分为四组：

- 基础规则：狼、羊、棋盘、岩石、移动、隔空吃羊、连续吃、胜负与和棋。
- 操作说明：选择狼、选择落点、结束连续吃、重新开始、返回和静音。
- 狼方基础技巧：双狼夹击、三狼协作、限制出口、保持机动性、利用岩石。
- 进阶策略：诱导羊群、先控制路线再吃羊、连吃前判断落点、避免为了单羊破坏整体包围。

内容采用短段落、棋盘图示和可演示局面，不做一篇难以阅读的长说明。

## 3. 局内指导系统

指导分为五级，避免直接替玩家解题：

1. 观察提示：指出应该关注的区域。
2. 目标提示：建议选择哪只狼或哪条路线。
3. 方向提示：提示下一步方向，但不直接执行。
4. 落点提示：显示一个推荐落点。
5. 路线演示：只在连续失败或主动请求时展示短路线。

基础教学、首次出现的新机制和失败后的针对性建议免费提供。提示内容必须绑定关卡教学目标，例如“双狼夹击”关卡不能只提示“往左走”。

## 4. 悔棋与广告边界

建议每局提供 1 次免费悔棋。额外悔棋可由玩家主动选择观看激励广告获得；广告失败时不扣除次数，并给出明确反馈。

悔棋规则：

- 只回退最近一次玩家操作。
- 连续吃过程中回退到该次连续吃开始前。
- 不回退奖励、广告领取记录或关卡解锁状态。
- 不在思考过程中强制弹出广告。

适合激励广告的场景：额外提示、额外悔棋、失败后的辅助重试、结算后的额外奖励。不应在首次规则教学、每步操作、胜利结算中强行打断。

## 5. 狼方策略知识库

长期维护以下策略标签，并关联到帮助内容和关卡：

- `双狼夹击`：两只狼从不同方向压缩羊群空间。
- `三狼包围`：两只狼限制路线，第三只狼完成捕捉。
- `诱导移动`：先走非捕捉路线，逼羊进入目标区域。
- `岩石借力`：利用岩石作为跳跃中点或阻断点。
- `链式捕捉`：第一次吃羊后判断是否继续连吃。
- `保持机动性`：避免所有狼同时进入无法转向的位置。

每张地图应至少有一个主要策略标签和一个辅助策略标签，玩家提示、关卡教学点和验收标准保持一致。

## 6. 反向用于平衡开发

每关建立以下记录：

- 目标策略。
- 普通玩家狼方胜率目标。
- 熟练玩家狼方胜率目标。
- 和棋率上限。
- 平均回合目标。
- 提示使用率和悔棋使用率。
- 玩家使用目标策略后的胜率变化。

判断规则：

- 玩家不会使用目标策略：优先改教学和提示。
- 玩家理解策略但仍频繁失败：调整初始位置、岩石或羊群路线。
- 狼方长期接近必胜：增加羊群逃生空间或提高羊 AI。
- 羊方长期必胜：增加狼的协作路线，减少封闭岩石或降低羊 AI。
- 和棋集中在单张地图：优先重排地图，不直接提高 AI。

## 7. 执行阶段

1. 建立帮助中心页面和统一入口。
2. 建立关卡教学目标与策略标签数据结构。
3. 接入免费基础提示和失败后建议。
4. 实现每局一次免费悔棋，并完成连续吃回退规则。
5. 增加激励广告接口，但先保留可关闭的开发测试开关。
6. 将提示、悔棋、胜负和策略标签纳入平衡模拟与数据记录。
7. 进行新手、普通玩家和熟练玩家三类场景验收。

## 8. 验收标准

- 玩家无需看广告即可理解基础规则并完成一局。
- 帮助中心在所有主要页面可访问，游戏中不会丢失当前关卡上下文。
- 提示逐级提供信息，不直接替玩家完成整局。
- 悔棋不会破坏奖励、存档和关卡解锁。
- 广告只出现在主动领取额外资源的场景。
- 每张主线地图都有策略标签、教学点和胜率目标。
- 模拟结果和真实玩家数据都能追溯到具体地图与策略。
