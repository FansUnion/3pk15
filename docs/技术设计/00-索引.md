# 技术设计 · 索引

## 职责

本目录只维护**高密度技术决策**：架构总览、工程红线、页面/SEO 定稿。  
玩法意图 → [`游戏创意/`](../游戏创意/00-文档索引.md)。平衡与 AI 数值 → [`产品质量提升/`](../产品质量提升/) + [`任务清单`](../任务/任务清单.md)。  
算法、字段、权重以**代码**为准；文档不抄实现说明书。

| 写 | 不写 |
|----|------|
| 层次边界、禁止事项、定稿决策 | 函数签名、权重表、字段默认值 |
| 改模块时必须对齐的红线 | 波次勾选、验收流水账 |
| 与创意 / 任务的指针 | 玩法复述、未冻结的 AI 胜率表 |

## 维护纪律

1. **人读入口只有 `07`；改内核先对 `01` 红线。**  
2. **改规则 / AI / 存档结构**：先改代码 + Vitest；红线变了再改 `01` 对应行；玩法意图变了才改创意。  
3. **改路由 / 语言 / 收录**：改 `06` + 实现；不要在总览里展开 SEO。  
4. **AI / 关卡平衡未冻结时**：技术设计只写入口与 fixture 红线；数值与矩阵放产品质量提升 / 任务清单，不进 `07`/`01` 正文。
