# 02 · 规则引擎与状态机

> 玩法口径：[`docs/游戏创意/02-核心对局规则.md`](../游戏创意/02-核心对局规则.md)、[03](../游戏创意/03-开局布局与坐标.md)。  
> UI 与 AI **只许**调用 core API，不得自行判定吃子。

## 棋盘约定

- `Pos = { r: 1..6, c: 1..6 }`，邻接四向，禁止斜向。  
- 岩石不可占用、不可穿越。  
- 狼 3、羊最多 15；同点至多一子，不与岩重叠。  
- 开局与创意 03 一致。

## 逻辑状态（要点）

`BoardState` 含：棋子、岩石、`eatenSheep`、`toMove`、连吃 `chain`（狼 id + 已吃次数 1..5）、`status`、`levelId`。

**不变量（每次 apply 后成立）**

1. 坐标合法；无两子同点；无子压岩。  
2. `eatenSheep` = 15 − 场上羊数。  
3. `chain` 仅在狼回合且上一手为该狼 jump 时非空。  
4. 非 `playing` 时不再接受对局着法（可 reset）。

## 公共 API（唯一入口）

| 函数 | 职责 |
|------|------|
| `createInitialState` | 开局 |
| `listLegalActions` | 合法着；连吃中只返回该狼 jump |
| `applyAction` | 应用着法；非法须失败且不改状态 |
| `endWolfTurn` | 主动结束连吃 / 狼回合 → 切羊 |
| `evaluateTerminal` / `refreshStatus` | won：`eatenSheep≥8`；lost：三狼无着 |

着法类型：`step` | `jump`（`through`=中间空点，`to`=羊位兼落点）。**隔空吃 = 狼—空—羊，冲到羊位。**

## 关键规则（实现写死）

| 项 | 约定 |
|----|------|
| 狼 jump | 中间空且无岩；落点有羊；曼哈顿 2、同线 |
| 连吃 | ≤5；有吃可不强制；UI 可「结束连吃」→ `endWolfTurn`；无剩余 jump 自动结束 |
| 羊 | 每回合一步 step；禁止向第 1 行后退 |
| 胜负时机 | jump 后查胜；羊走完 / 狼回合开始查三狼无着败 |

## UI 会话（配合 rules）

`IdleSelectWolf` → `ShowMoves` → `Animating` →（连吃）`ChainSelect` 或 `AiTurn` → `Terminal`。  
`Animating` / `AiTurn` 忽略棋盘点击。

## 测试最低集

开局着法、隔空吃几何、岩挡/紧贴不可吃、连吃第 5 次切羊、吃满 8 胜、羊禁后退、三狼堵死败、非法 apply、`endWolfTurn`、随机对局不变量。

## 性能

全盘着法成本可忽略。AI 搜索不得改全局单例；用不可变或显式 clone。
