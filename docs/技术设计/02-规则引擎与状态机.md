# 02 · 规则引擎与状态机

> 玩法口径：[`docs/游戏创意/02-核心对局规则.md`](../游戏创意/02-核心对局规则.md)、[03](../游戏创意/03-开局布局与坐标.md)。  
> **真相源代码**：`packages/game-core/src/rules.ts`（及同包 FSM）。  
> UI 与 AI **只许**调用 core API，不得自行判定吃子。

## 必须怎样

- 着法唯一入口：`listLegalActions` → `applyAction`；非法着法失败且不改状态。
- 连吃 ≤5；有吃可不强制；无剩余 jump 自动结束；UI 可主动 `endWolfTurn`。
- 隔空吃几何：同线 **狼 — 空 — 羊**，中间空且无岩，落到羊位。
- 胜：`eatenSheep ≥ 8`；败：三狼皆无合法着。
- 羊每回合一步、禁止向第 1 行后退。

## 不许怎样

- UI / Admin 自写吃子或胜负判定。
- 斜走、穿越岩石、同点叠子。
- 非 `playing` 时仍接受对局着法（reset 除外）。

## 测试红线

开局着法、隔空吃、岩挡/紧贴不可吃、连吃第 5 次切羊、吃满 8 胜、羊禁后退、三狼堵死败、非法 `applyAction`、`endWolfTurn`、随机对局不变量。

AI 搜索不得改全局单例；用不可变或显式 clone。
