# 05 · Admin 职责地图（Content & Quality Cockpit）

> 独立站 `/admin` 是开发者与平台方的**内容质量驾驶舱**，不是 CMS、不是用户后台、不是远程数据看板。

## 定位

| 角色 | Admin 回答的问题 |
|------|------------------|
| 开发者 | 配置合法吗？资源齐吗？AI 与线上一致吗？这关难不难？ |
| 内容制作 | 皮/盘玩家侧长什么样？解锁说得通吗？双语 blurb 像成品吗？ |
| 产品/平台 | 漏斗哪断？经济闭环通不通？难度会劝退吗？上架材料够吗？ |

## 硬约束

- 只读 `game-core` Catalog / LEVELS / QUEST + 静态资源 HEAD。
- **不写**玩家 `SaveGame`；改内容改源码后重部署。
- 羊 AI **必须**调用同一 `pickSheepAction`；禁止 admin 另写。
- 不做：写库、账号、远程漏斗图表、真广告控制台。

## 各台 JTBD

| 台 | 职责一句话 | 解决什么 |
|----|------------|----------|
| 皮肤台 | 身份感与投放验收 | 缺图、皮盘混看不清、解锁价离谱 |
| 关卡台 | 关卡质量与 SEO 文案验收 | 岩石违规、开局不可读、blurb 空/双语不一致 |
| AI 模拟台 | 难度校准与坏局诊断 | 过虐/过水、合围回归、复现局面 |
| 玩法说明 | 与玩家 `/how-to-play` **同源** | 两套规则文案漂移 |
| 经济与解锁 | 碎片→皮肤闭环只读表 | 任务奖励与皮价脱节 |
| 上线检查清单 | 漏斗断点本机勾选 | 挂站前「以为做完了」 |

## 批量胜率口径

AI 台「批量校准」：狼方为**合法着法均匀随机**（非狼 AI），羊方为线上同档 `pickSheepAction`。数字用于粗校准，不是玩家真实胜率预测。
