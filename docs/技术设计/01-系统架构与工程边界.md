# 01 · 系统架构与工程边界

## 部署优先级

| 优先级 | 目标 |
|--------|------|
| P0 | 独立站（Vercel）可玩 + 简易 `/admin` |
| P1 | 经济/广告闭环（碎片、皮肤、激励） |
| P2 | 门户 H5 瘦包（无 admin、默认零自建遥测） |
| 预留 | App（Capacitor）只留端口 |

玩家账号：**不做**（一期）。

## 技术选型（定稿）

| 层 | 选型 | 禁止 |
|----|------|------|
| 语言 | TypeScript strict | — |
| Web | Next.js App Router | 对局 AI 放 Server Route |
| UI | React + Tailwind | 组件内写吃子/胜负判定 |
| 棋盘 | SVG | Phaser / Unity |
| 逻辑 | `packages/game-core` 纯 TS | core 依赖 DOM / Next / zustand |
| 存档 | localStorage + `schemaVersion` | 无迁移的破坏性改结构 |
| 测试 | Vitest 测 core | 无单测合并规则/AI 行为变更 |
| 包管理 | pnpm | — |

## 分层

```
Shells: 独立站 / 门户瘦包 / App预留
  → Adapters: IStorage / IAds / IAdminGate
  → App UI: 玩家页 + /admin
  → game-core（零 DOM）: rules | fsm | ai | content
```

**原则**

1. 玩法真相源：`docs/游戏创意/`；实现约束：本目录 `02`–`04`。  
2. core 不依赖壳；差异进 Adapter，禁止 `if (portal)` 散落规则层。  
3. 后台与玩家共用 `pickSheepAction` / `applyAction`，禁止 admin 另写 AI。

## 包边界

| 包 | 职责 |
|----|------|
| `packages/game-core` | 规则、FSM、AI、只读关卡/皮肤/掉落 |
| `apps/web` | Next 壳、UI、adapters、admin |
| `content/skins` | SVG 资源 |

`game-core` **禁止**：`react`、`next`、`dom`、广告 SDK、直接读 `window`/`localStorage`。

## 多端与适配器

| 端 | `/admin` | 广告 | 遥测 |
|----|----------|------|------|
| 独立站 | 可开 | AdSense 或 Mock | 可后接，可关 |
| 门户 | 必须无 | 门户 SDK / Mock | **默认零自建远程** |

关键 env：`NEXT_PUBLIC_APP_SHELL`、`NEXT_PUBLIC_ADS_PROVIDER`、`ADMIN_ENABLED`、`ADMIN_ACCESS_KEY`。完整粘贴清单见 [`docs/任务/未完成.md`](../任务/未完成.md)「生产环境变量」。

### IAds

- `showInterstitial`：失败也 resolve，UI 继续。  
- `showRewarded`：**仅 ok** 发奖。  
- 结算组件禁止直接调第三方全局对象；一律走 Adapter。  
- 产品口径 → 创意 [09](../游戏创意/09-广告与变现设计.md)。

### IStorage

- `getItem` / `setItem`；键如 `wolf-sheep-save-v1`。  
- 读写失败 → 回退默认存档。

## 简易后台（约束）

- 独立站 `/admin`：只读皮肤/关卡 + AI 模拟台。  
- **不做**：写库改配置、运营报表、玩家账号。  
- `ADMIN_ENABLED=false` → 404；可选密钥。门户构建剔除 admin。  
- 模拟必须调用同一 `pickSheepAction`；默认不写 SaveGame。

## 工程纪律

| 禁止 | 原因 |
|------|------|
| UI 内写隔空吃/胜负 | 双端分叉、不可测 |
| admin 复制 AI | 与线上漂移 |
| 无 fixture 改规则/AI 合入 | 回归失控 |
| 为性能把 AI 挪服务端 | 架构红线 |

**CI 最低**：lint + `game-core` test + `check:skins` + web build。  
改规则/AI 必须带 Vitest fixture；新增 `wolf_set` 缺羊资源 → CI 失败。
