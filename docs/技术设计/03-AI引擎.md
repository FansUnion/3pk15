## 统一接口

```ts
pickSheepAction(state, { difficulty, rng, budgets? }): Action
// 前置：toMove==='sheep' && status==='playing'
// 每回合只选出一个羊 step
```

对局 UI 与 `/admin` 模拟台 **必须**同一函数。  
同 `state` + 同 RNG 种子 → 同着法（hard 降级须可记录 `degraded`）。

**禁止**：按玩家胜率动态改档；深度学习；服务端算步。

## 三档算法要点

| 档 | 算法 | 要点 |
|----|------|------|
| `easy` | 合法着均匀随机 | 不调用威胁/评价；可送吃 |
| `normal` | 单层贪心 + 评价 | 堵 jump、降狼机动、避送吃；夏秋同权重，难度差交给岩石 |
| `hard` | 浅层 minimax（羊一步 + 狼一步回应） | 根候选 top-K≈24；狼节点不展开完整连吃；超时/`maxNodes` → 降级 normal |

## 评价函数（越大越利羊）

初值权重（改权重须跑批量模拟，不改胜负规则）：

| 项 | 权重方向 |
|----|----------|
| 狼机动性 `totalWolfMoves` | 强惩罚 |
| 危羊 `jumpTargets` | 更强惩罚（堵吃优先） |
| 横连抱团 | 奖励 |
| 中线羊数 | 轻奖励 |
| 本步送吃 | 极强惩罚 |
| 距狼距离 | 微弱 |

终局叶：困死三狼 → +大数；狼吃满 8 → −大数。

## 预算（hard）

默认约 `maxNodes: 8000`、`maxMs: 40`（浏览器主线程可调）。超限 → `pickNormal` + `degraded=true`。

## Fixture 最低集

| ID | 期望 |
|----|------|
| 唯一挡吃 | normal/hard **必须挡**；easy 可不挡 |
| 送吃诱惑 | normal/hard **不送** |
| 预算耗尽 | hard 降级 normal |
| 冬日合围样例 | hard 优先减机动（待补齐见任务） |

## 修订纪律

- 改权重：更新本节 + 批量模拟摘要。  
- 改搜索结构：更新本文 + 加 fixture。  
- 未改本文不得合并 AI 行为变更。
