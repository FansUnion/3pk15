# 03 · AI 引擎

> 行为意图：[`docs/游戏创意/06-AI行为设计规格.md`](../游戏创意/06-AI行为设计规格.md)、[05-四季](../游戏创意/05-四季关卡设计.md)。  
> **真相源代码**：`packages/game-core/src/ai/`。  
> AI **只许**消费 `listLegalActions` / `applyAction`（见 [02](./02-规则引擎与状态机.md)）。

## 必须怎样

- 统一入口：`pickSheepAction(state, { difficulty, rng, budgets? })`；对局 UI 与 `/admin` **同一函数**。
- 三档绑定章节：春 easy、夏/秋 normal、冬 hard（意图表见创意，此处不复述）。
- 同 `state` + 同 RNG 种子 → 同着法；hard 降级须可观察（如 `degraded`）。
- hard 须有 fixture；改权重或搜索结构须更新本文 + 批量/fixture。

## 不许怎样

- 按玩家胜率动态改档。
- 深度学习或服务端另写一套 AI。
- Admin / 模拟台绕开 `pickSheepAction`。

## Fixture 红线

| 场景 | 期望 |
|------|------|
| 唯一挡吃 | normal/hard 必须挡；easy 可不挡 |
| 送吃诱惑 | normal/hard 不送 |
| 预算耗尽 | hard 降级 normal |
| 冬日合围 | hard 优先减狼机动（仍欠，见任务清单 `T-002 AI 行为定型`） |

算法细节（贪心权重、minimax 预算等）以代码为准；改行为前先读 `ai/` 再改文档红线。
