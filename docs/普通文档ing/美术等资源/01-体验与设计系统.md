# 01 · 体验与设计系统

> **体验承诺**：这是牧场上的猎杀棋。棋盘是主角；每一步应看懂「选了谁、能去哪、吃了没有」。  
> 体验目标与发布门槛见 [核心产品定义](../../产品核心/01-核心产品定义.md)；未完成工作只进入 [任务清单](../../任务/任务清单.md) 或 [需求池](../../任务/需求池.md)。
> 换皮 / 换音规格 → [`02-视觉与音效资源规格`](./02-视觉与音效资源规格.md)。

## 七条感官硬约束

1. 棋盘是唯一主角（官网首屏与对局皆然）。  
2. 1 秒可辨狼 / 羊 / 岩（约 36px）。  
3. 对比优先于「淡雅发灰」。  
4. 一页一主动作。  
5. 反馈是事件，不是只改数字。  
6. 铬层（顶栏底栏说明）永远轻于棋盘。  
7. 跨页同气质（色板、按钮、棋子同源）。

## 禁止

- 紫霓虹、赛博光效、Excel 灰线盘  
- 圆头两点眼三角耳的贴纸羊/狼  
- 对局底栏三块等高空心大框（表单感）  
- 铬层压过棋盘；对局中部硬推广告  

## 色彩 / 字体 / 控件

| Token | 用途 | 要求 |
|-------|------|------|
| `--surface` | 页底 | 暖纸/浅草，对比够 |
| `--board` | 棋盘场 | 有纹理；四季变色温 |
| `--ink` / `--muted` | 主文 / 次文 | 炭墨绿；次文可读不融底 |
| `--accent` | 主 CTA / 选中 | 一块够跳的实色 |
| `--danger` | 可吃提示 | 与草地可分的红系 |
| 羊/狼 | 子力 | 羊亮狼暗；冬不把羊洗成灰 |

- Display（Literata 等）：品牌、关卡大标题、结算  
- UI Sans + 中文 fallback；HUD 数字 `tabular-nums`  
- 对局底栏：图标工具条或浅实心；触控热区 ≥ 44px  

## 动效预算

| 类型 | 时长 |
|------|------|
| 落子/吃子 | ~200ms |
| 羊思考空窗 | ~600ms（分级见任务 `P-AI-PACING`） |
| 按钮按下 | ~80–120ms scale 0.97 |
| `prefers-reduced-motion` | 缩短或取消位移，保留状态变化 |

## 交互摘要

| 交互 | 目标 |
|------|------|
| 选中狼 | 光环 + 阴影 + 微放大 |
| 可走/可吃 | 柔光圆；羊位强环 + through |
| 落子/吃 | 200ms 事件 + 音 |
| 连吃 HUD | k/5 变色加重 |
| 羊回合 | 静默挡点；顶栏「羊回合」 |
| 胜负结算 | 先可读再弹层；主按钮「再来」 |
| 重置 | 二次确认 + 危险色 |
| 静音 | 全局真静 |
| 引导春日1 | 短、可跳过；勿与常驻说明重复 |

## 官网壳与广告缝

- 首屏：盘面海报主角 + 一主 CTA；次入口降对比；语言下拉不抢戏；页脚弱 Admin  
- 插屏在结算后；激励在结算/关卡缝，仅 ok 发奖；对局中部不硬推  

## 无障碍与性能底线

发布前底线：

- 核心按钮、弹窗和导航具有可见焦点，键盘不会陷入无法退出的区域。
- 关键图片具有有意义的替代文本；纯装饰图片不制造读屏噪音。
- 选中、可走、危险和终局不只依赖颜色表达。
- 触控目标不小于44px；375/390/414宽度无溢出、遮挡和不可点击区域。
- `prefers-reduced-motion` 下取消大位移但保留状态反馈；静音时不丢失规则信息。
- 资源失败回退默认皮肤；加载、空态和异常状态提供明确下一步。

完整读屏、全键盘棋盘操作、高对比主题和细粒度音量属于数据驱动的高级增强，见 [需求池](../../任务/需求池.md)。
