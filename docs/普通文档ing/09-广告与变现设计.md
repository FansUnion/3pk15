# 09 · 广告与变现设计

> 本文只描述**已落地**的产品口径（触发与权益）。真广告接入与后续变现见 [`docs/任务/任务清单.md`](../任务/任务清单.md)「广告与商业化」。不含 SDK 实现细节。

## 总原则

- **无付费商城**；靠广告 + 碎片闭环
- 独立站与第三方小游戏平台均可走广告（实现上经 Adapter；现行可为 Mock）
- 广告加载失败：**跳过广告，不卡住游戏**

## 插屏广告

- 触发：对局 **胜利或失败结算后** 弹出

## 激励视频

| 入口 | 权益 |
|------|------|
| 碎片补齐 | 皮肤碎片不足时，观看后领取 **通用碎片** |
| 双倍碎片 | 观看后 **30 分钟内** 所有关卡碎片掉落翻倍 |

与碎片经济的配合见 [07-内容进度-碎片皮肤任务.md](./07-内容进度-碎片皮肤任务.md)。

## 存档（现行）

| 载体 | 存档 |
|------|------|
| 独立站 | 浏览器本地（碎片、皮肤、关卡进度） |
| 第三方小游戏平台 | 浏览器本地（同上） |

两端现行均无账号。跨设备云同步见 [`docs/任务/P-ACCOUNT.md`](../任务/P-ACCOUNT.md)。

## 与关卡的商业节奏（设计意图）

- 夏日：高压拉扯，广告停留相关
- 秋日：爽感刷碎片
- 冬日：高卡关，拉长尝试与广告曝光
- 春日回刷：长期碎片供给
