# 工程边界总览

本目录只保留跨模块、长期有效且无法仅靠单个文件表达的工程边界。函数、类型、字段、权重、环境变量、页面清单和构建参数均由代码、测试和仓库脚本维护。

## 长期边界

- 规则、合法行动、AI和关卡内容属于共享游戏核心；正式游戏、Admin和模拟不得各写一套规则。
- Web、H5门户和未来应用商店版本只在平台壳、存储、广告与支付适配层产生差异，不分叉主线规则和24关内容。
- 存档结构变化必须可迁移或安全回退；结算和奖励必须避免重复发放。
- Admin是本地内容与质量工具，不是线上CMS，不写玩家正式进度。
- 规则、AI、存档或渠道适配发生变化时，必须有对应自动化验证和失败兜底。
