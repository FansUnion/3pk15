# 09 · 广告与变现设计

设计规格（触发与权益），不含广告 SDK 接入细节。

## 总原则

- 前期：**无付费商城**，靠广告 + 碎片闭环  
- 独立站与第三方小游戏平台均可走广告；存档能力不同（见下）  
- 广告加载失败：**跳过广告，不卡住游戏**

## 插屏广告

- 触发：对局 **胜利或失败结算后** 弹出  

## 激励视频

| 入口 | 权益 |
|------|------|
| 碎片补齐 | 皮肤碎片不足时，观看后领取 **通用碎片** |
| 双倍碎片 | 观看后 **30 分钟内** 所有关卡碎片掉落翻倍 |

与碎片经济的配合见 [07-内容进度-碎片皮肤任务.md](./07-内容进度-碎片皮肤任务.md)。

## 平台存档差异（变现相关）

| 载体 | 存档 |
|------|------|
| 第三方小游戏平台 | 仅浏览器本地缓存（碎片、皮肤、关卡进度） |
| 自建独立站 | 可支持账号 + 云端同步（源稿列为二期） |

## 与关卡的商业节奏（设计意图）

- 夏日：高压拉扯，广告停留相关  
- 秋日：爽感刷碎片  
- 冬日：高卡关，拉长尝试与广告曝光  
- 春日回刷：长期碎片供给  

## 源稿二期（非本期必做规格）

- 去广告限时礼包（付费，0–1 不做）  
- 广告曝光/点击/转化数据上报  
- 预留付费商城接口
