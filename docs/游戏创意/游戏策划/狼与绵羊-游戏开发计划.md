# 狼与绵羊游戏开发计划

# 一、我的需求理解（先给你确认）

## 整体项目背景

游戏名称 / 品牌 / 域名 / 宣传文案 → [`../游戏品牌/01-产品定位与模式边界.md`](../游戏品牌/01-产品定位与模式边界.md)（唯一事实源）  
载体：网页游戏，两套部署方案

1. 自建独立官网（长期运营，支持账号、皮肤存档、广告变现）

2. 第三方小游戏平台（Poki/CrazyGames，无账号、仅本地缓存、纯广告）
核心玩法：改良三狼十五羊棋，人机闯关，玩家仅操控 3 只狼对战 AI 15 只绵羊，无双人、无操控羊模式。

## 核心游戏底层规则（已全部定稿）

1. 棋盘：6×6 交点棋盘，36 点位，单点岩石作为地形障碍；

2. 棋子固定：3 狼（玩家）\+15 羊（AI），固定开局坐标；

3. 行动规则：狼先手、横竖移动、隔空吃（狼—空—羊）、单回合连续多吃；羊只能前进 / 横走、不能后退、无吃子；

4. 胜负：吃掉≥8 只羊通关；三只狼全无路可走则失败；

5. 关卡体系：春夏秋冬四大章节，AI 仅章节间升级，同章节 AI 固定，靠岩石数量调节对局难度；

6. 创新核心：岩石梯度平衡系统、四季心流关卡、单回合连续多吃。

## 商业化 \& 留存系统（前期无付费商城，纯广告驱动）

1. 碎片系统：通关 / 重复刷关 / 任务 / 看激励广告获取碎片；

2. 皮肤系统：狼皮、成套配套羊皮、独立棋盘皮肤，碎片兑换；岩石无皮肤；

3. 广告类型：对局插屏广告、激励视频广告（补碎片、双倍碎片时长）；

4. 存档区分：第三方平台仅浏览器本地缓存；独立站支持账号云端存档皮肤、碎片、关卡进度。

## 交付要求

1. 输出两份内容：①完整功能清单（分核心底层、关卡、AI、皮肤碎片、广告、页面交互、后台）；②从零到一开发排期 \& 开发顺序（分阶段，MVP 最小可玩版本→完整上线版→迭代优化版）；

2. 模块拆分清晰，区分「必须做」「可选后期迭代」，前期 0\-1 优先砍掉非核心功能，快速出可演示版本；

3. 适配网页端开发逻辑，兼顾第三方平台上架规则与自建站拓展性。

# 二、完整功能清单（分 7 大模块，区分【一期必做 MVP】/【二期拓展】）

## 模块 1：棋盘 \& 棋子底层渲染引擎（核心底层，必做）

### 必做 MVP 功能

1. 6×6 正交交点棋盘渲染，坐标系统（行 1\-6、列 1\-6）

2. 棋盘自适应缩放（PC / 手机网页端兼容，正方形等比例不拉伸）

3. 基础图层渲染层级：棋盘线条 \< 棋子 \< 岩石

4. 固定开局点位自动生成：3 狼固定坐标、15 羊固定前三行坐标

5. 单点岩石渲染，支持 0/1/2/4/6/8 块岩石动态生成

6. 棋子点击交互：仅选中狼可操作，羊群、岩石不可点击

7. 移动点位高亮提示：选中狼后，合法可走格子高亮显示

8. 吃子路径高亮：满足跳跃吃羊条件时，路径高亮提示玩家

### 二期拓展功能

1. 棋盘动态主题纹理切换（四季皮肤贴图加载）

2. 棋子动画：移动动画、跳跃吃羊动画、吃掉羊消失动画

3. 对局特效：连续多吃闪光特效、通关胜利特效

## 模块 2：游戏逻辑核心判定系统（底层核心，必做）

### 必做 MVP 功能

1. 基础移动判定：狼 / 羊仅横竖相邻一格移动，禁止斜走

2. 障碍阻断判定：岩石、棋子阻挡行走、跳跃路径

3. 狼吃子判定逻辑：同线「狼—空—羊」隔空吃，狼落到羊位

4. 连续多吃循环判定：单次回合内无限连吃，无次数上限

5. 羊群移动限制判定：羊禁止向后（向棋盘顶部）移动

6. 胜负条件实时计算

    - 胜利：场上被吃掉绵羊数量≥8，立即结束对局

    - 失败：3 只狼全部无合法移动点位，判定失败

7. 回合顺序控制：狼先手，玩家操作完成后自动切换 AI 羊群回合

8. 单局重置功能、退出对局返回关卡列表功能

### 二期拓展功能

1. 对局步数统计、单局时长统计

2. 操作撤销（消耗广告解锁）

## 模块 3：AI 羊群智能系统（必做，分三档 AI）

### 必做 MVP 功能

1. 三档 AI 逻辑完整实现

    - 简易 AI（春日）：羊群随机移动，无主动合围逻辑

    - 标准 AI（夏日、秋日共用）：主动抱团、横向铺防线、封堵狼缺口

    - 大师 AI（冬日）：预判狼走位、全方位合围封锁

2. AI 自动寻路，每回合自动批量移动所有绵羊

3. AI 适配岩石地形：根据岩石分割路线调整防守策略

### 二期拓展功能

1. AI 难度微调滑块（自建站后期）

2. AI 对局行为日志，用于关卡平衡性调试

## 模块 4：四季关卡闯关系统（必做）

### 必做 MVP 功能

1. 四大章节分页：春日、夏日、秋日、冬日，按顺序解锁

2. 章节解锁限制：通关上一章全部关卡才能解锁下一章

3. 单关卡岩石配置自动加载（春日 0\-2 块、夏日 2\-4 块、秋日 6\-8 块、冬日 0 块）

4. 关卡进度存储：记录已通关关卡，可重复游玩任意解锁关卡

5. 单局通关奖励发放：基础碎片自动发放

6. 关卡列表 UI：展示通关 / 未通关状态、章节主题标识

### 二期拓展功能

1. 每日挑战限定关卡（专属碎片奖励）

2. 关卡星级评分系统（根据剩余羊数量打分）

## 模块 5：碎片 \& 皮肤收集留存系统（一期必做，无付费）

### 必做 MVP 功能

1. 碎片道具体系

    - 通用万能碎片（兑换所有稀有皮肤）

    - 四季专属碎片（兑换对应季节棋盘）

2. 碎片获取渠道逻辑

    - 首次通关关卡固定掉落碎片

    - 重复刷通关关卡概率掉落少量碎片

    - 每日 / 每周任务系统，完成任务领取碎片

3. 皮肤图鉴系统

    - 基础皮肤：通关章节直接解锁（四季棋盘、原生狼羊）

    - 稀有皮肤：消耗碎片合成解锁（霜狼等）

4. 皮肤穿戴逻辑

    - 更换狼皮肤，场上所有羊自动配套同步外观

    - 棋盘皮肤独立自由切换，与狼皮肤互不绑定

5. 本地缓存存储（适配第三方小游戏平台）：缓存碎片、已解锁皮肤、关卡进度

### 二期拓展功能（仅自建独立站可用）

1. 账号登录（谷歌快捷登录 / 邮箱登录）

2. 云端存档同步碎片、皮肤、关卡进度（跨设备互通）

3. 月度节日限定皮肤，限时碎片兑换

## 模块 6：广告商业化系统（网页端核心变现，必做）

### 必做 MVP 功能（全平台通用，无付费商城）

1. 插屏广告：对局胜利 / 失败后弹出

2. 激励视频广告两个核心入口

    - 碎片补齐：皮肤碎片不足，观看广告领取通用碎片

    - 双倍碎片 buff：观看广告，30 分钟内所有关卡碎片掉落翻倍

3. 广告加载、异常兜底逻辑（加载失败跳过广告，不卡住游戏）

### 二期拓展功能（自建站）

1. 去广告限时礼包（后期新增付费，本次 0\-1 不开发）

2. 广告数据统计曝光、点击、转化数据上报

## 模块 7：页面、任务、基础辅助功能

### 必做 MVP 功能

1. 首页主界面：章节选择入口、皮肤图鉴入口、每日任务入口

2. 对局内 UI：实时显示已吃掉绵羊数量、剩余绵羊数量

3. 每日任务、每周任务面板，展示任务条件与碎片奖励

4. 基础设置面板：音量开关、游戏操作说明弹窗

5. 新手引导弹窗（春日第一关弹出，讲解移动、吃子基础规则）

### 二期拓展功能

1. 数据统计面板（总对局数、总收集皮肤数量）

2. 分享对局截图功能

3. 多语言切换（海外网页平台适配英文等）

# 三、从零到一完整开发顺序（分 3 大阶段，阶段 1=MVP 最小可玩版本，可直接演示上架第三方平台）

## 阶段 1：底层核心 MVP 开发（优先完成，可产出可玩 Demo，无皮肤、无广告最简版）

目标：实现完整下棋对局逻辑、四季关卡基础 AI，能正常闯关胜利 / 失败，纯本地运行

1. 搭建网页基础框架（HTML\+Canvas / 游戏引擎，前端渲染环境）

2. 开发棋盘渲染、坐标系统、3 狼 15 羊固定开局点位

3. 完成基础行走、跳跃吃子、连续多吃判定核心逻辑

4. 实现胜负判定、回合切换、单局重置功能

5. 开发三档 AI 羊群逻辑：简易 AI、标准 AI、大师 AI

6. 搭建四季章节关卡框架，加载对应岩石数量配置

7. 关卡本地缓存、章节解锁逻辑

8. 极简新手文字引导、对局基础 UI（吃羊数量展示）

9. 测试闭环：完整走完春日→夏日→秋日→冬日全部对局，胜负逻辑无 BUG

## 阶段 2：留存 \& 商业化功能开发（MVP 完善，可上架第三方小游戏平台）

目标：加入碎片、皮肤、广告、任务系统，完整商业闭环，仅本地缓存，不做账号

1. 开发碎片数值体系、关卡通关碎片掉落逻辑

2. 搭建皮肤图鉴、碎片合成、皮肤穿戴渲染逻辑

3. 开发每日 / 每周任务系统，任务碎片奖励发放

4. 接入网页广告 SDK，实现插屏广告、激励视频（双倍碎片 / 补碎片）

5. 完善全部页面 UI：首页、图鉴、任务、设置弹窗

6. 本地缓存全量数据：关卡进度、碎片、解锁皮肤

7. 全流程联调：闯关→拿碎片→看广告攒碎片→兑换皮肤→对局更换皮肤

## 阶段 3：二期拓展迭代（自建独立站专属，第三方平台无需开发）

目标：搭建自有站点完整能力，长期运营拓展

1. 开发账号登录体系（谷歌 / 邮箱）、云端数据库

2. 云端存档同步碎片、皮肤、关卡进度

3. 新增皮肤动画、对局吃子特效、棋盘主题纹理

4. 多语言适配、数据统计后台（广告、对局数据）

5. 拓展限时节日活动、每日挑战关卡

6. 预留付费商城接口（后期如需付费皮肤可直接启用）

# 四、开发优先级精简总结

1. 最高优先级（先做，无此游戏无法运行）：棋盘渲染、棋子移动 / 吃子核心逻辑、AI 系统、四季关卡

2. 中优先级（商业留存必备，上架必须）：碎片、皮肤、广告、任务系统、本地缓存

3. 低优先级（后期迭代，自建站专属）：账号、云端存档、动画特效、多语言、活动系统
