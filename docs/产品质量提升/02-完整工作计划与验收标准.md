# 完整工作计划与验收标准

## 阶段一：基线审计

### 工作内容

- 盘点首页、玩法、章节、关卡、猎场、皮肤、设置、任务和棋局页面。
- 记录桌面端、移动端、中文、英文、首次访问和已有存档状态。
- 走通新玩家、回访玩家、皮肤玩家、静音玩家和异常恢复路径。
- 建立关键指标基线：首屏理解、首局完成、首次胜利、重试、继续游戏和页面退出。

### 验收

- 所有关键页面有路径图和问题清单。
- P0/P1 问题全部有负责人和修复方案。
- 关键入口无死路、无重复语义、无不可恢复状态。

## 阶段二：新手体验和首局

### 工作内容

- 设计短引导流程和可跳过提示。
- 重新检查春日第一关的教学目标、岩石配置、首回合和胜负节奏。
- 确认首次胜利、失败和重试后的反馈。
- 增加继续游戏、回到关卡列表和退出首页的清晰路径。

### 验收

- 新玩家不看外部说明也能完成第一步有效操作。
- 第一局能展示核心玩法，不依赖复杂规则解释。
- 首次失败后玩家知道失败原因和下一步行动。

## 阶段三：关卡质量和地图打磨

### 工作内容

- 为现有关卡建立设计说明和难度分层。
- 对狼、羊、岩石布局进行规则引擎模拟和人工复盘。
- 确定固定布局、受控变体和解锁节奏。
- 建立关卡发布前检查表和回归测试。

### 验收

- 每关都有明确教学点、策略目标和预期难度。
- 没有明显的必胜、必败或无意义布局。
- 关卡难度递进可解释，玩家不会因突然增加规则而流失。

## 阶段四：角色、棋盘、动画和音效体验

### 工作内容

- 执行狼、羊、岩石的缩小识别测试。
- 统一主题皮肤的角色、棋盘、状态颜色和预览规则。
- 完善选中、可走、吃子、连吃、AI、胜负和无效操作反馈。
- 加入静音、低动效和资源加载失败兜底。

### 验收

- 正常桌面观看距离和移动端小屏都能分辨角色。
- 玩家能看懂每次操作结果，不依赖声音才能理解。
- 动画不会阻塞正常操作，静音和低动效状态可靠。

## 阶段五：长期内容和奖励

### 工作内容

- 梳理章节、关卡、任务、碎片、皮肤和重复游玩的关系。
- 明确每套皮肤的主题故事、解锁目标、展示位置和获得反馈。
- 让奖励既有即时反馈，也能推动下一次打开游戏。

### 验收

- 玩家知道如何获得下一项内容。
- 奖励不会掩盖核心玩法，也不会造成不公平优势。
- 皮肤页、关卡页和棋局页之间形成完整回路。

## 阶段六：商业化试运行

### 工作内容

- 先接入激励广告抽象层，不直接把 SDK 逻辑散落在页面。
- 定义广告资格、冷却、每日上限、奖励确认和失败兜底。
- 设计撤回、双倍奖励和碎片奖励的非强制入口。
- 建立事件埋点和实验开关。

### 验收

- 无广告时游戏完全可玩。
- 广告失败不会吞奖励或破坏对局状态。
- 新手阶段不被强制打断。
- 能区分展示、完成、奖励发放和异常事件。

## 阶段七：发布前质量门槛

### 工作内容

- 运行核心规则测试、资源检查、生产构建和端到端冒烟。
- 检查所有主要视口、语言、刷新、浏览器返回和本地存档恢复。
- 进行小规模真实用户测试，收集困惑点和流失点。

### 发布门槛

- 无 P0 问题，无未解释的 P1 问题。
- 核心路径成功率达到预设目标。
- 关卡、资源、声音和商业化均有失败兜底。
- 所有新增能力都有回滚或关闭开关。
