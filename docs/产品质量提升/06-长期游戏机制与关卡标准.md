# Fangrush 长期游戏机制与关卡标准

## 一、这份标准要解决什么问题

当前最重要的不是马上改某一张地图，而是先确定一套可以支撑几十张甚至上百张地图的长期规则。否则每遇到一张关卡不平衡，就会反复修改胜利条件、初始位置、羊数量和岩石数量，最后导致：

- 老关卡互相不一致。
- 玩家无法形成稳定认知。
- 奖励和商业化难以估算。
- AI 难度无法比较。
- 新地图生产依赖个人经验，无法复用。

长期标准的原则是：**核心规则稳定，关卡参数可配置，变化集中在布局、目标和策略，而不是随意改变基础玩法。**

## 二、长期核心规则

### 默认规则保持不变

- 棋盘：6×6 交点棋盘。
- 狼：3 只。
- 羊：15 只。
- 默认胜利目标：狼吃掉 8 只羊。
- 羊胜利：三只狼都没有合法行动。
- 狼可以单步移动、隔空吃子和连吃。
- 单回合最多连续吃 5 次。
- 岩石阻挡落点和路线。

默认 `targetEaten = 8`，这是主线关卡的长期基准，不随意修改。未来特殊关卡可以覆盖目标，但不能改变默认认知。

### 为什么保留吃 8 只

- 玩家容易理解。
- 一局不需要清空全部羊，节奏更快。
- 适合用奖励、章节和挑战目标扩展。
- 现有关卡、测试和掉落经济可以保持稳定。

未来可以增加 `targetEaten` 关卡参数，但只用于特殊关卡：例如 6、8、10 或限定连吃目标。特殊目标必须在关卡开始前明确展示。

## 三、6×6 棋盘的长期坐标标准

### 坐标和区域

棋盘使用行列坐标 `r1-c1` 到 `r6-c6`。设计文档和后台工具统一使用该坐标，不使用“左上、右下”等容易因视角变化而产生歧义的描述。

将棋盘划分为四类区域：

- **狼方起始区**：默认在上方，承担主动进攻和第一次选择。
- **羊群起始区**：默认在上方中部与下方关键位置，形成可被分割但不会立即崩溃的群体。
- **中场策略区**：决定路线、隔空吃和岩石价值。
- **边缘逃生区**：提供羊的移动空间，也防止棋局第一回合就结束。

区域只是设计语言，实际位置仍必须通过规则检查和模拟验证。

## 四、狼的初始位置标准

### 默认原则

狼的初始位置不应每关完全重新发明，而应遵循稳定的三狼结构：

- 一只承担主动探路。
- 一只承担中场转换。
- 一只承担侧翼和补位。

三只狼不能完全重叠在同一条路线，也不能在开局被岩石或羊群同时锁死。

### 狼的位置变化

不同章节可以改变狼的空间压力，但不能破坏玩家对三狼的基本理解：

- 春季：三狼关系清晰，至少一只狼容易被玩家选中并完成教学动作。
- 夏季：狼需要分工，不能只依靠一只狼完成全部目标。
- 秋季：岩石改变狼的路线和转换成本。
- 冬季：减少岩石，强调围堵和位置计算。

狼的初始位置可以按关卡主题变化，但每次变化必须有设计意图，例如“侧翼教学”“中路突破”“分工合围”，不能只为了看起来不同。

## 五、羊的初始位置标准

### 默认原则

羊不应完全随机散落，也不应完全挤成不可处理的一团。推荐使用三种可复用结构：

1. **温和羊群**：便于新手理解隔空吃，适合春季前段。
2. **分组羊群**：两到三个小群体，要求玩家选择先处理哪一组。
3. **阻挡羊群**：羊会形成路线压力，适合夏季和秋季。

羊的布局需要同时保证：

- 玩家能在前几步找到有效目标。
- 羊有足够移动空间，不能每次都走同一种路线。
- 羊群不会因为一个错误就完全不可逆。
- 羊不会全部远离狼，导致玩家长时间找不到吃子机会。

## 六、岩石的长期标准

岩石不是装饰，而是改变路线和策略的地图机制。

- 春季：0、1、2 个，逐步教学阻挡。
- 夏季：2 至 4 个，开始形成中场压力。
- 秋季：6 至 8 个，形成通道、断点和路线选择。
- 冬季：0 个或极少，强调空盘围堵和高阶位置计算。

岩石设计必须说明它的作用：

- 引导第一次移动。
- 保护或暴露某条吃子路线。
- 切断羊群的一侧。
- 形成单通道。
- 防止简单直线速胜。

不能只因为地图空就增加岩石，也不能只因为难度不够就堆满岩石。

## 七、固定地图、受控变体和随机地图

### 主线关卡：固定地图

主线使用固定的狼、羊、岩石布局。这样便于：

- 教学和复盘。
- 关卡难度测试。
- 玩家形成策略记忆。
- 设计者解释每个位置的意义。

### 重复游玩：受控变体

玩家完成主线后，可以从候选布局池中选择变体。变体可以改变部分羊的位置或一个非核心岩石，但必须：

- 通过合法性校验。
- 通过难度区间校验。
- 保留至少一个有效开局策略。
- 不改变主线教学目标。

### 高级模式：随机或每日地图

完全随机只用于高级模式、每日挑战或实验功能。随机地图必须经过生成器筛选，不能把未验证局面直接交给玩家。

## 八、长期难度曲线

难度不应只依赖 AI 档位。推荐使用五个维度：

1. 规则数量：玩家一次需要理解多少新规则。
2. 路线复杂度：可行路线有多少，是否需要提前规划。
3. 羊群压力：羊是否会形成阻挡和逃生路线。
4. 岩石密度：岩石是否切断路线或形成单通道。
5. 目标压力：吃子数量、连吃要求、回合限制和特殊目标。

每个章节只提高其中一到两个维度，避免同时增加 AI、岩石、目标和地图复杂度。

建议曲线：

- 春季：规则学习优先，AI 低压，目标固定。
- 夏季：羊群阻挡增加，岩石开始影响路线。
- 秋季：路线复杂、岩石密度高，要求分工和预判。
- 冬季：规则不再新增，主要提高位置计算和 AI 强度。

## 九、每张地图的标准参数

以后每个关卡必须具备以下长期参数：

```ts
type LevelDesignSpec = {
  id: string
  chapterId: 'spring' | 'summer' | 'autumn' | 'winter'
  indexInChapter: number
  rocks: Pos[]
  wolfOpening: string
  sheepOpening: string
  teachingPoint: string
  targetEaten: number // 默认 8
  maxPlies?: number
  ai: 'easy' | 'normal' | 'hard'
  expectedPlies: { min: number; target: number; max: number }
  difficulty: 1 | 2 | 3 | 4 | 5
  firstClearReward: RewardSpec
  repeatDrop?: RewardSpec
}
```

其中：

- `wolfOpening`：狼方开局结构和第一步意图。
- `sheepOpening`：羊群结构和逃生压力。
- `teachingPoint`：本关只教一个主要概念。
- `targetEaten`：默认 8，特殊关卡才覆盖。
- `maxPlies`：只在经过测试后使用，防止无休止对局。
- `expectedPlies`：用于发现过快、过慢和僵持关卡。

## 十、地图发布标准

每个关卡必须经过：

1. 设计意图评审。
2. 至少三个候选布局比较。
3. 规则合法性检查。
4. 随机狼、贪心狼、混合狼模拟。
5. easy、normal、hard 羊 AI 模拟。
6. 超长局和重复局面检查。
7. 新手人工测试。
8. 熟练玩家人工测试。
9. 奖励和下一关入口检查。
10. 生产构建和回归测试。

不能只用一个胜率数字决定地图是否合格。必须同时看：

- 首回合是否有意义。
- 玩家是否能发现目标。
- 最短和最长对局。
- 是否存在重复走位。
- 失败是否能解释。
- 玩家是否愿意重试。

## 十一、当前项目应采用的长期决定

1. 默认 `targetEaten` 继续为 8。
2. 主线地图采用固定布局，不采用完全随机开局。
3. 未来使用受控变体和每日挑战增加重复游玩。
4. 6×6 棋盘继续保留，不通过扩大棋盘解决深度问题。
5. 春季保持低压力，但每关增加一个清晰的新认知点。
6. 狼和羊的位置每关可以不同，但必须属于可解释的开局模板。
7. 岩石按章节递进，不能随意堆叠。
8. 先解决重复走位和超长对局，再扩充大量关卡。
9. 特殊胜利目标通过关卡参数扩展，不能修改全局默认规则。
10. 所有长期经验都以本标准作为后续地图开发的共同依据。

## 十二、当前下一步

下一步不是立即修改春日第一关，而是：

1. 为现有春日三关补齐 `wolfOpening`、`sheepOpening`、`teachingPoint` 和 `expectedPlies` 设计数据。
2. 确认 AI 长局的正式处理方式：最大回合、重复局面或两者结合。
3. 用标准模板重新复盘春日三关。
4. 再决定是否调整春日第一关的羊群位置、岩石或 AI。
5. 通过后把同一模板应用到夏季第一关。
# 关卡级平衡原则：季节、地图与 AI 的协同

## 核心结论

长期不把羊 AI 难度固定为“每季一个档位”。季节只定义整体难度方向和范围，最终平衡由关卡级配置完成。

每张地图综合使用以下变量：

- 狼和羊的初始位置：决定开局主动权和第一轮战术。
- 岩石数量与位置：决定路线、跳跃点、封锁和重复局风险。
- 羊 AI 难度：决定羊群的防守、逃跑和阻挡能力。
- `targetEaten`：当前主线统一保持 8，避免破坏核心胜利规则。
- `maxPlies` 与重复判和：防止棋局无限拖延。

## 参数层级

| 层级 | 职责 |
| --- | --- |
| 季节 | 定义整体难度曲线、主题和允许的参数范围 |
| 关卡 | 定义具体教学目标、地图结构和目标胜率 |
| 地图布局 | 完成主要平衡，决定狼方或羊方的空间优势 |
| 羊 AI | 进行最后的小幅校准，不替代地图设计 |
| 玩家数据 | 验证模拟结果，决定是否需要再次调整 |

## 调整顺序

发现某关胜率异常时，必须按以下顺序处理：

1. 检查狼、羊初始位置。
2. 检查岩石数量、位置和是否形成封闭通道。
3. 检查是否存在狼必胜、羊必胜、死局或重复循环。
4. 最后才调整羊 AI 难度。

不能用提高 AI 难度掩盖地图设计问题，也不能只靠减少岩石解决所有难度问题。

## 四季应用

- 春季整体友好，后半段逐步增加路线判断；允许个别关卡使用更高 AI，但不能突然跳难。
- 夏季开始要求羊群阻挡和路线规划，大部分使用 `normal`，局部过于偏狼方的关卡可以使用 `hard`。
- 秋季主要通过密集岩石、连吃顺序和路线选择制造挑战，AI 不应成为主要难度来源。
- 冬季作为综合挑战，整体可以使用 `hard`，但仍需保证棋局可结束、可复盘、不是随机必败。

## 验收目标

主线目标是季节难度逐步上升，而不是每张地图使用相同 AI：

- 春季普通玩家狼方胜率约 65% - 80%。
- 夏季普通玩家狼方胜率约 50% - 65%。
- 秋季普通玩家狼方胜率约 35% - 55%。
- 冬季普通玩家狼方胜率约 25% - 45%。
- 单张地图和棋率原则上低于 10%。

AI 模拟只是风险筛查，最终需要结合真实玩家数据判断地图是否真正公平。
