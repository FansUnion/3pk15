# Fangrush 文档中心

本文是项目文档的总入口。文档按用途分层，不按创建时间或篇数判断价值。

## 核心阅读顺序

1. [游戏创意](游戏创意/)：原始规则和产品设想的来源，只做小范围补充、勘误和格式整理，不以普通重构改写原意。
2. [产品核心](产品核心/README.md)：现行产品规则、质量标准、24关设计和玩家教学的唯一产品依据。
3. [任务清单](任务/任务清单.md)：现在做什么；[需求池](任务/需求池.md)：已提出但暂不做什么。
4. [商业成功](商业成功/00-索引.md)：商业指标、渠道和变现决策。
5. [产品扩展](产品扩展/README.md)：24关主线之后的关卡、赛季和挑战玩法。
6. [发行资料](../distribution/README.md)：公共品牌资料和各平台最终提交物。

## 目录职责

| 目录 | 职责 | 维护方式 |
|---|---|---|
| `游戏创意` | 创意源、原始规则与关键设想 | 受保护；不因现行实现不同而删除原始想法 |
| `产品核心` | 当前有效的产品契约和24关内容资产 | 高质量持续维护；同一结论只保留一处 |
| `任务` | 近期执行、暂缓需求和极少量完成证据 | 完成后从活动清单移除；无长期价值的完成记录可删除 |
| `商业成功` | 渠道选择、上架路径、变现原则和商业验证 | 政策与平台事实执行前重新核验 |
| `产品扩展` | 主线之后的未来游戏内容 | 只保留长期内容方向，不记录当前任务状态 |
| `综合文档` | 品牌、外部资源交付和独立站SEO | 只保留跨产品与外部协作需要的规格 |
| `历史参考` | 过去诊断和原始证据 | 默认不维护；有效结论吸收后可精简或删除原文 |
| `distribution` | Poki、CrazyGames及未来App商店的提交材料 | 自包含表单、构建、合规和交付物，不复制运行时逻辑 |

## 唯一来源规则

- 产品为什么这样设计、必须达到什么体验：产品核心。
- 关卡名称、文案、开局、策略、难度和验证结论：24关设计卡；运行配置以代码为准。
- 规则、AI、数据结构、路由、存档字段和脚本参数的实现细节：代码与自动化测试。
- 当前任务状态、优先级和依赖：任务清单。
- 暂不执行的功能需求：需求池。
- 渠道顺序和商业边界：商业成功。
- 平台封面、视频、说明、法律资料和提交清单：`distribution`。

产品文档不复制类型定义、函数流程、完整配置或代码路径清单。只有会影响玩家体验、验收或跨模块边界的技术约束，才以总体原则写入文档。

## 冲突处理

发现冲突时，先确认事实类型，再修改它的唯一来源；其他文档改为链接，不保留第二套结论。代码实现与产品契约不一致时，不自动认为代码或文档正确，应建立任务并明确由产品决策还是实现修复。
