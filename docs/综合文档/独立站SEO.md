# Fangrush 独立站 SEO 方案

> 本文只维护独立站的搜索定位、页面职责、索引边界和验收方法。品牌定位与渠道文案以 `普通文档ing/美术等资源/01-品牌文案和游戏上架材料.md` 为准；玩家实际显示文本以 i18n 代码为准；24 关名称和说明以关卡配置为准。

## 一、目标

SEO 首先帮助合适的玩家理解并进入游戏，不以关键词数量作为成功标准。独立站稳定承接四类意图：

1. 搜索 Fangrush 或三狼连猎的品牌用户。
2. 寻找浏览器策略棋、回合制棋盘游戏的品类用户。
3. 查询玩法、隔空捕食、连续捕食和胜负规则的学习用户。
4. 查询具体季节或关卡攻略的回访用户。

## 二、统一定位

- 英文：Fangrush is a compact strategy board game where three wolves face fifteen AI-controlled sheep on a 6x6 point grid.
- 中文：《三狼连猎》（Fangrush）是一款轻量策略棋，玩家操控三狼，在 6x6 交点棋盘上对阵十五只 AI 羊。
- 核心机制：狼跨过一个空点落到羊的位置完成捕食；单回合最多连续捕食五次。
- 内容规模：四季 24 关，岩石为关卡预设地形，玩家不能摆放岩石。
- 独立站边界：浏览器直接游玩、本地保存、无需账号、默认无广告。

英文面向玩家优先使用 `leap-capture` 或完整动作句，不把 `gap-rush`、函数名和测试术语作为理解前提。

## 三、页面职责与索引

| 页面类型 | 回答的问题 | 索引策略 | 内容来源 |
|---|---|---|---|
| 首页 | 这是什么，为什么值得玩 | index | 品牌基准 + i18n |
| 四季/章节 | 有哪些挑战，难度如何递进 | index | 章节与关卡配置 |
| 关卡介绍 `/hunt/*` | 本关特点、目标和教学点 | index | 关卡配置 |
| 实际对局 `/play/*` | 我现在怎样行动 | noindex, nofollow | 游戏状态 + i18n |
| 帮助中心 | 规则怎样理解 | index | 核心规则 + 可验证图例 |
| 皮肤/任务 | 有哪些收藏和进度 | index 价值较低，以转化为主 | 皮肤与奖励配置 |
| 设置/隐私 | 偏好和数据边界 | 隐私页 index；设置页不承担搜索落地 | 运行时配置 |
| Admin | 内部验收 | 禁止索引 | Admin 应用 |

## 四、标题与摘要模板

- 首页英文：`Fangrush | A 3-wolf strategy board game`
- 首页中文：`三狼连猎 · Fangrush`
- 章节：`Spring Hunts | Fangrush` / `春日猎场 · Fangrush`
- 关卡：`Spring 01 · First Hunt Meadow | Fangrush` / `春日 01 · 初猎之野 · Fangrush`
- 帮助：`How to play Fangrush | Rules and strategy` / `三狼连猎怎么玩 · 规则与策略`

标题必须简洁、唯一、与页面可见内容一致。摘要用一到两句说明页面价值，不写关键词列表，不在中文标题中堆入完整英文品类句。

## 五、关卡内容策略

24 关采用“双层表达”：

- 主名称提供季节感和记忆点。
- 关卡说明、教学点和策略标签解释真实机制。
- 稳定内部 ID（如 `spring-01`）负责存档、测试和链接，不参与营销表达。

关卡介绍页包含名称、简短挑战说明、目标、主要教学点和相邻关卡。实际对局页保持专注，不重复长篇 SEO 内容。

## 六、关键词使用原则

优先自然覆盖：

- 品牌：Fangrush、三狼连猎。
- 品类：strategy board game、browser strategy game、策略棋盘游戏。
- 机制：three wolves vs fifteen sheep、chain capture、隔空捕食、连续捕食。
- 使用条件：play in browser、no download、no account、浏览器直接玩、无需账号。

长尾内容只在确有对应页面时使用，例如 `how to play Fangrush`、`Fangrush rules`、`三狼连猎怎么玩`。不为不存在的攻略、多人模式、账号或摆岩石能力创建搜索承诺。

## 七、技术实现边界

- sitemap 与 llms-full URL 共享同一代码来源。
- `/play/*` 保持 noindex；`/hunt/*` 承担可索引关卡介绍。
- 每个索引页面提供 canonical、语言 alternate、独特 title 和 description。
- 首页与帮助页的结构化数据必须与可见内容一致。
- 生产环境 `NEXT_PUBLIC_SITE_URL` 使用 `https://fangrush.com`。
- 文案、关卡名称或路由变化后运行构建与 URL 门禁。

## 八、验收

- 搜索结果标题没有关键词堆叠或中英重复句。
- 首页十秒内能看懂“三狼、十五羊、6x6、隔空捕食、四季 24 关”。
- 关卡标题均唯一，名称与真实地图、开局和教学点一致。
- sitemap 不包含 Admin 和实际对局页。
- 页面不宣传尚不存在的账号、云存档、多人、摆放岩石或平台 SDK。
- 中文与英文表达同一规则，但各自自然，不逐字硬译。

## 九、维护流程

1. 定位变化先修改品牌基准。
2. 运行时文案同步到 i18n。
3. 关卡名称和说明只在关卡配置修改，再同步每关设计资料。
4. SEO 文档只在页面职责、索引策略或模板变化时更新。
5. 发布前检查重复标题、缺失翻译、错误玩法承诺、sitemap 和 robots。
