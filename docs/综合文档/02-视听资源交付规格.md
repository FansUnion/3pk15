# Fangrush 视听资源交付规格

> 本文面向美术、动画和音频制作者，说明交付物应达到什么要求。场景含义和最终验收见产品核心的视听资源标准；实际库存与路径由资源Catalog和检查脚本维护。

## 一、整体气质

- 牧场插画风、自然、清楚、略带策略张力。
- 狼有攻击性但不血腥；羊可爱但不是无表情白团。
- 棋盘像真实游戏场，不像表格或后台工具。
- 四季通过材质、色温和环境细节区分，不能只换背景颜色。
- 避免赛博紫、霓虹、过度写实、廉价按钮质感和大量装饰噪声。

## 二、角色与地形

| 资产 | 必须 | 禁止 |
|---|---|---|
| 狼 | 耳、吻、眼睛和身体轮廓清楚；实际棋盘尺寸仍像狼 | 圆头两点眼、与羊同明度、装饰遮住脸 |
| 羊 | 羊毛体块、脸和四肢可辨；与同套狼笔触一致 | 纯白圆饼、像花或云、缩小后没有脸 |
| 岩石 | 不规则多面体、明暗面和裂纹；儿童不误认成盾牌或按钮 | 圆角棕方块、金属盾牌轮廓 |
| 棋盘 | 交点和线路清楚，角色对比充分，主题不抢棋子 | Excel网格、四季同色、背景纹理盖住落点 |

角色源稿优先使用透明背景、稳定viewBox和足够留白。交付时同时提供大图预览与实际棋盘尺寸截图，不能只在放大状态验收。

## 三、动画

优先用代码完成位移、缩放、淡出、高亮和轨迹；只有轮廓或材质真的需要变化时才增加状态帧。

需要覆盖的主要事件：选择、狼移动、羊移动、捕食、连续捕食、威胁、单狼受困、胜负、奖励和装备。每段动画必须说明事件、时长、循环方式、透明背景和减少动态效果时的替代方案。

禁止用长转场遮挡棋盘、用动画替代规则信息，或让反馈结束前的输入状态不明确。

## 四、声音

- 正式短音效使用可构建、可解码的静态样本；优先WAV。
- 移动轻且短，捕食更清晰，连吃有同族递进，胜负短促收束。
- 狼移动、羊移动、捕食、连吃、威胁、受困和终局必须能听出类别差异。
- 手机扬声器不过小，耳机不过响；多个连续事件不能互相覆盖。
- BGM和环境音不得掩盖棋盘反馈，并支持平台暂停、静音和后台恢复。
- 每项资源必须保留来源、授权、可商用范围和必要署名；“网上可下载”不等于可商用。

## 五、交付与验收

每批资源至少提供：

1. 源文件和最终导出文件；
2. 尺寸、格式、循环/时长和命名说明；
3. 授权来源与商业使用证明；
4. 默认、浅色、深色或对应季节棋盘上的实装截图；
5. 手机实际尺寸和桌面尺寸检查；
6. 新旧版本对比及解决的问题。

技术门禁通过只表示文件可加载。角色是否好看、声音是否悦耳、皮肤是否值得收藏仍需要真实设备和人工审美确认。
