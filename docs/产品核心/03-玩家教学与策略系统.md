# 玩家教学与策略系统

## 1. 目标

玩家应在不离开核心体验的前提下理解规则、解决当前困惑，并逐步学会狼群协作。教学不替玩家完成关卡，也不为坏地图或AI缺陷兜底。

## 2. 三层关系

| 层级 | 解决的问题 | 表现 |
|---|---|---|
| 帮助中心 | 游戏规则是什么 | 可随时查阅的规则、图例和基础策略 |
| 局内指导 | 当前局面该观察什么 | 观察、目标、策略三级主动提示 |
| 策略知识库 | 为什么这样行动 | 统一策略名称、盘面信号、常见错误和适用关卡 |

帮助中心讲稳定规则；局内指导提供当前关卡线索；策略知识库建立可复用的思考语言。

## 3. 帮助中心内容

- **第一层：一分钟开局**。玩家目标、第一步操作、普通移动和隔空吃；进入第一关前不要求先读完全部规则。
- **第二层：完整规则**。胜负、回合顺序、连续捕食与主动结束、羊移动方向与无行动跳过、岩石阻挡、重复和行动上限。
- **第三层：看懂界面**。行动方、捕获目标、剩羊、绿色落点、红圈目标、连吃决策和声音/静音。
- **第四层：基础策略**。三狼分工、路线控制、保持机动和停止连吃；不在规则页展开逐关答案。
- **第五层：进度与辅助系统**。本地存档、关卡解锁、碎片、装扮、任务、广告可选性、分享与反馈。

玩家还需要逐步学会读懂羊方意图：哪只狼正在被围、哪条接应线正在关闭、羊是在止损还是准备收网。教学只解释已经能从盘面观察到的信号，不泄露AI精确下一步，也不把内部评分包装成规则。

帮助文案必须与正式规则一致；规则变化时同步更新并通过内容检查。

图例必须遵守以下标准：

1. 使用与正式棋盘一致的狼、羊、岩石和高亮资源；简化棋盘可以缩小，但不能声称CSS占位图与实战完全相同。
2. 动作规则使用“操作前 → 操作后”两帧或短动画，明确狼最终落点、羊是否移除和回合是否继续；只画一个红圈不能完整解释捕获结果。
3. 首发至少覆盖普通移动、隔空吃、继续/结束连吃、羊移动方向、岩石阻挡和三狼被困；重复预警可用界面截图说明。
4. 每张图只解释一个问题，配一句动作结论；不使用需要玩家自行解码的 ASCII 字符图作为主图例。
5. 图例从真实规则状态或共享渲染组件生成，并有合法行动测试，避免文档图与游戏行为分叉。
6. 图片数量服从认知负担。推荐4组核心图例而不是堆叠大量截图：移动与捕获、连吃决策、羊与岩石、终局与循环。

玩家语言优先说明“现在做什么、为什么、下一步是什么”，不直接暴露 `plies`、`mixed`、函数名、搜索预算和内部终局代码。自定义术语首次出现时必须配图或动作示例；中英文分别重写和验收，不做逐字互译。

## 4. 局内指导

- `观察`：指出当前最重要的盘面信号，不给落点答案。
- `目标`：说明本关战术目的和羊方当前主动意图。
- `策略`：给出与本关主策略一致的行动原则。
- 连续失败后可以提醒玩家主动打开提示，但不自动演示完整路线。

若三级提示仍无法让多数玩家理解关卡，应先检查地图、教学顺序和提示文案，再考虑高级路线演示。

## 5. 狼方策略知识库

| 标签 | 含义 | 典型盘面信号 | 常见错误 |
|---|---|---|---|
| 双狼夹击 | 两狼从不同方向压缩羊群空间 | 羊群集中且两侧有入口 | 两狼挤在同一路线 |
| 三狼包围 | 两狼限制路线，第三狼捕食 | 需要同时封住两条逃路 | 三狼都追最近的羊 |
| 诱导移动 | 先走非捕食路线，逼羊进入目标区 | 立即吃子会失去后续路线 | 只看当前可吃目标 |
| 岩石控道 | 利用岩石边界和空隙塑造通道；岩石不可落脚，也不是跳跃中点 | 岩石改变落点或换线成本 | 把岩石当成落点，或堵死唯一退路 |
| 链式捕捉 | 首次捕食后评估继续连吃 | 新的捕食线出现且仍有退路 | 有连吃就盲目继续 |
| 保持机动性 | 保留狼的转向、支援和撤退空间 | 狼靠近边线或狭窄区域 | 三狼同时进入死角 |

每张主线关卡至少绑定一个主要策略和一个辅助策略。帮助内容、关卡提示、设计卡和验收问题使用同一组标签。

## 6. 验收

- 新玩家能区分移动、捕食、连吃和结束回合。
- 提示从观察到策略逐层增加信息，不直接给出必胜落点。
- 同一策略在帮助、局内提示和关卡设计卡中的含义一致。
- 主要策略可以在真实棋谱中观察到，不只是文档标签。
