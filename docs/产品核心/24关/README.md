# 四季 24 关设计卡总表

> 目标：每个季节 6 张可用地图；每张地图都保留可复现配置、平衡数据、狼方计划、羊方防守、操作难度和棋谱证据。
>
> 当前总判断：`sheep-ai-v2` 固定种子审计为 `15 pass / 9 review / 0 reject`，地图新鲜度和动作质量门禁通过；9 个 review 留给长尾或首次接触路线复盘。自动结果允许进入玩家抽查，不代表人工验收完成。

## 状态定义

- `pass`：未命中对应版本自动门禁红线，可进入玩家试玩，但不代表已验收；必须同时注明代理和证据版本。
- `review`：存在需要解释的诊断差异。当前 9 关命中 `LONG_TAIL` 或 `FIRST_CAPTURE_RISK`，必须结合棋谱与人工试玩判断，但没有自动 reject。
- `reject`：命中明确拒绝规则，当前不得作为合格生产关卡，必须修订并复验。
- `用户试玩确认`：用户完成实际操作验证，确认难度、策略可理解性和体验符合预期。

### 风险标签如何解释

- `LONG_TAIL`：mixed 代理的 P95 达到行动上限 75%，需要排查有效防守、代理往返或地图拖延。
- `FIRST_CAPTURE_RISK`：mixed 建立首次捕食的覆盖不足 80%；若 chain-aware 仍能建立路线，先人工确认路线可读性，不直接改图。
- `winter-04` 原长尾已在 V2 关闭，当前转为首次接触路线复盘。

## 证据与当前动作

- 运行时唯一来源是 `packages/game-core/src/content/levels.ts` 和 `strategies.ts`：名称、前台文案、坐标、开局、AI、捕食目标、行动上限、难度与策略绑定均先改代码。Admin 和玩家端只消费这些配置。
- 单关设计卡负责名称含义、设计理念、玩家体验、策略解释、风险解释和人工反馈。卡片保留名称、前台文案和运行摘要作为可读投影，但不得反向定义规则；`level-doc-contract.test.ts` 保证关键投影与代码一致。
- 本总表是逐关当前标签与玩家验收进度的唯一来源；单关卡不另建一套状态矩阵。
- 日期型验证记录是固定种子、策略、样本、棋谱与历史结论的唯一来源；单关卡不复制会随运行过期的胜率。
- 自动数据只用于风险筛选，不解释为真实玩家胜率；旧数据必须标注“捕食后立即收手”，不得继续作为完整狼策略证据。
- 旧同构组已拆解；生产门禁现在禁止任何非空岩石布局与另一关相同、镜像或 180° 旋转等价。
- 当前统一动作是先人工复盘告警关，再按春、夏、秋、冬学习顺序试玩，不再为追求全部自动 `pass` 破坏合理策略差异。
- 当前任务、负责人和发布顺序只在 [任务清单](../../任务/任务清单.md) 维护。
