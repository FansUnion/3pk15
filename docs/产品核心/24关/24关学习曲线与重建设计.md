# 24关学习曲线与重建设计

> 本文是当前 24 关重建专项的设计依据。长期方法见[关卡与平衡设计标准](../02-关卡与平衡设计标准.md)，逐关最终资料仍维护在各关设计卡。

## 一、专项目标

不再以“修补现有 24 张地图直到自动门禁通过”为目标，而是重新建立一套玩家愿意学习和持续挑战的主线：

- 学习顺序清晰；
- 每关有唯一主要问题；
- 相邻关的视觉和策略可以用一句玩家语言区分；
- 春季前段允许宽容，后续羊方不明显送子；
- 高难关给狼留下可理解的取胜空间；
- 岩石服务路线和教学，不承担全部难度；
- 胜率、策略增益、失败归因和新鲜度共同决定放行。

## 二、四季体验职责

| 季节 | 玩家阶段 | 羊方表现 | 地图结构目标 |
|---|---|---|---|
| 春 | 学规则、首次捕食、基础协作 | 前两关宽容，后四关逐步减少送子 | 开阔和少石为主，视觉简单 |
| 夏 | 形成策略、管理退路 | 换线、封锁落点、诱导困狼 | 分流、侧翼、双通道和一个代表漏斗 |
| 秋 | 地形计算、连吃与收手 | 熟练利用关键点，但不靠密岩自动封锁 | 少石与密岩混合，只保留少量高密度题 |
| 冬 | 综合考试、开放盘和高压防守 | 接近最强公平防守，主动分散破坏覆盖 | 开放盘为主，允许 1–2 张少石变化，避免六张空盘同质化 |

## 三、24关AI难度曲线

### 1. 科学判断

玩家水平会随关卡提升，但AI难度不能通过“前期故意送、后期突然最优”制造。24关共享同一智力底线：有明确安全替代时不选严格劣着。难度增长来自五个可解释变量：

1. 安全候选从较宽的安全集合逐步收紧到接近最优；
2. 计划从即时反应逐步扩展到完整狼回合和多回合合围；
3. 防守从保羊逐步增加堵落点、围孤狼、地形利用和合理牺牲；
4. 高难档逐步减少随机并列选择，提高确定性和优势转化；
5. 地图、开局和AI不应在同一关同时大幅跳升，每两关采用“引入—练习”，季末采用综合考试。

### 2. 十二个两关阶段

这些是体验档位，不要求实现十二套独立算法；代码可以由少量稳定AI配置组合搜索预算、候选宽度和主动防守权重实现。

推荐使用五个生产画像承接十二阶段：`guided`（春01/02）、`foundation`（春03–06）、`tactical`（夏01–06）、`strategic`（秋01–06）、`expert`（冬01–06）。画像是关卡配置，不是玩家可选模式；同一画像内再由地图、开局和每两关教学职责形成细微递进。现有 `easy/normal/hard` 可以保留为底层算法或迁移兼容字段，但不能继续作为完整学习曲线的唯一表达。

| 关卡 | 玩家成长阶段 | 羊方目标行为 | 不允许出现 |
|---|---|---|---|
| spring-01/02 | 规则与第一次捕食 | 基础合法、安全候选中保留宽容和变化 | 严格劣着、连续送子、突然合围新手 |
| spring-03/04 | 短连吃与诱导 | 识别立即跳吃，开始堵一次落点 | 玩家尚未学会前就执行多步强封锁 |
| spring-05/06 | 双狼协作与春季综合 | 稳定避吃、限制单狼机动 | 仍靠随机送子让玩家通过 |
| summer-01/02 | 路线和退路 | 利用岩石与通道挡线，预判完整狼回合 | 只保羊、不看狼下一轮连吃 |
| summer-03/04 | 三狼分工与换线 | 主动针对孤立狼，持续一到两步封锁 | 每步更换目标、无意义来回 |
| summer-05/06 | 战术综合 | 识别退路、关键落点和短期牺牲 | Hard名称存在但行为与Normal无差异 |
| autumn-01/02 | 窄门与地形控制 | 用关键点压缩入口，形成区域防守 | 仅靠岩石天然封死而AI无决策 |
| autumn-03/04 | 长链风险与跨区 | 评估最大捕食链、跳吃终点和换区 | 只避开第一吃却送出后续长链 |
| autumn-05/06 | 时机与战略综合 | 为困狼接受合理牺牲，优势局主动推进 | 永远保子或只拖到回合耗尽 |
| winter-01/02 | 高压站位 | 多回合维持目标，主动切断狼间接应 | 可避免的明显送子和低级换位 |
| winter-03/04 | 专家防守 | 针对狼方固定套路，合围与反循环并重 | 连续可避免送子、优势下晃动、异常长尾 |
| winter-05/06 | 最强公平综合 | 接近本期生产AI上限，稳定兑现困狼机会 | 隐藏作弊、读取未来操作、靠纯拖延制造困难 |

### 3. 玩家体验目标

- 春季：学习玩家通常1–2次尝试可以理解并通过；失败主要用于教规则和基础策略。
- 夏季：通常1–3次尝试，必须开始组合两狼或三狼，而不是沿用单狼追羊。
- 秋季：通常2–4次尝试，失败应能归因到落点、连吃终点、地形或收手时机。
- 冬季：通常2–5次尝试；资深玩家不应无思考连续一次通过六关，但正确策略最终应有稳定胜路。

以上是上线前校准目标，不是假装已经获得的玩家数据。上线后按到达该季节的玩家分层统计首次通过、三次内通过、重试和流失，再调整目标区间。

### 4. 自动与人工门槛

- 全24关 Normal/Hard `chosenDominated=0`。
- 春季可以宽容，但不能用严格劣着制造教学。
- 夏季起必须预判完整狼回合；秋季起必须评估捕食链和地形控制；冬季必须具备多回合目标与优势转化。
- 相邻两关至少在地图问题或羊方防守方式上有清楚差异。
- 自动代理只能证明风险和胜路；季节尝试次数、压力和乐趣必须由真实玩家验证。

## 五、重建优先级

### 第一批：解决结构性重复和强风险

1. `winter-02`：从“当前空盘微调”改为开放非对称分散题；
2. `autumn-01`：减少岩石，建立一个清晰窄门；
3. `autumn-03`：重做长连吃窗口，避免稳定强路线；
4. `summer-06`：脱离 `summer-03` 漏斗结构；
5. `winter-06`：脱离 `winter-05` 镜像/近似布局。

### 第二批：拆解重复地图组

- `autumn-02/04/05`：只保留一到两张密岩题，其余改为少石、单门、双门等不同家族；
- `spring-06`：与 `spring-05` 拉开终关差异；
- `winter-04/05`：补充开放盘之外的视觉和策略变化。

### 第三批：保留关的可读性优化

对保留关检查教学文案、第一次有效决策、失败原因和前后衔接，不为追求数值整齐而强行改图。

## 六、快速生产与淘汰流程

每个重做关先生成 3 个明显不同的候选，不做十几个坐标微调。候选先过快速门禁：

- 结构合法；
- 与现有地图家族不重复；
- random、mixed、chain-aware 各 5 个固定种子；
- 线上同档羊 AI；
- 首次捕食、胜负、平局、连吃敏感和代表棋谱；
- 玩家语言可以说明该关与前后关的差异。

明显重复、无首吃、羊明显送子、稳定机械强胜或失败无法解释的候选立即淘汰。每关只保留 1–2 个候选进入更多种子、有限求解和人工试玩。
