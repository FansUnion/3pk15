# Fangrush 核心产品定义

> 本文回答“这是什么产品、主线为什么这样设计、首发必须达到什么标准”。具体规则实现以 `game-core` 为准，当前进度以 [任务清单](../任务/任务清单.md) 为准。

## 1. 产品承诺

Fangrush 是一款玩家操控三只狼、与羊群进行策略博弈的6x6棋盘闯关游戏。玩家应能快速理解目标，通过狼群协作、路线控制和连续捕食战胜羊群，并在失败后知道如何改进。

质量目标按顺序为：

1. 对局可信：规则正确、AI可理解、胜负可复盘。
2. 主线可信：四季24关难度递进明确，不靠随机坏局、无解局或无意义和棋制造难度。
3. 感知清晰：狼、羊、岩石、合法行动、捕食、连吃、胜负和下一步易于辨认。
4. 循环完整：首局学习、通关、奖励、重试和章节推进自然连接。
5. 商业化克制：广告、提示和付费不替代策略，也不阻断基础体验。

## 2. 主线规模与默认参数

- 棋盘：6x6。
- 阵营：玩家操控3只狼，羊方由AI控制。
- 开局：主线固定15只羊，狼先手；每关开局、岩石和参数固定，不在运行时随机摆放。
- 目标：采用竞速目标。狼在被全部困住前累计捕食8只羊即获胜；羊在狼达到8只前通过封锁让三狼都无合法行动即获胜。胜利时通常剩余7只羊，公平性来自双方目标的可达性和决策质量，不要求剩余棋子数量相等。
- 无行动：只要任意一只羊有合法移动，羊方就必须选择一只可移动的羊；只有全部存活羊都无合法移动时才跳过本次羊方行动。跳过计入1次单方行动，随后交还狼方，并继续参与重复局面与行动上限判定；它不直接判定任一方获胜。
- 内容：春、夏、秋、冬各6关，共24关。
- 扩展：数量变化、特殊目标、受控随机和高复杂规则只进入主线后的挑战内容。

选择24关是为了让每季拥有教学、应用、组合、压力、精通和季终六种职责，同时避免首发堆积未经打磨的内容。目标8是当前生产基线：它让玩家经历开局、建立优势和中局转换，又不必清空全部羊群；代表关固定种子对照显示，统一改为9或10会显著增加时长并放大关卡难度差异。目标8不被宣称为永久最优值，未来变更必须重新验证24关。

## 3. 四季职责

- 春季：学习移动、隔空吃、短连吃和基础岩石认知。
- 夏季：学习羊群阻挡、三狼分工和中场路线。
- 秋季：学习密集岩石、通道、断点和连吃顺序。
- 冬季：减少岩石提示，强调空盘围堵、位置计算和高压力防守。

相邻关卡不能只是换岩石或提高AI；每关必须有独立、可描述的战术问题。

## 4. 不可违反的产品边界

- 规则、合法行动、连吃、终局和AI入口在正式游戏、Admin与模拟中保持一致。
- 狼位、羊位、岩石、AI档位、目标和回合上限共同决定难度，不能割裂调优。
- 羊方AI只能使用公开局面、有限预算和可解释目标，不读取玩家未来意图或获得额外能力。
- 地图必须提供可理解的狼方胜法和真实羊方压力；自动模拟只能筛选风险，不能代替棋谱复盘和玩家验证。
- 帮助、奖励、皮肤和未来商业化服务于核心玩法，不能掩盖规则、地图或AI缺陷。

## 5. 玩家体验要求

- 新玩家不看外部说明也能完成首次有效操作，并理解移动、隔空吃和胜负目标。
- 每关有教学点、主要策略和可解释的常见失败原因。
- 狼、羊、岩石和行动状态在手机与桌面端均清晰；静音和减少动效时仍可理解。
- 胜利、失败、重试、退出和继续都有明确下一步，不丢失进度或重复发奖。
- 皮肤、任务和奖励只提供身份感与近期目标，不产生战力优势。

## 6. 首发质量门槛

- 24关不存在未解释的首回合死局、循环、高和棋或强制一边倒。
- 正式对局、重开、Admin、恢复和模拟使用同一关卡开局与规则。
- 首局、胜败、重试、刷新、返回、异常存档和资源失败均可恢复。
- 主要手机与桌面视口、静音和减少动效场景通过体验检查。
- 规则测试、资源检查、生产构建和小规模玩家验证通过。

## 7. 渠道产品边界

- 独立站：持续更新的无广告Demo、品牌和试玩入口；保留Admin开发能力，不以广告收入作为首发目标。
- H5门户：首发商业渠道；Poki和CrazyGames复用同一规则、24关和本地奖励体系，只在平台生命周期、广告、分享和账号适配层产生差异。
- App：H5数据达标后的扩张渠道；原生广告、IAP、自有账号和推送不得提前反向增加H5主线复杂度。
- 任意渠道广告失败、不可用或被拦截时，基础游戏、通关奖励和存档必须保持可用。
