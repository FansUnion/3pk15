# Fangrush 视觉资源验收标准

> 本文是皮肤、棋子、棋盘和其他玩家可见视觉资源的长期验收标准。它维护判断方法和放行门槛，不维护某一张皮肤的当前问题；具体诊断见对应专项记录，资源文件和构建产物是技术事实来源。

## 一、核心目标

皮肤首先要保证玩家看懂棋子，其次才是主题、审美和收藏价值。高级皮肤应当比默认皮肤有明确的主题升级，但不能因为装饰变复杂而降低识别度。

## 二、十项验收标准

1. **角色识别**：在实际棋盘尺寸下，玩家能迅速分辨狼、羊和地形。
2. **缩小可读性**：缩放到棋盘实际显示尺寸后，头部、身体、四肢和朝向仍不混成一团。
3. **轮廓成立**：狼有可识别的耳朵、脸部和身体轮廓；羊有可识别的头部、羊毛、身体和四肢。
4. **主题成立**：皮肤不只是换颜色，至少有颜色、轮廓或材质、主题细节中的两项变化。
5. **套装一致**：同一套狼和羊属于同一视觉世界，颜色、材质、光照和细节语言一致。
6. **对比度合格**：在对应棋盘背景和棋盘线之上，角色边缘、眼睛和关键区域清楚可见。
7. **层级不冲突**：棋子不被棋盘线、岩石、提示圈或其他视觉层遮挡。
8. **状态兼容**：选中、移动、捕食、加载和错误状态下，皮肤仍能识别，不产生错误占位感。
9. **收藏价值**：玩家能说出它和默认皮肤的区别，并理解为什么值得解锁；不能只依赖稀缺价格制造价值。
10. **平台一致**：独立站、H5 和未来 App 使用同一资源时，比例、路径、颜色和视觉含义一致。

## 三、放行等级

| 等级 | 条件 | 处理 |
|---|---|---|
| A | 识别、主题、对比度和收藏价值均明确 | 可作为生产皮肤 |
| B | 角色可识别，但主题或细节不足 | 可作为候选，先小修 |
| C | 造型容易误认、缩小不可读或套装不一致 | 不作为正式奖励，必须重制 |
| D | 资源缺失、构建不一致、遮挡或加载异常 | 阻塞发布，先修复技术问题 |

## 四、诊断方法

1. 读取 SVG 原始结构、viewBox、路径数量、颜色和描边。
2. 检查 64x64 原始尺寸和棋盘实际缩放后的轮廓可读性。
3. 检查默认、霜狼、Night Watch 三套资源的角色识别和主题差异。
4. 检查棋盘背景、棋盘线、岩石和选中状态下的对比度。
5. 运行资源引用、构建和产物检查。
6. AI 做技术和视觉结构初审；最终审美、收藏意愿和品牌方向仍需人工确认。

## 五、禁止事项

- 不用更高价格掩盖皮肤本身不好看。
- 不用更多颜色堆叠代替角色轮廓设计。
- 不让羊变成花朵、云团或无法辨认的装饰物。
- 不让狼失去耳朵、脸部和攻击性识别特征。
- 不在资源未达到 C 级以上时把它作为正式奖励宣传。

## 六、长期维护边界

- 资源文件：`packages/web-assets/public/skins/`。
- 皮肤目录和解锁规则：`packages/game-core/src/content/skins.ts`。
- 玩家预览：`apps/player-web/components/SkinPreview.tsx`。
- 对局棋盘渲染：`apps/player-web/components/BoardSvg.tsx`。
- 运行资源检查：`pnpm check:skins`、`pnpm check:assets`。
- 具体视觉诊断、修订原因和人工结论：对应专项或任务记录。
