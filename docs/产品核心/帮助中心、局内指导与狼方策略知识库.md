# 帮助中心、局内指导与狼方策略知识库

> 状态：已实现。本文是帮助内容、局内指导规则和狼方策略定义的产品唯一来源；运行时结构化数据以 `packages/game-core/src/content/strategies.ts` 为准。

## 1. 目标与边界

帮助系统让玩家理解规则、识别盘面和形成狼方策略，不替玩家执行行动，也不能掩盖坏地图或 AI 缺陷。当前只提供规则帮助和“观察、目标、策略”三级指导，不提供自动落点、完整获胜路线、悔棋或广告。

## 2. 系统关系

`策略知识库 → 关卡主/辅策略 → 局内三级指导 → 帮助中心解释 → 玩家数据验证`。

- 策略知识库定义统一术语、盘面信号、目标和常见错误。
- 每关绑定一个主策略和一个不同的辅助策略。
- 局内指导结合关卡教学点与策略定义生成提示。
- 帮助中心解释通用规则和六种策略，不泄露单关完整答案。
- Admin 展示同一份策略数据，供设计复核。

## 3. 帮助中心

入口覆盖首页、章节、关卡说明、设置和对局。对局内以弹窗打开，关闭后保留当前棋盘、回合和选择状态。

内容包括：胜负、普通移动、隔空吃、连续吃、羊 AI、岩石规则、ASCII 棋盘图例及六种狼方策略。中英文来自同一消息结构。

## 4. 局内指导

玩家可主动逐级查看：

1. **观察**：本关教学点 + 主策略盘面信号。
2. **目标**：本关玩家目标；英文使用主策略目标。
3. **策略**：本关狼方策略 + 辅助策略原则。

每关连续失败两次后提示可以求助，但不自动弹窗。存档按关卡记录提示使用次数和连续失败次数；胜利后清零该关失败连击。

## 5. 狼方策略知识库

| ID | 中文名 | 定义 | 主要盘面信号 | 常见错误 |
|---|---|---|---|---|
| `two_wolf_pincer` | 双狼夹击 | 两狼从不同方向压缩羊群空间。 | 羊群仍有两个逃离方向。 | 两狼从同一侧追逐。 |
| `three_wolf_surround` | 三狼包围 | 两狼限制路线，第三狼完成捕食。 | 两狼可控出口，第三狼有接近线。 | 三狼挤进同一路线。 |
| `induced_movement` | 诱导移动 | 先走准备路线，逼羊进入目标区域。 | 直追被挡，但羊有可预测回应线。 | 重复追逐，等待羊自行露口。 |
| `rock_lane_control` | 岩石控道 | 利用岩石边界、通道和断点限制路线。 | 岩石形成窄入口、出口或断路。 | 把岩石当落点或堵死狼的出口。 |
| `chain_capture` | 链式捕捉 | 首吃后比较继续连吃收益与落点风险。 | 捕食后又出现同线羊与可用落点。 | 有连吃就继续，不检查退路。 |
| `preserve_mobility` | 保持机动性 | 保持狼可转向并互相接应。 | 狼出口很少或共享同一空点。 | 为单羊牺牲整体行动空间。 |

岩石只能阻挡落点和塑造路线，不能作为普通移动、跳吃或连续跳吃的中点或落点。因此不使用“岩石借力”这一容易误解规则的名称。

斜线前为主策略，后为辅助策略。具体关卡教学点、玩家目标、狼方操作建议和实测证据仍维护在单关设计卡中。

## 7. 一致性与验收

- 每关必须有一个主策略和一个不同的辅助策略。
- 策略 ID 必须来自六项正式目录，中英文定义、盘面信号和常见错误不得为空。
- 帮助中心、局内提示、Admin 和关卡设计卡使用同一策略名称。
- 提示不得直接给坐标或替玩家走棋。
- 玩家理解策略仍频繁失败时，优先检查地图、开局和 AI；不能继续加提示掩盖问题。
- 自动测试只证明结构完整；策略是否符合真实胜线仍由棋谱复盘和玩家验证确认。
