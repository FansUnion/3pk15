# 奖励、皮肤与任务设计

> 本文维护奖励循环的产品关系。具体掉落数值、皮肤库存、任务周期和存档字段以代码为准。

## 1. 设计目标

奖励为玩家提供近期目标、收藏动力和再次游玩的理由，但不产生战力优势，也不能用重复刷取掩盖关卡本身缺乏乐趣。

## 2. 碎片与获取

- 通用碎片：用于兑换稀有狼羊主题套装。
- 四季碎片：用于对应季节棋盘主题。
- 首次通关提供确定性基础奖励。
- 重复通关可以提供受控的小额奖励。
- 每日/每周任务提供清晰、可领取且不可重复领取的目标奖励。
- 未来广告激励必须单独验证，不属于首发核心奖励来源。

## 3. 皮肤关系

- 狼皮肤是主要收藏对象，三只狼统一使用所选主题。
- 羊外观跟随狼主题成套切换，不单独产生战力或兑换体系。
- 棋盘主题独立解锁和装备，可以与狼羊主题组合。
- 岩石保持稳定识别，不建立独立皮肤经济。

皮肤的颜色、轮廓和背景不得降低狼、羊、岩石、合法行动与危险状态的可读性。

## 4. 任务边界

- 任务围绕开始对局、完成关卡、获得碎片等正常行为，不要求玩家故意失败或破坏策略。
- 周期刷新、进度记录和领取必须可恢复且避免重复发奖。
- 不通过任务强迫观看广告、使用特定付费皮肤或反复刷单一关卡。

## 5. 存档与商业化边界

- 首发本地进度应满足完整单机体验；跨设备云存档属于 [需求池](../任务/需求池.md)。
- 不看广告也能完成主线并获得基础奖励。
- 广告、付费皮肤和奖励倍率由未来渠道实验决定，不能反向改变24关平衡。

## 6. 验收

- 玩家知道奖励从哪里获得、能兑换什么以及下一项目标。
- 通关、重试、退出和恢复不会重复发奖或丢失奖励。
- 皮肤装备后在手机和桌面端仍能清楚识别棋子与状态。
- 奖励节奏需要玩家数据支持；“某季最适合刷取”等结论在验证前不得作为正式设计承诺。

## 7. 奖励表达标准

奖励正确发放只是工程底线，玩家还必须理解它的意义：

- 结算页明确区分首次通关、重复通关、任务和广告奖励，不把多种来源合并成无法解释的总数。
- 同时展示本局获得、当前持有、目标皮肤成本和剩余差额。
- 解锁反馈提供预览、立即装备、继续游戏三个清晰出口。
- 双倍掉落等临时效果显示生效范围、剩余时间和实际增加量。
- 广告按钮必须与普通继续选项同时出现；广告失败不撤销基础奖励，也不阻断离开结算页。
- 任务页显示进度、完成条件、奖励和已领取状态，不鼓励故意失败或无意义刷取。

在玩家数据证明现有表达清楚之前，不新增更多货币、随机箱、Battle Pass或复杂奖励倍率。
