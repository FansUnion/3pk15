## 状态定义

- `pass`：未命中当前自动门禁红线，可进入玩家试玩，但不代表已验收。
- `review`：命中和棋、长尾或策略敏感风险，必须完成棋谱复盘并决定修订或人工放行。
- `reject`：命中明确拒绝规则，当前不得作为合格生产关卡，必须修订并复验。
- `用户试玩确认`：用户完成实际操作验证，确认难度、策略可理解性和体验符合预期。

## 统一数据口径

- 规则参数来自 `packages/game-core/src/content/levels.ts` 和 `createLevelInitialState`，不在文档中另造一份配置。
- 分析模拟使用固定提交、固定种子、固定狼策略、羊 AI 档位和 hard 的 `maxNodes=80` 无时间上限模式；这与线上 hard 的 `4000` 节点/`12ms` 预算分开记录。
- 2026-07-17 全关首轮基线命令为 `pnpm --filter @wolf-sheep/game-core exec vitest run tests/balance-sim.test.ts -t "records a configured-AI baseline" --reporter=verbose`；每关 3 局，狼策略 `mixed`，随机种子为测试公式 `i + level.indexInChapter * 1000 + sheepDifficulty.length * 10000`，因此结果可重现但不是统一单一 seed。
- 2026-07-17 双策略矩阵按季运行，命令模板为 `$env:RUN_BALANCE_MATRIX='1'; pnpm --filter @wolf-sheep/game-core exec vitest run tests/all-level-balance-matrix.test.ts -t "<season> two-strategy" --reporter=dot`；四季必须分四次执行，避免单进程超过 Vitest worker RPC 心跳时间。每关 20 局，random/mixed 各 10 局，种子 `20260717..20260726`。
- 每局至少记录胜方、终局原因、plies、吃子数、首吃 ply、无合法行动、回合耗尽、重复和异常。
- 正式胜率必须来自足够样本和玩家验证；当前开发策略 `random`、`greedy`、`capture-first` 只是诊断工具，不等同于玩家胜率。

## 调优顺序

1. 先确认玩家理解教学点和规则。
2. 保持狼在第 6 行的玩家认知锚点，调整狼列位置与协作。
3. 在第 1–3 行重排羊阵；只有证据充分时才试探第 4 行。
4. 再检查岩石通道、分区和跳吃落点。
5. 最后校准羊 AI、`targetEaten` 和 `maxPlies`。

每次只改变一个主要变量，并在对应设计卡中保留改动前后数据和棋谱。
