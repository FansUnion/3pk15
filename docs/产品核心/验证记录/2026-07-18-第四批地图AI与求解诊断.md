# 第四批地图、AI与求解诊断（2026-07-18）

> 本文保存本轮生产试玩观察、代码排查、临时对照数据和处理建议。长期关卡生产口径已吸收进[关卡与平衡设计标准](../02-关卡与平衡设计标准.md)；本文不是新的长期规范，也不直接改写24关最终验收状态。

## 1. 玩家观察

- 资深玩家在春、夏、秋多关能较快识别并反复利用同一优势，怀疑羊方不会主动完成合围。
- `summer-06` 的部分岩石之间保留两个空点，容易形成可重复建立接触和捕食的通道；羊AI若不主动拆解，狼可持续兑现。
- 秋季岩石更密，部分岩石只隔一个空点。直觉上既像狼的捕食入口，也像羊的封锁瓶颈，但实际羊方经常表现迟钝。
- 多张地图若只是岩石数量或坐标小幅变化，即使单关可运行，整季仍会缺少新鲜感。
- 僵局突然出现、羊有优势却来回移动，会让玩家怀疑规则、地图和AI哪一项出了问题。
- 玩家希望引入深度求解，判断是否存在羊方理论必胜或至少短程强制封锁。

这些观察成立的深层问题不是“再调一枚岩石”，而是现有验收体系缺少拓扑、多样性、优势转化和求解证据四个维度。

## 2. 代码事实

### 2.1 岩石收益高度非对称

狼与羊都只能沿横纵移动，但羊不能向第1行方向后退；狼还可以执行“狼—空—羊”的隔空捕食。因此岩石关闭同一个点时，对双方价值并不相等：

- 岩石与边界可能压缩狼退路，帮助羊困狼；
- 岩石也可能稳定羊的站位方向，给狼形成可重复捕食的接触线；
- 隔一个空点更像单点瓶颈，隔两个空点更可能保留移动与捕食组合空间；
- 真实收益还取决于狼位、羊阵、轮到谁走和AI是否理解该通道。

因此不能用岩石数量、视觉密度或单个间距直接推断偏狼/偏羊。

### 2.2 现有模拟低估真实连吃

24关矩阵、候选验收器和Admin批量检查在狼第一次捕食后立即结束连吃。旧结果验证的是“单吃代理”，不是完整玩家策略。尤其是以连吃为主要体验的秋季地图，旧 `pass/review` 必须在新代理下复核。

更准确的战略狼必须把“主动收手”作为决策，与继续捕食共同比较。强制吃满5只也不能代表熟练玩家，因为继续连吃有时会降低机动并导致困狼。

### 2.3 羊AI缺少优势转化能力

- Normal只比较羊走一步后的静态评分，不能稳定规划几轮后的合围。
- Hard只继续搜索Normal即时最高分动作，可能排除“短期略差、数步后强制困狼”的动作。
- 终局评分明确区分狼胜和羊胜，却没有给僵局独立价值。
- 当前护栏能排除一部分即时送子，但不能证明战术牺牲、最长连吃和合围转换合理。

所以“羊占优但不积极封锁、最后横向晃动”是可由实现解释的风险。是否发生在某一真实对局，仍应由玩家问题包和棋谱确认。

### 2.4 AI与难度说明存在旧口径

当前正式配置是春季Normal、夏季02/05 Hard、其余夏秋Normal、冬季Hard；旧辅助函数和Admin说明仍保留“春Easy、夏秋Normal、冬Hard”。地图操作难度1至5是人工内容字段，不是AI系数。

### 2.5 岩石术语存在规则冲突

`spring-03` 的“用岩石当支点”和 `autumn-01` 的“岩石就是你的跳板/跳吃支点”会让玩家误以为岩石参与跳吃；真实规则中岩石既不能成为中点，也不能成为落点。应统一改为“岩石塑造通道、边界和空隙”，并对24关前台文案、教学点和策略说明执行一次规则术语扫描。

## 4. `summer-06`地形判断

当前四枚岩石配合偏右狼位形成压力线。玩家指出的“两岩之间两个空点”值得作为独立地形模式记录：它给狼保留横向调整和再次建立隔空捕食的空间。如果羊只优化当前一步而不改变接触结构，狼可以重复使用同一通道。

这类地图不是天然不合格。它只有在满足以下条件时成立：

- 玩家需要先建立通道控制，而不是开局即可机械重复捕食；
- 羊至少有一种可理解的拆解、封锁或换线回应；
- 战略狼明显优于随机狼，但胜利不主要依赖AI重复犯同一错误；
- 玩家能说出本关与其他四岩地图的区别；
- 关键通道在视觉和教学上可读。

当前 `summer-06` 的羊方防守仍写为“尚未验证”，因此旧 `pass` 只能视为单吃代理筛查通过，不能视为地形正式验收。

## 5. 地图相似性风险

当前24关已经使用空旷、边线岩石、漏斗、横断、分流、窄门、密岩和空盘等名称，但名称不同不等于体验不同。需要建立组合差异矩阵，比较：

- 岩石拓扑和关键空隙；
- 狼与羊开局；
- 首接触方向和首次捕食节奏；
- 主要与辅助策略；
- 羊方防守方式；
- 连吃/主动收手结构；
- 常见失败和终局分布。

若两关只有一枚岩石位移，而上述体验大多相同，应合并设计意图、重新布局或明确一关为何作为教学递进保留。地图多样性不能只追求坐标差异，也不能为了新鲜感一次改变所有变量。

### 5.1 本轮24关拓扑扫描

临时诊断按6×6去岩石后的移动图计算连通区、割点、死角、度数、同排/同列岩隙，并在水平镜像下比较岩石、狼位和羊阵。测试通过，临时文件随后删除。

- 24关开放区域都保持单一连通区，没有被岩石直接切成两个永久孤岛。
- 所有关卡开局狼均无立即捕食，首回合至少需要建立位置；这符合“先读盘再接触”的基础体验。
- 秋季六关全部使用5枚岩石；每关都有2组“岩石之间只留一个空点”的瓶颈，且多数关存在2至4个割点、2至3个死角，季节结构高度集中。
- `autumn-01`、`autumn-02`、`autumn-04`、`autumn-05` 使用相同岩石集合，主要依靠狼开局、名称和策略说明区分。
- `summer-03` 与 `summer-06` 使用相同岩石集合，拓扑指标也相同；当前主要差异来自狼位和设计意图。
- `spring-05` 与 `spring-06` 使用相同或水平镜像等价的双岩结构。
- 冬季六关全部无岩石，地形差异完全依赖开局；其中 `winter-05` 与 `winter-06` 的岩石、狼位、羊阵在水平镜像归一后完全等价，是当前最明确的重复地图风险。

这不能自动判定上述关卡应删除：相同地形可以通过不同开局、目标和策略形成递进。但若完整棋谱、首接触、主要胜线和失败原因也相同，就不能仅靠名称把它们视为独立关卡。`winter-05/06`、`summer-03/06` 和秋季四张同岩石地图应进入首轮组合差异复盘。

## 6. 理论必胜求解方案

用户提出的“深度完全穷举”有长期价值，但必须使用证明语义。

### 6.1 能解决的问题

- 某局面狼是否能在N层内强制达到8只；
- 某局面羊是否能在N层内强制困住三狼；
- 正式AI是否错失已经被证明的封锁动作；
- 某张地图是否存在极短强制胜负；
- 某条玩家反馈究竟是AI缺陷还是理论上无法突破。

### 6.2 不能直接承诺的结果

完整开局有18枚棋子、岩石、连续跳吃、300次行动和重复局面。即使棋子同类归并，状态数量和搜索深度仍很大；“规则有限”不等于能在浏览器或Vercel请求内完成全局证明。

首版求解器必须返回四态：狼已证明、羊困狼已证明、僵局已证明、未知。预算耗尽只能返回 `unknown`，不能用启发式评分伪装成必胜。

### 6.3 推荐架构

1. 在game-core旁建立纯离线分析模块，复用同一序列化、合法行动和终局规则。
2. 使用确定性极大极小搜索、alpha-beta、置换表、同类棋子归并和稳定动作排序。
3. 将主动结束连吃纳入狼方决策树。
4. 先支持8/12/16层的短程证明与主变化棋谱，再扩展特定残局。
5. 输出配置哈希、commit、规则版本、深度、节点、耗时、证明状态和关键棋谱。
6. Admin只导入和解释结果；长任务先由本地CLI运行，不依赖数据库，不在Vercel请求中直接执行。

## 7. 归因矩阵

| 观察 | 求解/复盘结果 | 归因 |
|---|---|---|
| 羊优势后晃动 | 已证明有短程强制困狼，AI未选 | AI缺陷 |
| 羊优势后晃动 | 只能证明可保僵局 | 防守合理，AI表达/僵局预警问题 |
| 狼反复利用固定间隙 | 羊存在明显拆解动作，AI反复错失 | AI缺陷为主，地图可继续评估 |
| 狼反复利用固定间隙 | 羊无有效回应且狼可机械强制胜 | 地图/开局缺陷 |
| 狼无法突破 | 已证明狼有胜线但当前代理找不到 | 狼代理或教学缺陷 |
| 狼无法突破 | 羊可从开局强制困狼 | 地图候选拒绝或重新定义挑战 |

## 8. 推荐执行顺序

1. 修复模拟器的连续捕食和主动收手决策，重新命名旧证据。
2. 建立岩石拓扑分析和24关差异矩阵。
3. 实现有限深度证明搜索器，不先承诺完整开局全解。
4. 修复羊AI终局效用、最长连吃风险和优势转化。
5. 修正Admin难度说明和所有违反真实岩石规则的前台/教学文案。
6. 重新运行24关矩阵并复盘代表棋谱。
7. 再决定AI档位、地图、开局和连吃上限；每次只改一个主要变量。
8. 用户最后进行24关人工验收，确认地图辨识度、策略可学性和资深玩家难度。

正式目标8、全局连吃5、三次重复和300行动上限在完成上述证据前保持不变。

- 当前被直接跳吃威胁的羊数量；
- 狼被限制的比例；

这些指标用于解释“羊胜”或“狼被困”的形成过程，不把某个分数直接当作平衡结论，也不替代有限求解和人工体验。

### 当前价值

- 如果羊胜局面仍有大量狼行动或直接跳吃机会，说明“羊胜”可能只是当前代理停止或规则优先级造成，不能直接称为羊方有效防守。
