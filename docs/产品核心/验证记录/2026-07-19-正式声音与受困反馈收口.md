# 正式声音与受困反馈收口（2026-07-19）

## 结论

声音的确定性开发已完成。玩家端不再把运行时合成音当作正式交付：14 类声音事件均有独立 WAV，合成音仅在样本加载失败时降级。新增羊危险辨识、单狼受困视觉/文字/声音反馈、浏览器音频恢复和设置页逐项试听。

主观听感仍需真实手机和桌面试听；它属于产品验收，不代表代码或资源缺失。

## 已实现范围

- 狼移动、羊移动、第一次捕食、连续捕食、新增威胁、单狼受困；
- 胜利、失败、重复或行动上限和局；
- 选中、非法操作、羊方思考、皮肤解锁、皮肤装备；
- 单狼只在“有合法行动→无合法行动”时提醒，持续受困不重复，恢复后再次受困可重新触发；
- 三狼全部受困时抑制单狼提醒，只保留正式失败反馈；
- 设置页提供 14 类中文/英文场景试听，局内和设置均可恢复被浏览器暂停的声音；
- 棋盘用虚线受困环和短文案表达状态，静音时仍可理解。

## 技术证据

- 正式资源：`packages/web-assets/public/sfx/`；
- 可复现生成：`packages/web-assets/scripts/gen-sfx.mjs`，内部噪声使用固定状态；
- 资源门禁：`packages/web-assets/scripts/check-public-assets.mjs`；
- 事件差分：`apps/player-web/lib/wolf-feedback.ts`、`play-store.ts`；
- 播放、预加载与降级：`apps/player-web/lib/sfx.ts`；
- 玩家反馈与试听：`PlayScreen.tsx`、`BoardSvg.tsx`、`app/settings/page.tsx`。

- `pnpm --filter @wolf-sheep/player-web test`：通过，9 个文件、28 项测试；
- `pnpm build:player`：通过，生产构建和 66 个静态页面生成成功。
- `pnpm release:check`：通过，22 项发布门禁全部 PASS；包含 106 项核心测试、28 项玩家测试、玩家/Admin 生产构建、资源和平台边界检查。

## 人工复核方式

打开玩家端设置页的“声音试听”，逐项播放；再进入真实对局确认羊新进入立即捕食线、单狼新近受困和三狼全部受困三个时序。只记录听不出区别、过响、像错误音或角色感不合适的项目。
