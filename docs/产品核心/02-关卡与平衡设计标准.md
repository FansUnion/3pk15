# Fangrush 关卡与平衡设计标准

> 本文是设计、修改、验证和发布主线关卡的唯一方法规范。逐关产品结论维护在 [24关](24关/README.md)，运行参数以关卡代码为准。

## 1. 平衡定义

平衡不要求完美博弈下双方五五开，而要求：

- 狼方有可理解、可执行、可复盘的获胜路径。
- 羊方能通过站位、封锁、分散、牺牲或拖延形成真实防守压力。
- 胜负主要由玩家决策决定，不依赖开局强制结果、AI明显失误或回合上限。
- 失败能归因于可学习的问题，例如错失夹击、过早捕食、进入坏通道或失去退路。
- 不同关卡表达不同战术，不用单纯增加岩石、提高AI或缩短回合制造难度。

## 2. 主线固定边界

- 主线默认3狼、15羊、吃8只羊；每关使用固定开局。
- 狼通常从第6行出发，列位可以调整；羊阵可按关卡覆写，但必须保持15只且完整复验。
- 岩石必须在棋盘内、不压棋子、不造成非法开局，并能解释其对路线或协作的作用。
- 回合上限用于控制节奏和终止异常长局，不能成为羊方唯一防守手段。
- Easy、Normal、Hard代表羊方防守稳定性和有限预测能力，不代表隐藏规则加成。

## 3. 开局与地图原则

狼位和羊位都是平衡变量：

1. 优先保持三狼从下方出发的玩家认知，只调整列位和协作关系。
2. 羊阵优先在第1至3行内重排，改变首接触、分组、通道和围困压力。
3. 只有证据证明上述调整不足时，才试验少量第4行羊位，并单独解释其教学影响。
4. 开局候选确定后再调整岩石；最后才校准AI、目标和回合上限。

岩石用于形成通道、分区、漏斗、伏击点和撤退路线，不用于永久隔离、堵死棋子或制造唯一被迫动作。

## 4. 每关设计卡

每张主线关卡必须维护：

- 关卡名称、名称含义、前台中文说明和设计理念。
- 玩家体验目标、教学点、主要与辅助策略。
- 羊方防守重点、狼方可执行计划和常见失败原因。
- AI难度、玩家操作难度、预期节奏和风险标签。
- 自动门禁状态、关键证据、玩家反馈和最终验收结论。

精确坐标、完整配置、代码行号和实现字段不在设计卡重复维护；需要查看真实局面时使用Admin或关卡代码。

## 5. 标准生产流程

1. **设计意图**：明确战术问题、教学点和失败应学到什么。
2. **候选布局**：比较至少三个单变量候选的首回合行动、接触机会、羊移动空间和狼协作空间。
3. **规则校验**：检查边界、重叠、岩石、合法行动、AI行动与初始终局。
4. **自动筛选**：固定AI版本、预算、策略和种子，记录胜负、和棋、平均/P95 plies与异常棋谱。
5. **人工复盘**：复盘狼胜、羊胜、最长局、首次捕食、连吃、重复和回合耗尽。
6. **玩家验证**：观察新手、普通、熟练玩家的理解、通关、重试和失败归因。
7. **发布评审**：将结论写回设计卡；有问题则单变量修订并重新走完整流程。

自动证据必须写明完整 commit、关卡配置、AI档位、狼策略、搜索预算、种子、样本数和运行警告。当前结论只引用一份版本化基线；旧实验保留在验证记录，但必须标记为历史证据，不能与当前胜率混用。

若调整 `targetEaten`，除胜负外还必须比较达到原目标后的边际 plies、困狼/重复/超步变化和新增决策价值。全局目标变更不得用于掩盖少数地图的开局、路线或AI问题。

## 6. 状态定义

- `pass`：未命中自动门禁红线，可以进入玩家试玩，不代表已验收。
- `review`：存在策略敏感、长尾、和棋或其他风险，必须复盘并决定修订或人工放行。
- `reject`：命中明确拒绝规则，不能进入生产版本。
- `玩家确认`：真实操作后确认难度、策略和体验符合设计。
- `已验收`：规则、自动证据、关键棋谱和玩家验证均达到发布标准。

## 7. 风险与拒绝条件

- 开局存在无法避免的极短强制胜负。
- 岩石造成永久堵死、无互动分区或无意义唯一走法。
- 狼方主要依赖羊AI无法识别基础威胁。
- 羊方只能重复移动等待回合耗尽。
- 同一配置的结果主要由随机流或模拟器缺陷决定。
- `opening`、岩石、终局或回合计数在正式游戏、重开、Admin和模拟中不一致。

完整状态空间不以全局穷举作为首发证明。小残局可以精确搜索，完整关卡必须结合固定种子模拟、异常棋谱和玩家测试。
