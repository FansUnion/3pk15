# Fangrush 核心产品总体规划

> 本文只维护从当前状态走向首发的阶段路线。活动任务和完成状态以 [任务清单](../任务/任务清单.md) 为准，逐关结论以 [24关设计卡总表](24关设计卡总表.md) 为准。

## 产品目标

玩家操控三只狼，在可理解、可复盘的固定关卡中战胜羊群。首发版本的核心不是关卡数量或商业功能，而是四季24关具有清晰教学、可执行胜法、真实防守压力和稳定体验。

## 当前判断

- 规则、固定开局、AI、Admin试玩、帮助与提示、24关内容档案和自动候选门禁已具备。
- 自动门禁当前为 11关 pass、13关 review、0关 reject。`pass` 只表示自动证据未发现高风险，不等于玩家验收。
- 24关均完成生产侧验证，但玩家最终确认仍为 0/24；因此当前准确状态是“可供生产试玩，尚未完成玩家验收”。
- 历史“产品质量提升”和“用户体验提升”中的长期原则已进入核心要求；仍未完成的体验验证进入当前任务或需求池。

## 首发路线

### 阶段一：24关玩家验证

- 先试玩13张 review，抽查11张 pass。
- 检查策略是否可理解、失败是否可归因、羊方压力是否真实、操作反馈是否清晰。
- 问题必须区分规则、AI、地图、教学、界面或资源原因，不能只凭单次胜负调参。

完成标准：每关有玩家结论；问题关有复现证据、处理决定和复测结果。

### 阶段二：发布质量收口

- 验证首局、胜败、重试、退出、继续、奖励和本地存档恢复。
- 覆盖手机/桌面主要视口、静音、减少动效、刷新、返回和异常数据。
- 运行规则测试、资源检查、类型检查、生产构建与关键路径冒烟。

完成标准：无阻塞问题，无未解释的重要体验问题，24关达到发布条件。

### 阶段三：首个 H5 门户上架

- 独立网站继续作为演示和更新入口，当前不接 AdSense。
- 在 Poki、CrazyGames 等候选中选定首个平台，核对当期政策、账号、素材、包体、SDK和审核要求。
- 平台适配不得分叉规则、AI或关卡内容；广告失败不得影响基础游戏。

完成标准：选定平台的发布包、商店材料、平台检查与提交流程全部可执行，并保留独立站演示版本。

## 后续方向

- 玩家数据证明需要时，再启动悔棋、高级提示、AI节奏选项或专业资源升级。
- 主线稳定后，才启动独立挑战赛、周期关卡和长期赛季。
- 有真实流量后再验证广告频次、留存和商业转化，不用商业指标掩盖核心体验问题。

## 决策顺序

玩家理解与操作可靠性 -> 24关质量 -> 发布稳定性 -> 平台适配 -> 商业实验 -> 长期内容。

任何阶段发现规则、地图或数据链路问题，应回到对应唯一来源修复并重新验证，不通过增加文档或隐藏参数绕过问题。
