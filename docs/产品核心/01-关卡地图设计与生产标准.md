# Fangrush 关卡地图设计与生产标准

> 状态：现行。本文定义每张主线或扩展地图从想法到发布的生产流程。核心产品边界见 [核心产品需求与质量要求](00-核心产品需求与质量要求.md)。

## 1. 地图的职责

地图不是“放几块岩石”。每张地图必须让玩家学习或运用一个明确策略，并以可复盘的固定局面提供合理压力。

一张合格地图需要同时回答：玩家这关学什么、狼怎样创造机会、羊怎样形成可理解的抵抗、岩石为何在此处、失败后玩家应理解什么。

## 2. 主线地图固定规则

- 每关固定狼、羊、岩石、AI、目标吃子数和回合上限；相同关卡从相同盘面开始。
- 主线默认 3 狼、15 羊、`targetEaten = 8`。羊保持基础阵，狼可从批准的底线模板中选择。
- 岩石不得与开局子重叠、不得相邻、不得堵死狼首回合。
- 春季强调理解和信心；夏季强调路线与分工；秋季强调岩石通道与连吃顺序；冬季以空盘高压和狼机动性为主。
- 受控变体、每日固定种子、特殊目标、狼羊数量变化只用于扩展内容，不能混入四季主线。

## 3. 每关必填设计卡

在新增或改动地图前，先填写以下字段；字段不完整不得进入代码：

```text
关卡 ID / 季节 / 序号 / 内容层级
教学点 / 主要策略标签 / 辅助策略标签
固定狼开局 / 固定羊开局 / 岩石坐标与每块岩石的作用
AI 档位与版本 / targetEaten / maxPlies
预期最短、目标、最长回合
目标玩家群体与目标胜率 / 和棋上限
常见错误 / 预期失败原因 / 复盘要点
```

## 4. 标准生产流程

1. **设计意图**：写清策略、教学点和失败应学到的内容。
2. **候选布局**：至少准备 3 个候选；比较首回合狼行动、直接吃子机会、羊移动空间、狼协作空间和岩石路线。
3. **规则校验**：检查边界、重叠、岩石、首回合行动、AI 合法行动与初始终局。
4. **AI 模拟**：使用固定 AI 版本、节点预算、种子和至少两种狼策略，记录胜负、和棋、平均/P95 回合与异常棋谱。
5. **人工复盘**：查看最短胜局、最长局、羊胜局、狼胜局、首次吃子、首次连吃和超步局。
6. **玩家验证**：分别观察新手、普通、熟练玩家的理解、通关、重试和失败原因。
7. **发布评审**：将设计卡、参数、结果、异常与最终理由写入关卡台账，再合入主线。

## 5. 平衡调节顺序

出现异常时严格按以下顺序处理：

1. 玩家是否理解规则和本关教学点。
2. 狼、羊与岩石的固定开局是否提供合理路线。
3. 是否有循环、死角、无意义和棋、必胜或必败。
4. 羊 AI 的逃跑、抱团、阻挡和随机性是否符合档位。
5. 回合上限和 `targetEaten` 是否符合地图节奏。

AI 不是修复坏地图的第一工具；增加岩石也不是默认难度方案。

## 6. 验收标准

主线关卡发布前必须满足：

- 设计卡完整，布局合法，首回合可操作。
- 目标策略、岩石作用、失败原因和难度理由可解释。
- 无未解释的高和棋、死局、循环或极端一边倒结果。
- 平衡数据使用完整关卡参数，异常对局有可复查棋谱。
- 新手知道本关关注什么；熟练玩家能通过策略稳定改善表现。

## 7. 扩展内容的额外规则

精通关、挑战赛与每日内容可以改变目标、数量或使用受控变体，但必须：明确标示特殊规则、使用通过筛选的候选布局、保留稳定可重复的高水平胜法，并与主线数据和解锁进度隔离。

未来地图生产、受控随机和内容更新方案见 [产品扩展](../产品扩展/README.md)。
