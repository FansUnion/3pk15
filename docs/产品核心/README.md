# 产品核心

本目录是现行产品定义和24关内容资产的唯一入口。建议按下列顺序阅读，不需要从日期型报告开始。

## 七份长期规范

| 顺序 | 文档 | 回答的问题 |
|---|---|---|
| 1 | [核心产品定义](01-核心产品定义.md) | 游戏是什么、主线为什么是24关、首发必须达到什么 |
| 2 | [关卡与平衡设计标准](02-关卡与平衡设计标准.md) | 一张地图怎样设计、调优、验证和验收 |
| 3 | [玩家教学与策略系统](03-玩家教学与策略系统.md) | 怎样教玩家理解规则并学会狼方策略 |
| 4 | [奖励、皮肤与任务设计](04-奖励皮肤与任务设计.md) | 通关后给玩家什么目标，哪些奖励边界不能违反 |
| 5 | [产品体验与商业化原则](05-产品体验与商业化原则.md) | 怎样统一玩家旅程、商业边界、跨平台和反馈判断 |
| 6 | [视觉资源验收标准](06-视觉资源验收标准.md) | 棋子、皮肤、棋盘和视觉资源怎样诊断与放行 |
| 7 | [界面文案与交互验收标准](07-界面文案与交互验收标准.md) | 玩家端与 Admin 的信息怎样表达、排序并完成交互验收 |

## 24关内容

- [24关总览](24关/README.md)：唯一逐关状态入口，当前为11关 `pass`、13关 `review`、0关 `reject`、玩家确认0/24。
- `24关/春夏秋冬-关卡.md`：每关名称、文案、设计理念、策略、模拟证据、风险和玩家验收记录。
- 精确坐标和运行参数以关卡代码与Admin为准，设计卡不复制代码行号和完整配置。

## 验证记录

[验证记录](验证记录/README.md)保存特定版本的模拟矩阵和问题归因。它们是产品结论的证据，不是当前任务清单，也不覆盖24关总览。

## 维护规则

- 产品为什么这样设计：更新对应的七份长期规范。
- 某一关的名称、策略、风险或玩家结论：更新对应设计卡。
- 逐关状态：更新24关总览。
- 模拟参数和某次运行结果：新增日期型验证记录。
- 当前进度、优先级和负责人：只更新 [任务清单](../任务/任务清单.md)。
- 类型、坐标、函数、字段和脚本参数：只由代码与测试维护。
