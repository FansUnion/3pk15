# 产品核心

本目录是现行产品定义和24关内容资产的唯一入口。建议按下列顺序阅读，不需要从日期型报告开始。

## 24关内容

- [24关总览](24关/README.md)：唯一逐关状态入口；羊 AI V3 当前为 `12 自动未发现风险 / 12 需要复核 / 0 自动拒绝`，仅保留 8 局定向玩家确认。
- `24关/<level-id>.md`：逐关名称含义、设计理念、策略解释、风险和玩家反馈；其中名称、前台文案和运行摘要是代码配置的受测试投影，不是第二份配置源。
- 精确坐标和运行参数以关卡代码与Admin为准，设计卡不复制代码行号和完整配置。

## 验证记录

[验证记录](验证记录/README.md)保存特定版本的模拟矩阵和问题归因。它们是产品结论的证据，不是当前任务清单，也不覆盖24关总览。

## 维护规则

- 产品为什么这样设计：更新对应的九份长期规范。
- 某一关的名称、策略、风险或玩家结论：更新对应设计卡。
- 逐关状态：更新24关总览。
- 模拟参数和某次运行结果：新增日期型验证记录。
- 当前进度、优先级和负责人：只更新 [任务清单](../任务/任务清单.md)。
- 类型、坐标、函数、字段和脚本参数：只由代码与测试维护。
- 重复边界：运行事实归代码，当前逐关状态归24关总览，某次数据归验证记录，设计理由和人工观察归单关卡；任务文档不再复制以上内容。
