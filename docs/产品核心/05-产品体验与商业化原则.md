# Fangrush 产品体验与商业化原则

> 本文是 Fangrush 长期产品判断参考，统一考虑游戏核心、用户体验、内容设计和商业化。它维护稳定原则、判断框架和阶段边界，不维护具体问题状态、模拟结果或代码细节。近期任务以 [任务清单](../任务/任务清单.md) 为准，广告与渠道实现以 [渠道能力与广告商业化总方案](../商业成功/渠道能力与广告商业化总方案.md) 和代码为准。

## 一、产品目标

Fangrush 的核心不是单纯让狼赢，也不是尽可能增加关卡、广告或系统，而是让玩家在可理解的压力下，通过自己的判断完成狼方闯关，并愿意继续挑战、复盘和收藏。

一个合格的产品闭环是：

```text
看懂目标
  -> 看懂棋盘和规则
  -> 做出一次有意义的狼方决策
  -> 得到清楚的结果反馈
  -> 理解奖励和下一步
  -> 愿意继续挑战或收藏
```

任何商业化、教学、AI 或视觉设计，都不能破坏这个闭环。

## 二、三条基本原则

### 1. 广告不应该是玩家的主要任务

玩家来玩游戏，不是来完成广告任务。广告可以出现在自然停顿点，但不能成为进入关卡、领取基础奖励、解锁主线或解决规则困惑的必要条件。

广告不得打断以下过程：

- 首次教学和规则理解。
- 选择狼、移动、跳吃和连续跳吃。
- 羊 AI 行动和局面判断。
- 关键失败反馈和需要立即重试的时刻。

独立站 Demo 默认无广告；H5 首发优先采用结算后的自愿激励广告；App 只有在真实数据证明核心体验稳定后再评估低频插屏和其他广告形式。

### 2. 广告需要具备足够的可见价值

广告不是不能展示，而是必须让玩家在观看前就知道价值，并在观看后立即感受到结果。玩家通常愿意为以下事情看广告：

- 更快获得已经理解用途的碎片。
- 更快接近明确的皮肤目标。
- 获得额外但非必需的收藏进度。
- 在不改变胜负公平性的前提下减少重复劳动。

激励广告前应说明奖励、持续时间和适用范围；广告成功才发放额外奖励；取消、失败、无填充或冷却不能吞掉基础奖励，也不能阻断下一关。

广告效果不能只看点击率，还要看广告完成率、广告后继续率、会话长度、留存和玩家投诉。点击率上升但首局完成率或回访下降，不属于成功。

### 3. 先建立理解，再扩大商业化

推荐顺序：

```text
理解玩法
  -> 完成一关
  -> 理解奖励
  -> 产生收藏目标
  -> 自愿加速收藏
  -> 高参与玩家付费或长期回访
```

如果玩家不知道岩石是什么、为什么不能走、碎片有什么用途，增加广告、货币、皮肤和任务只会增加复杂度。

## 三、玩家体验设计框架

每个页面都应在短时间内回答“这是哪里、现在能做什么”，并只保留一个主任务和一个主按钮。正常、加载、空/锁定、错误、恢复和终局状态都必须有明确出口；页面局部正确不能替代首页到关卡、对局、结算、跨季节和末关的端到端连续性。

玩家端使用动作和结果语言；Admin 先展示结论、用途和建议动作，再展开参数与原始证据。任何数字都要明确来源、版本、样本和不能证明什么，不能把代理模拟称为玩家胜率，也不能把设计节奏写成实测平均。

### 1. 玩家不应被迫解释系统

玩家不应该猜测：

- 当前关卡是否已经完成。
- `Continue` 是继续未完成关卡还是重新游玩。
- 通关后下一步应该做什么。
- 哪些棋子可以操作，哪些只能观察。
- 非法落点的原因是什么。
- 岩石是障碍、保护物还是跳吃路线的一部分。
- 奖励是立即获得、未来生效还是仅用于展示。

界面应优先显示状态、原因和推荐动作，而不是只显示内部术语或装饰。

### 2. 通关结果优先服务于继续游戏

建议的通关层级：

```text
完成的关卡和终局结果
  -> 基础奖励确认
  -> 下一关（主行动）
  -> 再玩一次（次行动）
  -> 分享战绩（辅助行动）
  -> 观看广告获得额外奖励（自愿行动）
```

`Next hunt` 通常应是主按钮，因为玩家完成一关后的自然意图是继续主线；广告不应抢占主线按钮，但应在奖励区域附近清楚说明价值。

### 3. 视觉必须表达规则含义

狼、羊、岩石和合法落点应有稳定的视觉层级：

- 狼是玩家可操作对象，选中、悬停、焦点和非法动作反馈明确。
- 羊是局面对象，不应呈现成可被玩家选择的控件。
- 岩石是固定地形，必须说明它对移动和跳吃的作用。
- 棋子不应被棋盘边框遮挡。
- 加载占位不应看起来像错误皮肤或错误棋子。

鼠标、触控、键盘、读屏和移动端布局应使用同一套交互语义。

## 四、游戏核心与商业化边界

### 核心免费内容

- 基础规则和完整对局。
- 24 张核心关卡。
- 每个季节的基础主题识别。
- 基础通关奖励。
- 不看广告也能继续主线。

### 可作为长期收藏内容

- 高级狼羊套装。
- 夜间或特殊季节主题。
- 特殊棋盘表现和非战斗特效。
- 与主线公平性无关的外观内容。

### 不应商业化的内容

- 基础规则理解。
- 首次教学。
- 主线下一关。
- 基础通关奖励。
- 影响胜负的战力、移动能力或必胜提示。

## 五、货币和季节主题原则

每种货币都必须满足：

```text
能获得 -> 看得到余额 -> 有明确目标 -> 能兑换 -> 兑换立即生效
```

基础季节棋盘适合随章节免费获得或自然解锁，因为它承担世界观和章节识别功能。季节碎片只有在存在真实对应兑换目标时才应展示和持续发放。

通用碎片可以服务于高级狼羊套装；季节碎片可以服务于对应季节的高级主题。没有实际用途的货币不应以完整系统的形式向玩家承诺。

商业化的长期价值来自玩家形成收藏目标，而不是不断增加货币种类。

## 六、AI、自动化和人工验收的分工

Admin AI 是关卡验收助手，不是人工试玩的替代品。

### 自动化和模拟负责

- 规则是否异常。
- 地图、开局、岩石和终局是否可复现。
- 指定 AI 难度下的胜率、回合耗尽和异常终局。
- 固定种子棋谱和反例。
- 候选关卡是否值得人工复盘。

### 人工试玩负责

- 玩家是否看得懂规则和棋盘。
- 难度是否有压力但不依赖明显 AI 失误。
- 关卡说明、局内指导和奖励是否自然。
- 视觉、触控、声音和移动端体验。
- 玩家是否愿意继续玩和主动观看激励广告。

Admin 页面应先展示结论、风险和建议动作，再展示种子、模拟次数、胜率、棋谱和 CSV 等证据。页面数据与仓库自动化测试使用同源 `game-core` 规则和 AI 入口，但不代表是同一份测试结果。

正确验收链路是：

```text
规则自动化
  -> 固定种子模拟
  -> 候选验收和棋谱
  -> Admin 结论与证据
  -> 人工试玩
  -> 生产放行
```

## 七、跨平台一致性

独立站、H5 和未来 App 的规则、关卡、AI、奖励基础语义必须一致；平台差异只体现在广告、分享、存档、登录、统计和原生能力适配。

- 独立站：无广告 Demo 和品牌入口。
- H5 平台：使用平台允许的广告和生命周期能力，先验证自愿激励广告。
- App：数据达标后再接入原生广告、去广告 IAP、皮肤 IAP、账号和云存档。
- Admin：面向管理、模拟、复盘和验收，不进入玩家公开构建。

## 八、用户反馈处理协议

用户可以只提供现象，不必提供完整技术错误信息。开发侧必须主动完成：

```text
现象复现
  -> 读取代码链路
  -> 检查同类入口
  -> 判断规则/地图/AI/UI/奖励/平台/技术归因
  -> 添加或运行回归测试
  -> 修复或说明未确认原因
  -> 桌面、手机、平板回归
  -> 更新反馈记录和任务状态
```

不能把无法复现的推测写成已确认结论；如果是浏览器或平台能力限制，必须提供可用的降级路径。

## 九、阶段决策

### 当前阶段

- 优先收口通关、下一关、奖励、棋盘交互、帮助、分享和 Admin 易用性。
- 独立站保持无广告。
- H5 先准备和验证结算后的自愿激励广告。
- 先确认四季基础主题和真实兑换目标，再决定季节碎片展示范围。

### 数据达标后

- 测试低频局间广告。
- 评估 App 壳和一种主广告网络。
- 评估去广告 IAP 和少量高级皮肤 IAP。
- 根据真实收藏和留存数据扩展内容。

### 暂缓

- 悔棋、高级提示、账号、云存档、复杂任务、订阅、Battle Pass、多广告网络和专业资源大规模重制。

## 十、长期判断问题

每一次扩大范围前，都要回答：

1. 玩家当前是否已经理解核心玩法？
2. 这个功能解决的是明确问题，还是增加了复杂度？
3. 它是否影响首局完成、继续率或公平性？
4. 是否有真实数据或玩家反馈支持？
5. 是否能在一个平台先验证，再扩展到其他平台？
6. 失败时是否有清晰降级路径？

若无法回答这些问题，需求应留在需求池，而不是直接进入开发。

## 十一、长期生命周期总纲

Fangrush 的长期价值不来自无限增加规则，而来自一个稳定、可反复生产内容的核心：玩家持续学习如何用三狼创造接触、控制路线、判断连吃和保留机动，地图与羊方防守不断提出新的空间问题。

产品生命周期分为四段：

| 阶段 | 核心目标 | 主要内容 | 不应做 |
|---|---|---|---|
| 上线前 | 证明核心好玩且24关有差异 | 规则、地图、AI、教学、结算、复现和发布质量 | 为收入提前削弱体验或堆系统 |
| 首发验证 | 证明玩家看懂、愿意继续 | 24关、基础收藏、真实数据、平台自然广告节点 | 根据少量情绪反馈频繁改核心规则 |
| 内容成长 | 服务普通与熟练玩家 | 精通关、挑战防守、每周固定挑战、少量新皮肤 | 无筛选随机地图和高频活动债务 |
| 稳定运营 | 形成可持续内容和收入 | 季节扩展、挑战赛季、App/IAP评估、长期收藏 | 让付费能力决定胜负 |

上线前最重要的不是“功能完整”，而是完成三个证明：

1. **核心证明**：不靠羊AI明显失误，狼方仍有可学习的获胜乐趣。
2. **内容证明**：相邻关卡有真实的新问题，不是换坐标和文案。
3. **持续证明**：规则、AI、地图和证据可以版本化扩展，不必每次重写系统。

## 十二、稳定核心与可扩展变量

长期架构把内容分为三层，避免商业化或新模式反向破坏主线。

### 稳定核心

- 6×6交点棋盘、3狼15羊、双方基础移动与捕食几何。
- 狼方主动选择是否继续连吃，羊方通过封锁和困狼防守。
- 合法行动、岩石、终局、重复和回放在所有平台一致。
- 主线目标8和基础规则在首发证据完成后冻结为一个规则版本。

### 关卡变量

- 狼位、羊阵、岩石拓扑、羊AI档位、行动上限和教学目标。
- 这些变量必须随关卡配置、配置哈希、规则版本和验证记录一起保存。
- 调整一个变量后，原胜率、棋谱和求解结论自动失效，必须重新生成。

### 挑战变量

- 连吃限制、不同捕食目标、限定狼、最少行动、固定残局和特殊胜利条件。
- 它们属于主线之后的公开挑战，不隐藏覆盖主线规则，也不作为付费战力。

这一分层保证未来可以增加内容，同时让老关卡、回放、攻略和测试保持可解释。

## 十三、玩家分层与难度产品

单一难度不可能同时满足第一次接触游戏的玩家和熟悉原版3打15的资深玩家。难度不应只通过“羊更聪明”实现，而应组合理解、执行和防守压力。

推荐首发采用：

- **主线标准防守**：每关固定并经过验证的AI，承担教学和完整24关体验。
- **挑战防守**：关卡通关后开放，使用更强、确定、公开的羊AI，记录独立徽章或成绩，不阻断主线。
- **新手辅助**：连续失败后提供策略原则、盘面信号或可选低压重试；不偷偷改变规则，不把明显送子称为Easy。
- **Admin求解/研究模式**：用于证明和诊断，不作为生产对手。

Easy、Normal、Hard若继续保留，应共享同一目标函数和基础防错，只在候选宽度、搜索深度、节点预算和受控随机性上形成稳定梯度。Easy可以看得浅，但不应主动选择被严格支配的明显送子；Hard不能读取玩家未来操作，也不能使用隐藏规则。

难度成功不以“玩家输得多”定义，而以以下组合定义：

- 随机狼明显弱于策略狼；
- 普通玩家通过学习能够提升；
- 熟练玩家仍需计算和取舍；
- 失败原因能被解释；
- 高难AI能把优势转化为困狼，而不是单纯拖延。

## 十四、连吃机制的长期决策

连吃是 Fangrush 最有辨识度、最能制造高光时刻的规则之一，同时也是最容易放大地图和AI缺陷的能力。

### 主线建议

- 全局上限暂时保留5，不根据当前有限数据直接降为2。
- 春季主要出现1至2吃的可读机会；夏季增加2至3吃与主动收手；秋季强调通道和连吃终点；冬季要求在高压下判断是否值得继续。
- 地图不能保证玩家每次都吃满，上限5代表空间，不代表设计目标。
- 模拟器必须把“继续捕食”和“主动结束”作为同等动作分析；总是吃满与总是单吃都不是熟练玩家代理。

### 为什么不出售连吃次数

把第3至5次连吃做成广告或付费解锁会直接改变胜负能力，使关卡、排行榜和攻略失去公平口径，也会让免费玩家怀疑失败来自付费限制。它不属于合适的商业化内容。

连吃适合产生的长期价值是：

- 连吃纪录、徽章、战绩卡和排行榜指标；
- “最多只能连吃2次”或“完成一次4连吃”的公开挑战；
- 更强的捕食动画、音效、棋子拖尾和主题反馈；
- 精通关中的高风险高收益路线。

商业化应围绕连吃的表达、收藏和挑战内容，而不是出售基础能力。

## 十五、内容组合与新鲜感

长期留存需要“熟悉规则、持续出现新问题”，不是每关都增加新规则。24关和未来扩展采用内容组合管理：

- 每张地图归入明确拓扑家族，并记录瓶颈、捕食通道、封锁通道和边线口袋。
- 相邻关至少在拓扑、开局接触、主要策略、节奏或羊方防守方式中有一项显著差异。
- 一个季节6关至少覆盖3种主要结构；季节终关综合该季能力，不复制前关布局只提高AI。
- 水平镜像、同岩石不同狼位和同开局不同文案都要经过棋谱级差异验证。
- 新地图先回答“玩家将学会什么”和“为什么愿意再玩一关”，再讨论坐标。

当前拓扑扫描已经发现冬季05/06等价、夏季03/06同岩石、秋季四关同岩石集合。它们不是自动删除项，但必须优先证明棋谱、策略和情绪节奏不同；无法证明时应重做，而不是靠名称维持数量。

## 十六、AI与理论求解的长期角色

生产AI、模拟代理和理论求解器是三种不同产品：

| 系统 | 目的 | 可以使用启发式 | 可以声称必胜 |
|---|---|---:|---:|
| 生产羊AI | 提供有压力但可玩的防守 | 是 | 否 |
| 平衡模拟代理 | 批量发现风险和反例 | 是 | 否 |
| 证明求解器 | 证明特定局面的强制结果 | 只有排序可用 | 仅完整覆盖时 |

理论上有限的棋类也可能需要多年搜索、残局数据库和大量计算才能求解。Checkers 的求解工作使用了专门搜索与残局数据库；这说明求解器应作为长期工程，而不是在Admin里增加一个“穷举”按钮就视为完成。

Fangrush 推荐采用递进路线：

1. 有限深度证明：寻找N层内强制捕食、困狼或僵局。
2. 特定残局表：对棋子较少或候选地图关键局面精确求解。
3. 离线长任务：对少数首发关尝试根节点证明，输出未知也属于有效结果。
4. AI对照：若已证明羊可短程困狼而生产AI错失，计为优势转化缺陷。

完美AI不直接进入主线。一个永不犯错且总能兑现理论结果的对手，可能适合研究或大师挑战，却会消除普通玩家的学习空间和胜利希望。

## 十七、商业化长期模型

商业化建立在“玩家愿意继续”之后，而不是用付费修补难度或内容不足。

### 独立站

- 继续作为无广告Demo、品牌、SEO和反馈入口。
- 不为短期广告收入破坏首局速度和产品可信度。

### H5门户

- 中插只请求在结算后、重新开始、进入下一关等自然中断点；是否实际展示和冷却由平台SDK决定。
- 僵局或失败出现时先完整解释结果，不立即盖广告；玩家点击“再来一局”形成继续意图后，平台可以处理自然广告机会。
- 激励广告必须由玩家主动选择，并提前说明确定回报；最适合双倍碎片、临时收藏加速和非胜负型外观进度。
- 不看广告、广告失败、无填充或广告拦截时，基础对局、重试、下一关、存档和基础奖励必须正常。

Poki将 `commercialBreak` 定义在关卡间、重开等自然停顿，将 `rewardedBreak` 定义为玩家主动选择换取明确奖励，并由平台决定是否真的展示广告。CrazyGames同样要求中插远离活跃玩法，并由平台控制约3分钟的冷却；其官方建议关注体验而不是广告数量。AdMob也将插屏定位在关卡间等自然转场。

### App

- 只有H5验证留存、会话和收藏需求后再投入。
- 收入组合优先为低频广告、去广告买断和纯外观IAP。
- 关卡包或高级挑战可以付费，但不能出售更高连吃次数、弱化羊AI或主线必需提示。

## 十八、数据与决策门槛

长期运营不能只看胜率和广告收入。建议建立五组指标：

### 核心体验

- 首局开始率、首次捕食率、首关完成率。
- 失败后重试率、通关后下一关点击率。
- 玩家能否区分移动、捕食、连吃、岩石和终局。

### 平衡质量

- 随机狼、策略狼、资深玩家之间的表现差异。
- 首次捕食、最长连吃、主动收手、困狼、重复和行动上限。
- AI明显送子率、短程封锁机会命中率和优势转化行动数。
- 理论证明状态与模拟结果是否冲突。

### 内容质量

- 相邻关地图辨识度、主要策略理解率和重复布局比例。
- 每关失败原因分布，而不只看通关率。
- 哪个季节带来继续率，哪个季节造成非学习型流失。

### 留存与收藏

- 会话长度、每次会话关卡数、D1/D7回访。
- 碎片用途理解、皮肤预览、兑换和装备率。
- 挑战模式、每周挑战和战绩分享参与度。

### 商业健康

- 激励广告主动选择率、完成率和广告后继续率。
- 中插后退出率、首个广告前游戏时长和每活跃用户广告数。
- 广告收入增长是否伴随留存、评分或会话下降。

任何调优都先写假设和成功/停止条件。例如“把秋季04改Hard”必须说明希望提高哪类玩家的压力、允许通关率下降多少、是否增加僵局和如何回滚。没有指标和回滚的改动不是调优，只是换一种猜测。

## 十九、长期执行顺序

### 上线前

1. 完成战略连吃/收手代理、拓扑差异矩阵和有限深度求解器。
2. 修复羊AI终局价值、最长连吃风险和优势转化。
3. 重跑24关并重做无法证明差异的重复地图。
4. 完成24关玩家确认和真实设备体验。
5. 保持目标8、连吃5和核心奖励边界稳定，除非新证据明确推翻。

### 首发后0至3个月

1. 观察首局、重试、下一关、关卡流失和广告后继续率。
2. 先修理解和异常，再调整个别地图或AI。
3. 验证通关后“挑战防守”和每周固定挑战，不立即扩张大量关卡。

### 数据稳定后

1. 为最受欢迎季节增加2至3张精通关。
2. 推出一个短挑战赛季和一套高质量收藏主题。
3. 评估App、去广告和外观IAP；不让App需求反向污染H5主线。

长期成功标准不是“地图、广告和系统越来越多”，而是核心规则稳定、地图问题持续新鲜、玩家能力可成长、内容可验证、商业化不损害信任。
