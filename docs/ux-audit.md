# Fangrush 前台体验体检

本体检只覆盖现有网站的页面、流程、布局、文案、交互、响应式和可用性，不包含后续资源生产 backlog。

## 首轮发现

- [x] 确认首页在 1280px 桌面视口下原来使用窄的 `max-w-lg` 容器，已改为宽桌面容器并保留手机端内边距。
- [x] 玩法页、章节页、关卡页和设置页统一使用桌面宽度约束。
- [ ] 检查 1024px、1280px、1440px 和 1920px 下的内容密度、空白比例和首屏层级。
- [ ] 检查 375px、390px、414px 手机宽度下的横向溢出、按钮拥挤和棋盘高度。
- [ ] 检查所有页面的返回、继续、开始、退出、重试和装备路径。
- [ ] 检查中英文文案、按钮命名、状态提示和错误提示的一致性。
- [ ] 检查键盘操作、焦点状态、图片替代文本、颜色对比度和减少动画模式。
- [ ] 检查慢加载、资源加载失败、刷新、返回浏览器和本地进度恢复。

## 用户场景矩阵

- [ ] 新用户：首页 -> 玩法 -> 第一关 -> 第一次走子。
- [ ] 直接开始：首页 -> 第一关 -> 胜利/失败 -> 重试。
- [ ] 中途离开：首页 -> 继续游戏 -> 原关卡。
- [ ] 皮肤用户：皮肤 -> 预览 -> 装备 -> 进入游戏。
- [ ] 移动用户：手机触控选狼、选择位置、连锁、结束回合。
- [ ] 设置用户：静音、减少动画、切换语言、返回游戏。
- [ ] 异常用户：刷新、返回、资源失败、无效操作、不可用关卡。

## 评级规则

- P0：无法开始、无法继续、状态丢失或出现严重误导。
- P1：核心流程明显困惑、操作容易失败或移动端不可用。
- P2：页面层级、文案、间距、视觉一致性或跳转体验问题。
- P3：细节润色和高级体验改进。
