# 06 · 生成方法论与脚本约定（可复用）

## 1. 美术流水线

```mermaid
flowchart LR
  brief[Brief禁止项与关键词]
  ref[定妆参考图]
  pick[选1套方向]
  make[描SVG或导出WebP]
  size[64与36目视]
  shot[对局截图验收]
  ship[过线才入仓]
  brief --> ref --> pick --> make --> size --> shot --> ship
```

1. **Brief**：写清「侧脸狼、蓬松羊、不规则岩；禁止圆头三角耳」。  
2. **定妆**：可用图像生成 2～4 张参考，**只作造型**。  
3. **生产稿**：手描 SVG 或透明底 WebP。  
4. **双尺寸**：64 好看不够，**36 必须可辨**。  
5. **进局截图**：与真 HUD 一起看，过线再替换 `public/skins/**`。  

反例打回：通用圆动物、羊狼同明度、岩石=圆角方、棋盘=灰线表。

## 2. 音效流水线

```mermaid
flowchart LR
  table[事件与时长表]
  proto[程序化或CC0裁切]
  loud[响度归一]
  ear[耳机试听]
  wire[接入sfx.ts]
  table --> proto --> loud --> ear --> wire
```

成品主声禁止电话蜂鸣感。

## 3. 脚本与目录 SSOT

| 用途 | 位置 |
|------|------|
| 资源生成/批处理（待建） | **`apps/web/scripts/assets/`** |
| 皮肤文件存在性 CI | `packages/game-core/scripts/check-skins.mjs`（`pnpm check:skins`） |
| 运行时皮肤 | `apps/web/public/skins/` |
| 品牌/OG | `apps/web/public/brand/`、`public/og.png` |
| 音效 | `apps/web/public/sfx/` |
| Catalog | `packages/game-core/src/content/skins.ts` |

**约定**：生成脚本放 `apps/web/scripts/assets/`（如 `gen-sfx.mjs`、`README.md` 说明怎么跑）；**不要**把一次性 Python 散落无文档。校验仍走 game-core `check:skins`。

```
产品：Fangrush / 三狼连猎
资产：default狼+羊 | frost狼+羊 | 岩石 | 棋盘{季节}

```
