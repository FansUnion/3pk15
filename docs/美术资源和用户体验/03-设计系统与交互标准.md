# 03 · 设计系统与交互标准

> **体验承诺**：这是牧场上的猎杀棋。棋盘是主角；玩家每一步都应看懂「选了谁、能去哪、吃了没有」。  
> 对局时序数值以 [`商业成功标准`](../商业成功/商业成功标准.md) 为准。  
> 仍弱项见 [`任务清单`](../任务/任务清单.md)（任务可见性、绘制层次等）。

## 体验承诺（可观察）

1. 打开对局：一眼是羊狼棋盘，不是后台表格。  
2. 选中狼：光环 + 微放大；可走柔光、可吃强环（含隔空中间点）。  
3. 落子 / 隔空吃：约 200ms 动画 + 音；连吃 HUD 显示 k/5。  
4. 羊回合：挡点击；顶栏能看出「轮到羊」。  
5. 胜负：先能读结果，再弹层；主按钮「再来」。

## 禁止（集中）

- 紫霓虹、赛博光效、Excel 灰线盘  
- 圆头两点眼三角耳的贴纸羊/狼  
- 对局底栏三块等高空心大框（表单感）  
- 铬层（顶栏底栏说明）压过棋盘  
- 对局中部硬推广告

## 1. 色彩 Token（意图）

| Token | 用途 | 要求 |
|-------|------|------|
| `--surface` | 页底 | 暖纸/浅草，对比够，禁一片灰绿雾 |
| `--board` | 棋盘场 | 有纹理；四季变色温 |
| `--ink` | 主文 | 炭/墨绿，对比达标 |
| `--muted` | 次文 | 可读不融底 |
| `--accent` | 主 CTA / 选中 | **一块够跳的实色** |
| `--danger` | 可吃提示 | 与草地可分的红系 |
| 羊/狼识别 | 子力 | 羊亮狼暗，色相分离；冬不把羊洗成灰 |

## 2. 字体

- Display（Literata 等）：品牌、关卡大标题、结算  
- UI Sans + 中文 fallback（PingFang / YaHei / Noto）  
- HUD 数字：`tabular-nums` + 略重  

## 3. 控件

- 对局底栏：图标工具条或浅实心  
- 列表卡：轻浮起或左边色条  
- 触控热区 ≥ 44px，视觉可更瘦  

## 4. 动效预算

| 类型 | 时长 |
|------|------|
| 落子/吃子 | ~200ms |
| 羊思考空窗 | ~600ms |
| 按钮按下 | ~80–120ms scale 0.97 |
| `prefers-reduced-motion` | 缩短或取消位移，保留状态变化 |

## 5. 对局信息层级

```
┌──────────────────────────────┐
│ ←  关卡名               ···  │  薄
│ ┌ 吃0/8 │ 羊15 │ ●狼回合 ┐  │  浮起；数字重
├──────────────────────────────┤
│     棋盘 ≥70% 视高            │
├──────────────────────────────┤
│ （微提示仅新手）               │
│  ↺重置   🔊静音   ✕退出      │  轻于棋盘
└──────────────────────────────┘
```

## 6. 全交互标准（摘要）

| 交互 | 目标 |
|------|------|
| 选中狼 | 光环 + 阴影 + 微放大 |
| 可走/可吃 | 柔光圆；羊位强环 + through |
| 落子/吃 | 200ms 事件 + 音 |
| 连吃 HUD | k/5 变色加重 |
| 羊回合 | 静默挡点；顶栏色点「羊回合」 |
| 胜负结算 | 先可读再弹层；主按钮「再来」 |
| 重置 | 二次确认 + 危险色 |
| 静音 | 全局真静 |
| 引导春日1 | 短、可跳过；勿与常驻说明重复 |
| 广告 | 结算/关卡缝；失败可跳过 |

## 7. 官网壳

- 首屏：盘面海报主角 + 一主 CTA  
- 次入口降对比  
- 语言下拉不抢戏  
- 页脚弱 Admin  

## 8. 无障碍与性能底线

- 正文对比可读；焦点环可见  
- 棋子/底板控制体积，不挡首屏 LCP  
- 空态：无进度/无任务有短句引导去开玩  
- 弱网：资源失败回退默认皮，不白屏死  

## 9. 广告缝（不伤爽）

- 插屏：结算后  
- 激励：结算或关卡缝；仅 ok 发奖  
- 对局中部不硬推双倍
