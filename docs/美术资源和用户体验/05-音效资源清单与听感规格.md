# 05 · 音效资源清单与听感规格

> **体验承诺**：普通走子轻、隔空吃与连吃有层次、胜负有情绪但不吵；静音键必须可信。  
> 代码入口：[`apps/web/src/lib/sfx.ts`](../../apps/web/src/lib/sfx.ts)。以代码事件名为准。

## 1. 事件表

| 事件（sfx.ts） | 文件 | 听感隐喻 | 时长预算 | 忌 |
|----------------|------|----------|----------|-----|
| step | `public/sfx/step.wav` | 轻木棋「笃」 | < 120ms | 手机按键音 |
| jump | `public/sfx/capture.wav` | 短 whoosh + 轻咔（隔空吃） | < 200ms | 枪声/血腥 |
| chain | `public/sfx/chain.wav` | 同族音高递进 | < 200ms | 过长旋律 |
| win | `public/sfx/win.wav` | 短上行解决 | < 500ms | 长 fanfare |
| lose | `public/sfx/lose.wav` | 短下行 | < 400ms | 刺耳报警 |
| select | （无 wav） | 程序化轻音 | — | 抢对局声 |
| invalid | （无 wav） | 程序化提示 | — | 刺耳 |
| ai | （无 wav） | 程序化弱提示 | — | 像转圈 loading |

## 2. 契约

- `settings.muted === true` → **完全无声**  
- 有文件优先；`select/invalid/ai` 与缺文件时走程序化兜底，**成品主声不得只剩蜂鸣感**  
- 首次用户手势后再解锁 AudioContext  
- 与画面 ~200ms 反馈大致对齐  

## 3. 层次（可观察）

| 层 | 例子 | 期望 |
|----|------|------|
| 操作 | step / select | 轻、短、可连点不烦 |
| 规则反馈 | jump / invalid | 比操作更醒目一层 |
| 情绪 | chain / win / lose | 连吃递进；胜负短促有收束 |

## 4. 听感过线

换音后：戴耳机走一遍 step → jump → chain → win/lose；确认静音键全局静音。
