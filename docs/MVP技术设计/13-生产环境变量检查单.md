# 生产环境变量检查单（独立站 / 门户）

部署前逐项确认。模板：仓库根 [`.env.example`](../../.env.example)；Next 本地还会读 `apps/web/.env.local`。

## 独立站（standalone）· Vercel 粘贴清单

在 Vercel → Project → Settings → Environment Variables（Production）写入：

```bash
NEXT_PUBLIC_SITE_URL=https://fangrush.com
NEXT_PUBLIC_APP_SHELL=standalone
NEXT_PUBLIC_ADS_PROVIDER=mock
ADMIN_ENABLED=true
ADMIN_ACCESS_KEY=请换成强口令
NEXT_PUBLIC_TELEMETRY=0
```

| 变量 | 建议 | 说明 |
|------|------|------|
| `NEXT_PUBLIC_SITE_URL` | `https://fangrush.com` | `metadataBase`；本地用 `http://localhost:5000` |
| `NEXT_PUBLIC_APP_SHELL` | `standalone` | 含 `/admin`（仍受 ADMIN_ENABLED 控制） |
| `ADMIN_ENABLED` | 公网建议 `true` + 密钥 | `false` 则后台 404 |
| `ADMIN_ACCESS_KEY` | **非空强口令** | 公网必填 |
| `NEXT_PUBLIC_ADS_PROVIDER` | 先 `mock`；有号后 `adsense` | 见 `src/lib/ads.ts` |
| `NEXT_PUBLIC_TELEMETRY` | `0` | 默认关自建遥测 |
| `NEXT_PUBLIC_FAIL_ADS` | **勿设** | 仅本地调试广告失败 |

## Vercel 项目设置（monorepo）

推荐：

1. **Root Directory**：`apps/web`  
2. **Install Command**：`cd ../.. && pnpm install`  
3. **Build Command**：`cd ../.. && pnpm --filter @wolf-sheep/web build`  
4. **域名**：Add `fangrush.com`（及可选 `www`）→ 按提示在 Cloudflare DNS 加记录  

## 门户瘦包（portal）

| 变量 | 建议 | 说明 |
|------|------|------|
| `NEXT_PUBLIC_APP_SHELL` | `portal` | middleware 屏蔽 `/admin` |
| `ADMIN_ENABLED` | `false` | 双保险 |
| `NEXT_PUBLIC_ADS_PROVIDER` | `portal_sdk`（或暂 `mock`） | `PortalAds` 待接 JSBridge |
| `NEXT_PUBLIC_TELEMETRY` | `0` | 零自建远程遥测 |
| `NEXT_PUBLIC_SITE_URL` | 门户页正式 URL（有则填） | 可选 |

构建：`pnpm build:portal`

## 上线后冒烟（人工）

1. `https://fangrush.com` 首页 → 章节 → 春日可胜可败  
2. 刷新仍有进度；清站点数据后进度清空  
3. 设置 → 隐私说明可开  
4. 独立站：`/admin` 有钥可进  
5. 勿在生产打开 `NEXT_PUBLIC_FAIL_ADS`
