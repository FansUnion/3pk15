# 09 · 工程规范与测试策略

## 1. 仓库与包

| 包 | 职责 |
|----|------|
| `packages/game-core` | 规则、FSM、AI、关卡/皮肤/掉落只读配置 |
| `apps/web` | Next 壳、UI、adapters、admin |
| `content/skins` | SVG 资源（可被 web 引用） |

- 包管理：**pnpm** workspace。  
- `game-core` **禁止**依赖：`react`、`next`、`dom`、广告 SDK。

---

## 2. TypeScript

- `strict: true`。  
- 公开 API 显式导出类型；避免 `any`（测试夹具除外且需注释）。  
- 坐标统一 `Pos = { r: 1|2|3|4|5|6; c: 1|2|3|4|5|6 }`（或 number + 运行时断言）。

---

## 3. 代码分层禁令

| 禁止 | 原因 |
|------|------|
| 在 React 组件内写隔空吃/胜负判定 | 无法单测、双端分叉 |

| admin 复制一套 AI | 与线上行为漂移 |
| core 读 `localStorage` / `window` | 破坏纯函数与测 |
| 无 fixture 改规则合入 | 回归失控 |

---

## 4. 测试金字塔

### 4.1 单元（Vitest，`game-core`）— **必须**

| 模块 | 覆盖要点 |
|------|----------|
| `rules` | 走子、隔空吃、连吃≤5、岩石挡、羊禁后退、胜≥8、败无步 |
| `fsm` | 状态迁移；非法 action 拒绝；`endWolfTurn` |
| `ai/easy` | 输出合法羊步；种子可复现 |
| `ai/normal` | 偏好抱团/堵口的可测断言（弱断言 + fixture） |
| `ai/hard` | 对已知残局选出合围/锁死步（fixture） |
| `content` | 岩石不相邻；档位与章绑定；皮肤文件存在 |

**门槛建议**：rules+fsm 行覆盖 ≥ 85%；AI 以 fixture 通过为主，不强求行覆盖。

### 4.2 集成（可选轻量）

- `playLevel` 脚本：加载某关 → 人机脚本序列 → 期望终局。  
- Admin 模拟：同 core 调用，无浏览器。

### 4.3 E2E（后期）

- Playwright 冒烟：首页 → 春日 1 → 走一步。MVP 可人工验收替代。

---

## 5. Fixture 规范

```
packages/game-core/tests/fixtures/
  jump-capture-basic.json
  chain-eat-cap-5.json
  sheep-no-retreat.json
  hard-surround-mate.json
```

格式建议：

```json
{
  "name": "chain-eat-cap-5",
  "board": { "...": "序列化局面" },
  "actions": ["..."],
  "expect": { "eaten": 5, "phase": "sheepTurn" }
}
```

AI fixture：固定 `seed` + 局面 → `expectMove` 或 `expectProperty`（如「落点使狼合法步数减少」）。

---

## 6. CI

每次 PR：

1. `pnpm lint`  
2. `pnpm --filter game-core test`  
3. `pnpm check:skins`（资源存在性）  
4. `pnpm --filter web build`（含 `PORTAL=1` 可选 job）

规则或 AI 改动：**必须**附带/更新 fixture。

---

## 7. Git / 分支（建议）

- `main` 可部署独立站。  
- 功能分支短生命周期。  
- 提交信息：中英均可，说清「为何」。

---

## 8. 性能预算（棋盘）

- 单次 `listLegalMoves` / `pickSheepAction`：目标 &lt; 16ms（中端手机）。  
- hard AI 若超：降深度或加缓存（见 05）；**禁止**为性能把 AI 挪服务端。

---

## 9. 日志

- 玩家侧：默认无 console 噪音；`DEBUG_GAME=1` 可打 action 流。  
- Admin 模拟台：可开详细逐步日志（仅 admin）。  
- 不上报对局内容到自建服务器（合规见 11）。
