# 10 · 简易后台设计（独立站 `/admin`）

> 服务对象：开发者 / 运营自测。**不是**玩家功能；门户构建必须剔除。  
> 与功能清单附录「不做 AI 行为日志后台 / 广告报表」不冲突：本后台是 **配置预览 + AI 模拟台**，无持久化运营报表。

---

## 1. 目标与非目标

| 做 | 不做 |
|----|------|
| 只读浏览皮肤 Catalog、章节关卡配置 | 在线改配置写回仓库/数据库 |
| AI 模拟台：选档、摆残局、逐步/自动跑羊 | 玩家账号体系、权限 RBAC |
| 玩法说明 Markdown 预览 | 广告曝光报表、付费后台 |
| 可选访问密钥 | 本期强制登录（可二期加） |

---

## 2. 访问控制

```
ADMIN_ENABLED=true
ADMIN_ACCESS_KEY=  （可选）
```

| 模式 | 行为 |
|------|------|
| `ADMIN_ENABLED=false` | `/admin` → 404 |
| 仅 enabled | 可进；适合本地与私有预览 URL |
| enabled + key | 进站前输入密钥；存 `sessionStorage`；错则拒 |

中间件或 layout 校验；**密钥只比对常量**，不建用户表。

门户：`ADMIN_ENABLED=false` 且不编译 admin 路由。

---

## 3. 信息架构

| 路径 | 页面 | 说明 |
|------|------|------|
| `/admin` | 首页 | 入口卡片：皮肤 / 关卡 / AI / 说明 |
| `/admin/skins` | 皮肤台 | 列表 + 大图预览狼/羊/棋盘 |
| `/admin/levels` | 关卡台 | 章节表；点关看岩石布局示意 |
| `/admin/ai` | AI 模拟台 | **核心**；见 §5 |
| `/admin/docs` | 玩法说明 | 渲染内置 Markdown（见 12） |

导航：顶栏回玩家首页（新标签）+ 退出密钥会话。

---

## 4. 皮肤台 `/admin/skins`

**数据源**：与线上同一 `SkinCatalog` + `content/skins`。

功能：

1. 表格：id / kind / unlock / 资源路径 / 校验状态（绿/红）。  
2. 点选：并排预览 `wolf` + `sheep`（wolf_set）或 board。  
3. 「校验全部」：复用 `check:skins` 逻辑，缺文件标红。

不提供上传；改资源走 Git。

---

## 5. AI 模拟台 `/admin/ai`（技术要点）

### 5.1 为何存在

- 调参评价函数、验证 hard 合围、复现坏局，无需反复打完整关卡。  
- **必须**调用 `packages/game-core` 的 `pickSheepAction` / `applyAction`，与玩家对局同一二进制逻辑。

### 5.2 UI 布局

```
┌──────────────┬────────────────────────────┬─────────────┐
│ 控制面板      │  SVG 棋盘（可点击摆子）      │ 日志 / 分数  │
│ 档位 easy…   │                            │             │
│ 关卡模板加载  │                            │             │
│ 种子 seed    │                            │             │
│ 单步羊 / 连跑 │                            │             │
│ 导出局面 JSON │                            │             │
└──────────────┴────────────────────────────┴─────────────┘
```

### 5.3 控制项（详细）

| 控件 | 行为 |
|------|------|
| AI 档位 | `easy` / `normal` / `hard` → 传入 `pickSheepAction(state, tier, rng)` |
| 加载关卡 | 选 `LevelConfig`：岩石 + 标准开局，再可手动改 |
| 清空 / 标准开局 | 重置 |
| 摆子模式 | 点交点循环：空→狼→羊→岩→空；强制 ≤3 狼、羊数可超开局以便残局 |
| 合法约束开关 | 「严格模式」开：禁止非法重叠；关：调试用 |
| Seed | 固定 RNG，复现 easy/采样 |
| 轮到 | 强制 `phase`：wolfTurn / sheepTurn / wolfChain |
| **羊单步** | 调 AI 一步并 apply；日志打印选中 move + evaluate 分项（若 hard） |
| **自动跑 N 步** | 循环羊步或「狼随机合法 + 羊 AI」对打 |
| **导出 / 导入 JSON** | 局面序列化，便于贴进 fixture |
| 评价面板 | 显示当前 `evaluate(state)` 总分及分项（normal/hard 调试用） |

### 5.4 技术约束

1. 模拟状态在 React state / zustand；**默认不写** `SaveGame`。  
2. 自动跑设上限（如 200 步）防死循环；超时提示。  
3. hard 搜索若卡 UI：`requestIdleCallback` 或 async chunk +「计算中」；禁止阻塞到浏览器无响应。  
4. 日志可复制；格式：`[sheep] (3,2)->(3,3) score=... reason=...`。

### 5.5 与 05 文档对齐

- 分项权重、搜索深度改动后，在本台用同一 fixture 局面肉眼确认，再固化单测。  
- easy 的随机：改 seed 应改变抽样；同 seed 应一致。

## 6. 关卡台 `/admin/levels`

- 树：春夏秋冬 → 关卡列表。  
- 右侧：6×6 示意（岩石黑点、狼羊开局位）。  
- 展示：`ai` 档、岩石数、首次/重复掉落摘要。  
- 校验徽章：相邻岩石、越界 → 红（读 content 校验结果）。

不可编辑保存；改 `levels.ts` 后热刷新查看。

---

## 7. 与 CI 的关系

- Admin 无单独 E2E 必过项。  
- 建议：`game-core` fixture 中至少 2 个来自「模拟台导出」的真实坏局。

---

## 8. 安全注意

- 预览 URL 勿长期挂公网无密钥。  
- 密钥放 Vercel env，不进仓库。  
- Admin 不含玩家存档读写（避免误清）。
