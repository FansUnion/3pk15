# 04 · 规则引擎与对局状态机

> 玩法口径以 [`docs/游戏创意/02-核心对局规则.md`](../游戏创意/02-核心对局规则.md) 为准。  
> 本文定义**可实现、可单测**的技术规格。UI 与 AI **只许**调用本文规定的 API，不得自行判定吃子。

对应功能 ID：A01–A18（见 `01-MVP完整功能清单.md`）。

---

## 1. 坐标系与棋盘

- 点：`Pos = { r: 1..6, c: 1..6 }`，共 36 交点。  
- 邻接：上下左右，`|dr|+|dc|=1`，禁止斜向。  
- 岩石：`Set<PosKey>`，`PosKey = `${r},${c}``。岩石点不可占用、不可穿越。  
- 棋子：狼 3、羊最多 15；任意时刻同一点至多一个棋子，且不与岩石重叠。

### 开局（与创意 03 一致）

羊：`(1..3, 1..5)`；狼：`(6,2)(6,3)(6,5)`。

## 2. 核心数据类型（逻辑模型）

```ts
type Side = 'wolf' | 'sheep'
type Difficulty = 'easy' | 'normal' | 'hard'

type Piece = { id: string; side: Side; r: number; c: number }

type BoardState = {
  pieces: Piece[]
  rocks: Set<string>          // PosKey
  eatenSheep: number          // 本局累计，0..15
  toMove: Side
  // 连吃上下文：仅 wolf 回合且刚完成一次隔空吃后可能非空
  chain: null | { wolfId: string; count: number } // count = 已完成隔空吃次数 1..5
  status: 'playing' | 'won' | 'lost'
  levelId: string
}

type StepMove = {
  type: 'step'
  pieceId: string
  to: Pos
}

type JumpMove = {
  type: 'jump'
  pieceId: string
  through: Pos   // 中间空点（无子无岩）
  to: Pos        // 目标羊位兼落点
}

type Action = StepMove | JumpMove
```

**不变量（每次 `applyAction` 后必须成立，单测可 `assertInvariants`）**

1. 所有 piece 坐标在 1..6。  
2. 无两子同点；无子与岩石同点。  
3. `eatenSheep` = 15 - 场上羊数（开局 15，被吃递减场上）。  
4. `chain !== null` 仅当 `toMove==='wolf'` 且上一手为该狼的 jump。  
5. `chain.count` ∈ 1..5。  
6. `status!=='playing'` 时不再接受对局着法（可接受 reset）。

## 4. 着法生成细节

### 4.1 狼 · step

对狼所在点四邻：目标在盘内、无岩石、无棋子 → `step`。

### 4.2 狼 · jump（隔空吃）

对四个方向 `d`：

- `through = pos + d`  
- `to = pos + 2d`  
- `through` 在盘内、**无岩石、无棋子**  
- `to` 在盘内且有**羊**  
- 严格曼哈顿距离 2、同线；紧贴不可吃  

生成 `jump{ through, to }`。

### 4.3 连吃

`apply` 一次 `jump` 后：

1. 移除 `to` 处羊；狼移到 `to`；`eatenSheep++`。  
2. 若 `eatenSheep >= 8` → `status=won`，清 chain，结束。  
3. 否则令 `newCount = (chain?.count ?? 0) + 1`。  
4. 若 `newCount >= 5` → 清 chain，`toMove='sheep'`。  
5. 否则临时 `chain={wolfId, count:newCount}`，用**新状态**再算该狼 `list jumps`：  
   - 若仍有 jump → 保持 `toMove='wolf'` + chain（玩家可继续或——产品口径：不强制有吃必吃，允许「主动结束连吃」吗？）  

**产品澄清（技术写死）：**  
创意：「不强制有吃必吃」针对开手；连吃中若存在 jump，**允许玩家选择结束回合**（UI 提供「结束连吃」），对应 API：

```ts
function endWolfTurn(state: BoardState): BoardState
// 仅当 toMove==='wolf'；清 chain；toMove='sheep'；然后若三狼全无着且下一手是狼则已在羊走完后检测
```

若连吃中无剩余 jump → 自动 `endWolfTurn`。

### 4.4 羊 · step

四邻中：

- 禁止 `to.r < piece.r`（向第 1 行方向后退）。  
- 允许 `to.r > piece.r`（向第 6 行）或 `to.r === piece.r`（横移）。  
- 目标空且无岩石。

### 4.5 羊方回合粒度

**每回合只走动一只羊**（一个 `step`）。然后 `toMove='wolf'`，并检测三狼是否全无合法着 → `lost`。

## 5. 胜负检测时机

| 时机 | 检测 |
|------|------|
| 任意 jump 后 | `eatenSheep>=8` → won |
| 狼回合结束切到羊之前 | （一般不需） |
| 羊一步完成后 | 若所有狼 `listLegalActions` 在「假想 toMove=wolf, chain=null」下为空 → lost |
| 狼回合开始时（chain=null） | 若当前狼方合法着为空 → lost |

实现建议：集中函数 `refreshStatus(state)`。

## 6. 对局 UI 状态机（与 rules 配合）

逻辑层 `BoardState` 之上，UI 会话：

```
IdleSelectWolf
  → (选中狼) ShowMoves
  → (点 step) Animating → 应用 → AiTurn 或 Terminal
  → (点 jump) Animating → 应用 → ChainSelect | AiTurn | Terminal
ChainSelect
  → (点 jump) Animating → …
  → (结束连吃) AiTurn
AiTurn
  → (AI 返回 action) Animating → … → IdleSelectWolf | Terminal
Terminal
  → 结算 UI
```

守卫：Animating 忽略点击；AiTurn 忽略棋盘点击。
