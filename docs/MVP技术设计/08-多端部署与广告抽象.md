# 08 · 多端部署与广告抽象

## 1. 三端目标

| 端 | 构建产物 | 含 `/admin` | 广告 | 遥测 |
|----|----------|-------------|------|------|
| **独立站**（P0） | `apps/web` 全量 Next | ✅ | Adapter → 真实/Mock SDK | 可后接匿名统计；MVP 可关 |
| **门户 H5**（P2） | 同仓 `PORTAL=1` 瘦包 | ❌ 路由剔除 | Adapter → 门户 SDK 或 Mock | **默认零自建远程遥测** |
| **App**（预留） | Capacitor 包 WebView | ❌ | Adapter → 原生广告桥 | 另议 |

**原则**：`game-core` 与玩家 UI 共用；差异全部进 Adapter，禁止 `if (portal)` 散落在规则层。

| 默认 Storage | localStorage | localStorage（注意分区） |

## 3. IAds 端口（必实现）

```ts
type AdResult = { ok: true } | { ok: false; reason: 'cancelled' | 'failed' | 'unavailable' }

interface IAds {
  /** 结算插屏；失败也必须 resolve，由调用方继续 UI */
  showInterstitial(): Promise<AdResult>
  /** 激励视频；仅 ok 时发奖 */
  showRewarded(placement: 'fragment_topup' | 'double_drop'): Promise<AdResult>
  /** 预加载（可选） */
  preload?(): void
}
```

### 产品规则（与创意 09 对齐）

| 场景 | 行为 |
|------|------|
| 胜/负结算 | `showInterstitial` → 不论 ok/fail → 展示奖励/再来 |
| 补碎片 | `showRewarded('fragment_topup')` → **仅 ok** 加通用碎片 |
| 双倍 30min | `showRewarded('double_drop')` → **仅 ok** 写 `buffs.doubleDropUntil` |
| SDK 未就绪 | 立即 `{ ok: false, reason: 'unavailable' }`，不阻塞 |

### MockAds（开发与 CI）

- `FAIL_ADS=1`：全部 fail。  
- 默认：instant resolve ok（可配置延迟模拟加载）。  
验收：F04 必须用 Mock 失败路径跑通。

### 真实接入（独立站上线前）

- 选一家合规 Web 广告（如 AdSense / 第三方 H5 联盟）；封装进 `AdsenseAds` 实现同一接口。  
- **禁止**在结算组件直接调第三方全局对象。

### 门户

- 实现 `PortalAds`：按门户文档调 JSBridge；无文档时 MVP 可 Mock，上架前替换。
