# 06 · 关卡配置与存档数据模型

## 1. 关卡配置（只读，进仓库）

```ts
type AiTier = 'easy' | 'normal' | 'hard'
type ChapterId = 'spring' | 'summer' | 'autumn' | 'winter'

type LevelConfig = {
  id: string                 // e.g. spring-01
  chapterId: ChapterId
  indexInChapter: number     // 1-based
  name: string
  rocks: { r: number; c: number }[]
  ai: AiTier
  /** 首次通关奖励 */
  firstClearReward: { universal?: number; season?: Partial<Record<ChapterId, number>> }
  /** 重复通关：概率掉落 */
  repeatDrop?: { chance: number; universal?: number; season?: Partial<Record<ChapterId, number>> }
}
```

### 约束校验（构建或单测）

- `rocks` 点在 1..6；不重叠；**两两不相邻**（四邻意义）；不占开局狼/羊点（或文档允许占空点——定稿：岩石不占开局子位）。  
- 春日三关岩石数 0/1/2；夏 2–4；秋 6–8；冬 0。  
- `ai` 与章节绑定：spring→easy；summer/autumn→normal；winter→hard。

### 章节解锁

- `spring` 默认解锁。  
- 解锁下一章 ⇔ 本章全部 `level.id` 均在存档 `clearedLevels` 中。

---

## 2. 掉落表

可放在 level 内或独立 `drops.ts`。数值首版建议：

| 事件 | 通用碎片 | 说明 |
|------|----------|------|
| 首次通关 | +10 | 可再加少量季节碎片 |
| 重复通关 | 30% 概率 +2 | 可配置 |
| 双倍 buff 生效中 | 上述 ×2 | 见广告设计 |

精确数可调；改动只动配置。

## 3. 存档 SaveGame

```ts
type SaveGame = {
  schemaVersion: 1
  clearedLevels: string[]
  unlockedChapters: ChapterId[]
  fragments: {
    universal: number
    season: Record<ChapterId, number>
  }
  unlockedSkinIds: string[]
  equipped: {
    wolfSetId: string
    boardId: string
  }
  guide: { spring1Done: boolean }
  buffs: {
    doubleDropUntil: number | null  // epoch ms
  }
  lastPlayedLevelId?: string
}
```

### 存储

- Adapter：`IStorage.get/set('wolf-sheep-save-v1', json)`  
- 独立站 / 门户：localStorage（注意门户存储分区）。  
- 读写失败：回退默认存档并打日志。

### 迁移

```ts
function migrate(raw: unknown): SaveGame
// schemaVersion 缺失 → 当 v0 空档升级到 v1
// 未来 v2：写 migrateV1toV2，禁止静默丢字段
```

### 默认存档

- 解锁 spring；皮肤 `wolf-default`、`board-spring`（或 board-default）；碎片全 0；guide false。

## 4. 与后台关系

后台 **不写** SaveGame；只读 LevelConfig / SkinCatalog。模拟对局结果默认不落盘（可可选下载 JSON）。
