# 05 · AI 三档设计与实现规格（专册）

> 行为目标来源：[`docs/游戏创意/06-AI行为设计规格.md`](../游戏创意/06-AI行为设计规格.md)、[`05-四季关卡设计.md`](../游戏创意/05-四季关卡设计.md)。  
> 本文是 **实现唯一规格**：算法、接口、评价函数、搜索、降级、模拟台、测试与校准。  
> AI **只许**消费 `listLegalActions` / `applyAction`（见文档 04），禁止在 AI 内复制吃子几何。

对应功能 ID：B01–B06。

---

## 1. 设计目标

| 档位 `Difficulty` | 章节 | 设计意图 | 目标狼胜率（校准用，非运行时作弊） |
|-------------------|------|----------|-----------------------------------|
| `easy` | 春日 | 弱、可学规则 | 65%–75% |
| `normal` | 夏日、秋日 | 会堵会抱团 | 夏 45%–55%；秋靠岩石到 70%–80% |
| `hard` | 冬日 | 强力合围 | 35%–40% |

**禁止**：根据玩家历史胜率动态改 AI；禁止深度学习模型；禁止服务器算步。

**回合**：羊方每次调用 AI 只选出 **一个** `step` 着法。

// 前置：state.toMove==='sheep' && state.status==='playing'

## 3. 共用工具函数

### 3.1 克隆与试着

```ts
function after(state: BoardState, action: Action): BoardState
// = applyAction(clone(state), action) 且 refreshStatus
```

### 3.2 威胁分析（标准/大师用）

在「假想狼回合、`chain=null`」视图下：

```ts
function wolfThreatMap(state: BoardState): {
  totalWolfMoves: number      // 所有狼 step+jump 数之和
  jumpTargets: PosKey[]       // 狼 jump 的 to 位置（危羊）
  vulnerableSheepIds: string[]
  mobilityByWolf: Record<string, number>
}
```

实现：临时把 `toMove` 视为 wolf、清 chain，对每只狼跑着法生成（只读，不改原状态）。

### 3.3 送吃检测

羊走到 `to` 后，若下一狼回合存在 jump 的 `to===` 该羊新位置（该羊被吃），则记 `isSacrificial=true`。

## 4. 评价函数（标准档主用；大师档作叶节点评估）

定义标量 `score`：**越大对羊越有利**。

| 项 | 符号 | 定义 | 初值权重 `w` | 说明 |
|----|------|------|--------------|------|
| 狼机动性惩罚 | `M` | `totalWolfMoves` | **-3.0** | 围死方向 |
| 暴露吃点惩罚 | `J` | `jumpTargets.length`（危羊数） | **-8.0** | 堵吃优先于单纯减机动 |
| 横连奖励 | `H` | 同排相邻羊对数（四邻横边） | **+1.2** | 抱团/防线 |
| 中线控制 | `C` | 羊在 r=2,3,4 的数量 | **+0.4** | 轻量 |
| 送吃惩罚 | `S` | 若本步 isSacrificial 则 1 否则 0 | **-15.0** | 强烈避免主动送吃 |
| 距狼曼哈顿 | `D` | 可选：羊到最近狼距离之和 | **+0.05** | 微弱；防过度贴脸送吃 |

```
score = wM*M + wJ*J + wH*H + wC*C + wS*S + wD*D
```

**校准规则**：只允许改权重表；不允许改胜负规则。校准记录写入本节修订历史或单独 `ai-tuning-log.md`（可选）。

终局叶：若 `after` 后 `lost`（狼被困）→ 羊胜，返回 **+1e6**；若 `won`（狼吃满 8）→ **-1e6**。

## 5. 简易档 `easy`

### 算法

1. `actions = listLegalActions(state)`  
2. 若空则异常  
3. `i = floor(rng.nextFloat() * actions.length)`  
4. 返回 `actions[i]`

### 要点

- **不调用** `wolfThreatMap` / 评价函数。  
- 不过滤送吃。  
- 复杂度 O(合法着法数)。

### 验收

- 固定种子序列，黄金着法快照测试。  
- 春日关人工试玩观感：会乱走、会送吃，可接受。

---

## 6. 标准档 `normal`

### 算法（单层贪心）

1. `actions = listLegalActions(state)`  
2. 对每个 `a`：  
   - `s1 = after(state, a)`  
   - 若 `s1.status==='lost'`（对狼而言羊赢）→ 高分直接可选  
   - `sc = evaluate(s1) + (isSacrificial(state,a) ? wS : 0)`（送吃也可并入 evaluate(s1)）  
3. 取 `sc` 最大；并列者用 `rng` 均匀选一个。  
4. 返回该 action。

### 要点

- **不搜索狼的回应树**（除 evaluate 内假想狼着法计数）。  
- 必须体现：堵 `jumpTargets`、降 `totalWolfMoves`、避免送吃。  
- 秋日与夏日 **同一套权重**；难度差交给岩石配置。

### 验收

- 构造「羊横移一步即可挡住唯一 jump」局面 → normal 必须选挡。  
- 构造「两步等价」→ 多次种子下分布合理。  
- 夏日目标胜率人工/模拟粗测落在 45%–55% 区间附近（允许首版偏差，迭代权重）。

## 7. 大师档 `hard`

### 7.1 思想

在 normal 评价基础上，增加 **极浅对抗搜索**：

- 根节点：羊方选一步。  
- 后继：对每个候选羊着，生成状态 `s1`，再枚举 **狼方一步**（`listLegalActions` 在狼回合；若狼可连吃，搜索简化策略见下）。  
- 狼方一步后状态 `s2`，对 `s2` 用 `evaluate`（羊视角）。  
- 羊选使「狼方最优回应后的评价」仍最好的着（minimax：羊 max，狼 min）。

### 7.2 连吃简化（必须写清，避免搜索爆炸）

狼方模拟时若存在 jump：

- **策略 H1（推荐首版）**：狼方节点只展开 **一步 jump 或一步 step**（不在搜索内展开完整 5 连吃）。评价依赖 `evaluate` 对威胁的敏感。  
- 日后若 CPU 余裕，可加 H2：对 jump 后强制再展开有限链（最多再 1 次 jump）。

### 7.3 候选剪枝

根节点若 `actions.length > K`（建议 **K=24**）：先用 normal 的 evaluate 对每个着法打分，只保留 top-K 再搜索。

### 7.4 预算与降级

默认 `budgets = { maxNodes: 8000, maxMs: 40 }`（浏览器主线程；可配置）。

```
nodes = 0
start = now()
每个 after / listLegal 计节点
若 nodes>maxNodes 或 now()-start>maxMs：
  立即中止搜索，返回 normal.pick(state, rng)  // 降级
  并打日志 flag degraded=true（后台模拟台可见）
```

### 7.5 伪流程

```
function pickHard(state, rng, budgets):
  actions = prune(listLegal(state), K, evaluate)
  bestScore = -inf
  best = []
  for a in actions:
    s1 = after(state, a)
    if s1.status == lost: return a   // 羊直接困死三狼（罕见）
    if s1.status == won: score = -1e6; goto record
    // 狼回应
    wolfActs = listLegal(asWolfView(s1))
    if wolfActs empty: return a      // 已困死
    minScore = +inf
    for wa in wolfActs:             // 可再 prune
      if overBudget: return pickNormal(...)
      s2 = after(s1, wa)
      minScore = min(minScore, evaluate(s2))
    record: 更新 bestScore / best 列表
  return rngChoice(best)
```

### 7.6 验收

- 冬日空板：hard 相对 normal 应更常封堵逃脱格（用例局面集）。  
- 超时降级可测：把 `maxMs=0` 强制走 normal。  
- 目标胜率 35%–40%：通过后台批量模拟（见 §9）调权重/预算，不改规则。

## 9. AI 模拟台（与文档 10 共用规格）

输入：

- `levelId` 或自定义 `rocks[]`  
- `difficulty`  
- `seed`  
- `maxPlies`（防死循环，建议 400）  
- 模式：`auto` 整局 | `step` 逐步  

流程：

1. `createInitialState`  
2. 循环：若 wolf 回合，可用 **狼方简单启发式或脚本狼**（后台可选「随机狼 / 贪心狼」）以便测羊 AI；默认 **贪心狼**（每步最大化 eaten 或 jump 优先）用于压测羊。  
3. 羊回合：`pickSheepAction`  
4. 直至 terminal 或 maxPlies  

输出：胜负、步数、eatenSheep、是否 degraded、着法列表（可选下载 JSON）。

**批量**：N 局同关卡同难度不同 seed → 统计狼胜率，对照创意目标区间。
