## Video ads · 视频广告

**English:** Two types: **`midgame`** and **`rewarded`**. Follow advertisement requirements for placement and frequency.

**中文：** 两类：**`midgame`（中插）** 与 **`rewarded`（激励）**。展示时机与频次须遵守广告要求页。

### Request API

```javascript
const callbacks = {
  adFinished: () => { /* resume */ },
  adError: (error) => { /* resume; no reward */ },
  adStarted: () => { /* mute + pause */ },
};
window.CrazyGames.SDK.ad.requestAd("midgame", callbacks);
window.CrazyGames.SDK.ad.requestAd("rewarded", callbacks);
```

### Audio & input during ads

**English:**

- On **`adStarted`**: mute game audio and pause input/gameplay.
- On **`adFinished`** / **`adError`**: unmute and continue.
- Mute **only when the ad actually starts** — not at request time.
- Rewarded: grant reward **only** on `adFinished`; **never** on `adError`.

**中文：** 仅在 **`adStarted`** 时静音并暂停；结束或出错后恢复。激励**仅**在 `adFinished` 发奖。

### Adblock detection

```javascript
await window.CrazyGames.SDK.ad.hasAdblock();
```

### Error codes（常见）

| code | Meaning / 含义 |
|------|----------------|
| `adsDisabledBasicLaunch` | Basic Launch 期间广告禁用 |
| `unfilled` | 无填充 |
| `adblock` | 被广告拦截 |
| `adCooldown` | 请求过早；通常 midgame 间隔约 **3 分钟**（文档称会考虑 rewarded 与 **preroll**） |
| `other` | 其他 |

**⚠ 口径差异（抓取内）：** Requirements 页称过早 midgame 可能被 SDK **忽略且对用户无感**；SDK 广告页则返回 **`adCooldown`**。集成时按现行 SDK 行为处理两种情况，并保证游戏可恢复。

## AdBlock product rules · AdBlock 产品规则

**English (Requirements):**

- Must **not** make the whole game unplayable or punish players for AdBlock.
- May gate **extra** content (skins, some levels) with a prompt; avoid popup spam; don’t leave dead rewarded buttons.
- Disabling AdBlock often needs a refresh → **save progress first**.

**中文：** 不得因 AdBlock 锁死核心主线或惩罚玩家；可限制额外内容并提示；勿留无效激励按钮；提示刷新前先存档。

## Banners · 横幅

**English:** Static sizes include 728×90, 300×250, 320×50, 468×60, 320×100. Responsive candidates include larger formats (970×90, 160×600, etc.—see official list).

**中文：** 静态常见尺寸含 728×90、300×250、320×50 等；另有响应式候选尺寸，见官网列表。

### Hard rules · 硬规则

**English / 中文：**

- Refresh interval **≥ 30s** (same container) / 同容器刷新间隔 ≥ 30 秒  
- Max **120** refreshes per size per session / 每 session 每尺寸最多刷新 120 次  
- Must be fully inside the game window and fully visible / 须完全在游戏窗内且可见  
- Do not render/refresh during video ads / 视频广告播放中不可渲染或刷新  
- Use `clearBanner` / `clearAllBanners` when hiding / 隐藏后建议 clear  

### Disabled scenarios

| code | When |
|------|------|
| `bannersDisabledBasicLaunch` | Basic Launch |
| `bannersDisabledMobileApp` | Embedded mobile app |

**English (Requirements):** Banners only on useful screens; average open time **≥ 5s**; don’t block UI; not during active gameplay; max **2** per screen.

**中文：** 仅有用屏；平均打开 ≥5s；不挡 UI；活跃玩法中勿出；每屏最多 2 个。

---

## Frequency · 频控（产品层）

**English:** Midgame roughly **at most once every 3 minutes**; platform manages frequency. Do not build your own aggressive timers. Early requests → ignore or `adCooldown` (see conflict note above).

**中文：** 中插约最多每 3 分钟一次；频控归平台，勿自建激进计时器。

## Fangrush 提示

**English:** Map midgame to natural board breaks (between levels / restart). Keep a non-rewarded continue path. Basic Launch will hit `adsDisabledBasicLaunch` — treat as expected, not a hard failure. Standalone fangrush.com stays ad-free per channel plan.

**中文：** midgame 对齐局间/重开等自然断点；始终保留普通继续。Basic 出现 `adsDisabledBasicLaunch` 属预期。独立站按渠道路线保持无广告。
