## Two-stage launch · 双阶段发布

**English:** Games follow **Basic Launch** then **Full Launch**, to shorten initial go-live time and evaluate real-world performance before global release.

**中文：** 游戏遵循 **Basic Launch（基础发布）→ Full Launch（全面发布）**，缩短首发时间，并在全球发布前用真实数据评估表现。

### Basic Launch · 基础发布

**English:**

- Limited audience for a temporary **~2-week** period.
- Requires **Basic Implementation**; **no** CrazyGames-specific integration required; **Basic QA** only.
- **Monetization disabled** (video ads, banners, in-game purchases).
- If metrics are good, proceed toward Full Launch.

**中文：**

- 有限受众、约 **2 周**临时测试。
- 要求 **Basic Implementation**；**不要**做 CrazyGames 特定集成；仅 **Basic QA**。
- **变现关闭**（视频广告、横幅、内购）。
- 指标良好则可进入完整实现与 Full Launch。

### Full Launch · 全面发布

**English:**

- Selected for **global release**.
- Requires **Full Implementation** of CrazyGames requirements + **Full QA**.
- Monetization **enabled**; revenue share starts.
- During Full integration, the Basic build may remain available to its initial limited audience.

**中文：**

- 面向全球发布。
- 须完成 **Full Implementation** + **Full QA**。
- **开启变现**并开始收入分成。
- 完整集成期间，Basic 版仍可对初始有限受众开放。

**English:** Full Implementation **includes** Basic requirements. The QA tool guides submission step by step.

**中文：** Full 要求**包含** Basic 要求。QA 工具逐步引导提交与审核。

## Progression metrics · 晋级指标

**English:** Advancement to Full Launch is based on engagement metrics benchmarked against other games on the platform:

1. **Average playtime**
2. **Conversion to gameplay**
3. **Retention**

During Basic Launch you can monitor performance and gather feedback. After ~2 weeks you are notified of next steps.

**中文：** 进入 Full Launch 依据与平台其他游戏对比的参与度指标：

1. **平均游戏时长（average playtime）**
2. **转化为游玩（conversion to gameplay）**
3. **留存（retention）**

Basic 期间可监控表现与反馈；约 2 周后通知下一步。

### Outcomes · 2 周后可能路径

**English / 中文：**

| Outcome | English | 中文 |
|---------|---------|------|
| All metrics meet/exceed | Invited to update → implement Full requirements → submit Full Launch | 邀请更新 → 完成完整实现 → 提交 Full Launch |
| Some metrics meet | May be invited to improve and request another Basic Launch | 可能邀请改进并再申请一轮 Basic |
| Most metrics below | Cannot proceed to Full Launch; resubmit as a **new game** with **significant improvements** | 不能进 Full；若再交须作为**新游戏**且含**重大改进** |

## Bypass Basic Launch · 可跳过 Basic 的情况

**English:** In some cases you may go directly to Full Launch, for example:

- Multiplayer titles that need a larger audience
- Games already published elsewhere and **specifically invited** by the CrazyGames team

Multiplayer docs also note: whether Basic is required can be decided by QA (e.g. single-player component that does not need a large test audience may still need Basic).

**中文：** 下列情况可能直接 Full Launch：

- 需要更大受众的**多人游戏**
- 已在别处发布且被团队**特别邀请**的游戏

多人相关页亦说明：是否必须 Basic 可由 QA 判定。

## SDK 在流程中的位置（概览）

**English:** The CrazyGames SDK bridges your web game and the platform (ads, integration, etc.). Full Launch requires proper SDK use per Requirements. Basic Launch does **not** require CrazyGames-specific integration; if you still load ad APIs during Basic, the game must remain playable while ads are disabled.

**中文：** SDK 连接游戏与平台（广告、集成等）。Full Launch 须按 Requirements 正确使用 SDK。Basic **不要求**特定集成；若仍接入广告 API，在广告禁用时游戏须仍可正常游玩。

细则见 [04](./04-HTML5-SDK与游戏模块.md)、[05](./05-广告模块-视频与横幅.md)。

## Fangrush 提示

**English:** Dual-portal strategy usually starts **non-exclusive**. Basic Launch can validate fun/metrics **without** shipping a full CG SDK monetization build first; plan a separate Full build with SDK + ads. Do not treat Basic as “Poki Web Fit”—different funnel and ads-off period.

**中文：** 双门户策略通常先**非独家**。可用 Basic **先验证乐趣与指标**，再单独做 Full 包（SDK + 广告）。勿把 Basic 当成 Poki 的 Web Fit——漏斗不同，且 Basic 阶段广告关闭。
