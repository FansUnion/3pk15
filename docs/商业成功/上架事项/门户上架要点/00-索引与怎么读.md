# 门户上架要点 · 索引与怎么读

> **非官方工作摘要。** 以各门户现行文档与已签合同为准。  
> **本夹职责：** 把「该盯什么、为什么重要、Fangrush 如何取舍」写清楚。  
> **本夹不负责：** 写代码、勾工程进度、替代 `distribution/*` 执行清单。

> **分阶段要做什么：** 双门户并行的阶段事项清单见 [双H5门户上架计划](../双H5门户上架计划/00-总览与共用阶段.md)（S0 申请前基线 → S4 运营）；本夹提供规则与口径，不替代那份总规划。

## 双门户与独立站

- **A 层**对 **CrazyGames 包**与 **fangrush.com 演示站**同样有益（干净构建、跨设备、引导、广告断点纪律等）。  
- **B / C 层分站执行**：API、漏斗、封面规格、包体目标不同——见 [02](./02-Poki专属事项.md) 与 [03](./03-CrazyGames专属事项.md) 文首对照表。  
- 渠道阶段以 [渠道发展路线](../../渠道发展路线.md) 为准，通常**先非独家**，勿把任一站的独家条款默认套到另一站。

## 首次阅读顺序

1. 本文（边界与分层）  
2. [01 · 多平台通用](./01-多平台通用质量事项.md) — **先把产品质量做扎实**  
3. 接 Poki → [02](./02-Poki专属事项.md) 的 B1–B4（B5 知悉即可）  
4. 接 CrazyGames → [03](./03-CrazyGames专属事项.md) 的 C1–C4（C5 知悉即可）  
5. 细则 → [Poki 双语](../../2个H5平台官网资料/Poki官方文档双语整理/00-索引与阅读边界.md) / [CrazyGames 双语](../../2个H5平台官网资料/CrazyGames官方文档双语整理/00-索引与阅读边界.md)
6. 渠道阶段与是否独家 → [渠道发展路线](../../渠道发展路线.md)
7. 工程勾进度 → `distribution/poki/` 或 `distribution/crazygames/`（**不在本夹**）

## 刻意不进主清单的主题

- **趋势题材 / 换装特例**等与棋盘策略无关的品类细则 → 见 Poki 双语 [02](../../2个H5平台官网资料/Poki官方文档双语整理/02-内容质量-AI-趋势与本地化.md) Trends；本夹不展开。
- **CrazyGames 多人 / Partners / Web Games 101** → 见 CG 双语索引「刻意压缩」或官网。  
- **Unity 专用体积优化**等 → 官方引擎页；Fangrush 为 HTML5/TS。

## 写作约定（便于续写）

每条重要事项尽量含：**为什么重要 → 适用/不适用（含 Fangrush）→ 准备与关键口径 → 常见误区（可选）→ 细则链接**。  
禁止只留「英文优先」一类空标题。
