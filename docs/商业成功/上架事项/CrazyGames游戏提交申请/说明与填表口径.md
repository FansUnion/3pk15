## 这页在干什么

[developer.crazygames.com/submit](https://developer.crazygames.com/submit) 是开发者门户里的 **提交游戏向导**：

- **目的：** 上传可玩 Web 构建，经 QA 后进入 **Basic Launch**（或具备条件时走 Full）。
- **Basic：** 有限受众约 2 周测数据；**广告/变现关闭**；SDK **非**硬门槛。
- **Full：** 全球向发布前须完整实现（含 SDK、广告等）；通过后才开分成。
- **不是：** 签约页面本身；也不是「点提交 = 已全球上架赚钱」。

```mermaid

  submit["Submit 向导"] --> qa["Initial QA"]
  qa --> basic["Basic Launch"]
  basic --> full["Full Launch"]
  full --> live["变现与更新"]
```

## Basic / Full 选哪个（Fangrush 定稿）

| 时机 | 选 |
|------|-----|
| **第一次提交** | **Basic** |
| Basic 指标过线，或团队特邀，且 Full 就绪（S2b）已完成 | 再走 **Full** |

勿在未接 SDK / 未备齐三封面视频时误选 Full。

## 四步向导

| 步 | 名称 | 要做什么 |
|----|------|----------|
| 1 | **Upload** | 上传 Web/HTML5 可玩包；引擎选 HTML5（未列出的引擎也按 HTML5 上传） |
| 2 | **QA** | 用 Portal QA / Preview 自测可玩与基础要求 |
| 3 | **Details** | 游戏名、说明、操作、封面/视频等元数据 |
| 4 | **Submit** | 确认后正式送审 |

页面顶部的 **Basic / Full** 切换决定本轮提交按哪套要求审核——首交选 **Basic**。

## Upload 步字段（Fangrush 首交 · Basic）

| 字段 | 填什么 |
|------|--------|
| Game name * | `Fangrush`（最长 **35** 字符；须与**游戏内标题**一致；勿用「狼与绵羊」） |
| Game engine * | **HTML5**（自定义棋盘 / TypeScript / Next.js；**不要**选 Unity） |
| Does your game save progress? * | **No, the game does not need progress save** |
| The game supports mobile devices | **勾选** |
| The game is an online multiplayer game | **不勾** |
| The game supports CrazyGames muting audio through SDK | **不勾**（Basic 首交未接 SDK `muteAudio`） |

### 进度选项怎么理解（重要）

表单只有三种答案：

1. No, the game does not need progress save
2. Yes, using the Data Module from the CrazyGames SDK
3. Yes, linked to a game account on the game’s backend, associated with the CrazyGames User

**首交选 1。** 含义：当前**没有**接 CrazyGames **Data Module**，也**没有**绑 CG User 的后端云存——不要勾两个 Yes（会构成对平台能力的承诺）。
产品仍可有**浏览器本地进度**；那不等于 Data Module / CG User。日后接 Data Module 再改答并重提。

### Full / 已接 SDK 后再勾

| 选项 | 何时勾 |
|------|--------|
| muteAudio through SDK | 已实现并遵守平台 `muteAudio` 后 |
| Data Module 进度 | 已接 Data Module 且提交时勾选 Progress Save 后 |
| CG User 后端进度 | 已接 User + 自有后端关联后（棋类首发通常不做） |
| online multiplayer | 仅当真做联机时 |

## Details 步要准备什么（事项清单）

- **英文描述**与**操作说明**（短、清楚；与实机一致）
- **三封面**（风格一致）：横版 1920×1080（16:9）、竖版 800×1200（2:3）、方形 800×800（1:1）
- **横 + 竖无声预览视频**（约 15–20s、≤50MB 等，以现行要求为准）
- 规格见 [双语 03 · 封面](../../2个H5平台官网资料/CrazyGames官方文档双语整理/03-游戏封面与素材.md) 与 [`distribution/crazygames/README.md`](../../../../distribution/crazygames/README.md)

### Billing 提醒

尽早在 Portal **Account → Billing** 配置 Tipalti 等收款信息；未完成支付设置有时会影响提交。细则见 [双语 06 · 支付](../../2个H5平台官网资料/CrazyGames官方文档双语整理/06-用户数据内购排行榜支付与FAQ.md)。

## 提交前检查

- [ ] 向导顶部选的是 **Basic**（首交）
- [ ] Game name 为 `Fangrush`，与游戏内标题一致，≤35 字符
- [ ] Engine 为 **HTML5**
- [ ] 进度选 **No… does not need progress save**（未接 Data Module / CG User）
- [ ] 勾了 mobile；**未**勾 multiplayer；**未**勾 muteAudio（未接 SDK 时）
- [ ] 上传包可玩、英语可用、无外站广告网、无 Admin
- [ ] Details 封面/文案已按平台规格准备（或按当时 Portal 必填项备齐）
- [ ] 已读 [Requirements](https://docs.crazygames.com/requirements/intro)
- [ ] 在 Portal 完成 Submit

---

## 提交后

1. 等待 **Initial QA**；可能要求改包再传。
2. 通过后进入 **Basic Launch**（约 2 周；广告关闭属正常）。
3. 按时长 / 转化 / 留存对标结果：过线 → 做 Full 就绪再交 Full；部分过 → 改进后再 Basic；多数不过 → 不能进 Full，再交须当新游戏并重大改进。
4. 阶段事项详见 [02-CrazyGames分阶段事项](../双H5门户上架计划/02-CrazyGames分阶段事项.md)。

平台总览仍见 [海外平台接入小白指南 · E3](../../海外平台接入小白指南.md)。
