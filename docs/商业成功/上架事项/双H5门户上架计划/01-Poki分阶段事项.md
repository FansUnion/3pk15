## 前置：S1 准入（动作在 00）

- 完成 early access 提交，并**获得 P4D / 可上传后台**后，再进入下文 S2。  
- 未获准前：继续打磨 Demo、并行准备缩略图与集成材料，但**不能**当作已上架。  

**进入 S2 的门槛：** 已获 Poki 开发者后台访问权限。

---

## S2 · 集成与素材就绪

### 要做的事

- 完成平台要求的 **SDK 与游戏生命周期**就绪（加载完成、玩法开始/结束等验收口径）  
- 完成 **插屏 / 激励广告事件**就绪（自然断点；广告期静音停输入；激励仅成功发奖）  
- 满足门户技术纪律：如 **16:9** 画布、防父页滚动、初始包体目标、无未批准外链 / 无 GA、无第二广告网  
- 通过 **Inspector**（或等价预发 QA）自查，无阻断级问题  
- 准备并上传 **静态缩略图**（进 Web Fit 前必须）；动画缩略图可按发布阶段稍后补齐  
- 产出可提交的**干净门户构建**（无 Admin / 调试 / 独立站专用能力）  

### S2 完成标准

**构建可在 Inspector 按平台规则自测通过；静态缩略图已备；具备进入 Playtesting / Fit 的条件。**

---

## S3 · 测试漏斗

### 要做的事

- 完成 **Playtesting**：观看足够录像（申请 Player Fit 前达到平台要求的最低条数）  
- 申请并迭代 **Player Fit**，直至达到进入 Web Fit 的时长类门槛  
- 上传静态缩略图后申请 **Web Fit**；按 CTR / 停留 / C2P 三类结果迭代  
- 未过线时：按指标回到引导、内容、加载或缩略图，再测——**不跳过 Web Fit**  
- 通过后进入 **Final Review**；按结果准备签约或按反馈大改  

### S3 完成标准

**通过 Web Fit 与 Final Review（或明确未通过及下一步：改版重提 / 换作品）。**

指标摘要见 [门户上架要点 · 02 · B1](../门户上架要点/02-Poki专属事项.md)。

---

## S4 · 发布、收款与更新

### 要做的事

- 完成**合同签署**（注意 Web Exclusive vs 非独家；年限与 Discord/YT Playables 冲突以合同为准；双门户战略下慎签死网页独家）  
- 知悉并配合 **Technical / Soft / Global** 发布节奏（以对接人与现行文档为准）  
- 在 P4D 配置 **Billing**（电汇或 PayPal 等）  
- 建立发布后更新流程：Bug / 内容 / 语言经后台提交；缩略图变更遵守平台阶段规则  
- 用后台 DAU、参与度、收入、错误与反馈指导迭代  

### S4 完成标准

**游戏进入正式发布路径；收款信息已配置；具备可持续更新能力。**

---

## 首发可延后（非 S2 硬门槛）

下列知悉即可，避免淹没首接：

- 平台用户账号 / 云存档  
- 自定义分析事件（如 `measure`）  
- Fullscreen 资格申请  
- CLI 上传（有后台 UI 上传即可）  
- 动画缩略图 A/B（软发布阶段再做）
