## 与仓库其他文档的边界

### `docs/商业成功/门户上架要点/` — 分层精华（先读）

**English:** The [门户上架要点](../门户上架要点/00-索引与怎么读.md) folder is a short, decision-oriented digest: multi-platform quality vs Poki-only constraints. This bilingual series remains the detailed official-doc mirror; use the要点 folder first when scoping work.

**中文：** [门户上架要点](../门户上架要点/00-索引与怎么读.md) 是决策向分层摘要（多平台通用 vs Poki 专属）。本系列仍是官网细则对照；定范围时先读要点夹。

### `distribution/poki/README.md` — 执行清单

**English:** The README is the **action checklist** for Fangrush integration: what to implement, verify in Inspector, and ship. This digest explains **why** those items exist. When they conflict, follow the official site and your contract; use the README to track completion.

**中文：** `distribution/poki/README.md` 是 Fangrush 的**执行清单**（集成什么、Inspector 验什么、发什么包）。本系列解释**规则来源与含义**。若有冲突，以官网与合同为准；用 README 勾进度。

### `distribution/poki/early-access-form.md` — 填表口径

**English:** The early-access form doc covers **portal admission** (Request early access)—not upload, signing, or live release. Fields like studio name, engine (HTML5/Custom), and demo URL belong there. This series does not duplicate form copy.

**中文：** `early-access-form.md` 只管**开发者后台准入表单**（[Request early access](https://developers.poki.com/share)），不涉及上传包体、签约或正式上架。工作室名、引擎口径、Demo 链接等填表细节见该文件，本系列不重复。

### `distribution/poki/compliance.yaml` — 机检 / AI 合规矩阵

**English:** [`compliance.yaml`](../../../distribution/poki/compliance.yaml) encodes **project compliance status** (required vs recommended, missing/partial/ready/verified, evidence paths). This digest is human-readable context; the YAML is the AI- and checklist-facing matrix.

**中文：** [`compliance.yaml`](../../../distribution/poki/compliance.yaml) 承载**项目合规状态**（必做/建议、缺失/部分/就绪/已验证、证据路径）。本系列供人读；YAML 供 AI 与验收勾选。
