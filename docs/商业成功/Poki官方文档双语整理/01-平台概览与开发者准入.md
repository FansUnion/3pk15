## Working With Poki / Getting Started

**English:** Poki is one of the largest web gaming platforms, connecting 100M+ monthly active users across 100+ countries. Developers get tools, playtesting, QA, marketing, and user-acquisition support. Onboarding covers SDK integration, web performance, revenue model, and platform-exclusive benefits.

**中文：** Poki 是全球最大的网页游戏平台之一，连接 1 亿+ 月活、覆盖 100+ 国家。开发者可获得工具、试玩测试、QA、营销与获客支持。入门路径包括 SDK 集成、Web 性能、收入模式及平台独家权益。

## Why choose Poki?

**English:**

- **Playtesting** — Playtests and Player Fit Tests show how real players interact within hours.
- **Development tools** — Free tools include **Poki Inspector** (requirements/guidelines QA), **Netlib** (online multiplayer), and **AUDS** (Arbitrary User Data Store for backend data).
- **Revenue sharing** — 100% when the user arrives directly (bookmarks, search, social, your community); 50/50 when the user comes via Poki.com or Poki marketing.
- **Curation** — Hand-picked catalog; published games get homepage spotlight.
- **Marketing & UA** — Poki takes publisher-style roles (marketing, QA, etc.).

**中文：**

- **试玩测试** — Playtesting 与 **Player Fit Test** 可在数小时内观察真实玩家行为。
- **开发工具** — 免费工具含 **Poki Inspector**（对照要求/指南做 QA）、**Netlib**（联机）、**AUDS**（后端数据存储）。
- **收入分成** — 用户直达（书签、搜索、社交、自有社区）开发者 **100%**；经 Poki.com 或 Poki 营销来的用户 **50/50**。
- **人工策展** — 上架游戏保证首页曝光位。
- **营销与获客** — Poki 承担发行方类职责。

## Web exclusivity（网页独占）

**English:** In return for platform investment, Poki asks for **web exclusivity**—the game is published on the open web **only on Poki**. Steam, mobile app stores, and consoles are **not** restricted. Discord and YouTube Playables count as web platforms and conflict with exclusivity if you publish there separately.

**⚠️ Term length — pending confirmation:** Welcome/Why Poki pages describe exclusivity in general terms. **Deal Types** states default exclusive deals are **5 years**; **Bonus Level** overview mentions **7 years** for the default Web Exclusive model. **Verify in your contract and the current official Deal Types page before signing.**

**中文：** 作为平台投入的回报，Poki 要求 **Web 独占**：开放网页端仅在 Poki 发布；Steam、手游商店、主机**不受限**。Discord、YouTube Playables 视为网页平台，另行发布会与独占冲突。

**⚠️ 年限待核：** 入门页仅概括独占；**Deal Types** 写默认 **5 年**；**Bonus Level** 概述写 **7 年**。**签约前以合同与现行 Deal Types 为准。**

## Limited Access

**English:** Poki for Developers and playtesting tools are in **closed beta** with limited access. Apply via the [game submission / early access form](https://developers.poki.com/share). Every submission is reviewed; not all get replies—Poki contacts you if there is a fit.

**Review criteria:**

| Area | What Poki looks for |
|------|---------------------|
| **Quality** | UX/feel, core loop, fun potential—compare with top homepage titles |
| **Player fit** | Would Poki’s global audience enjoy this? |
| **Tech** | Technical soundness; mobile + desktop web optimization |

Cross-device games are favored. Poki works with many engines (Defold, PlayCanvas, Pixi, Three.js, optimized Unity, etc.). **Fangrush:** position as HTML5 / TypeScript / custom web engine.

**After acceptance:** two mandatory steps—implement **Requirements** and the **Poki SDK**.

**中文：** Poki for Developers 与试玩工具处于**封闭测试、限量准入**。通过[提交/早期准入表单](https://developers.poki.com/share)申请；非每份都有回复，合适则联系你。

**审查维度：** 质量（UX、核心循环）、**Player fit**（全球受众是否喜欢）、技术（跨端与优化）。跨设备游戏更受欢迎。

**通过后必做：** 满足 **Requirements** + 集成 **Poki SDK**。

## Poki for Developers (P4D)

**English:** P4D is the hub for team settings and per-game lifecycle: upload, analyze, update, and track testing/release metrics.

### General Team Settings

- **General** — Team info, primary contact, country (used after tests pass).
- **Users** — Roles: **Developer** (full access) or **Developer Support** (can upload versions; restricted earnings/metrics after release).
- **Billing** — After release, add/review billing for payouts.

### Games tab — five testing levels

1. Add your game  
2. Get player feedback (Playtesting)  
3. **Player Fit Test**  
4. **Poki Web Fit Test**  
5. **Final Poki Review**

After release: gameplays, earnings, engagement, monetization metrics.

### Versions

Upload builds; **Actions** → open in **Inspector** or **Preview** (live-like on Poki).

### Playtests / Settings / Thumbnails

- **Playtests** — Review/request recordings.  
- **Settings** — Title, up to **four** suggested categories, SEO-friendly description (Poki may adjust categories).  
- **Thumbnails** — Upload/update; **live** thumbnail changes need team review (drag-and-drop upload).

**中文：** P4D 统一管理团队与游戏：上传、分析、更新、跟踪测试与发行数据。

**团队设置：** General（团队/联系人/国家）、Users（Developer 全权限 / Developer Support 上传受限且发布后收益指标受限）、Billing（上架后账单）。

**Games 五步：** 添加游戏 → 玩家反馈 → **Player Fit Test** → **Web Fit Test** → **Final Poki Review**。上架后可见游玩、收入、参与、变现指标。

**Versions：** 上传版本；**Actions** 可开 **Inspector** 或 **Preview**。

**Playtests / Settings / Thumbnails：** 试玩录像；最多 **4** 个建议分类与描述（Poki 可调整）；缩略图上线后变更需审核。

## Poki Inspector

**English:** Primary pre-launch QA tool: verify SDK usage, runtime behavior, and Requirements compliance.

**Access:**

- Standalone: upload game folder to Inspector.  
- From P4D: **Versions → Actions → Inspector** (pre-loaded).

**Workflow:**

1. **QA Module Checklist** (left)—complete each module; check off passes; mark failures.  
2. **Event Log**—every SDK event fired; compare with [SDK docs](https://sdk.poki.com/general-info.html).  
3. **Game Details**—loading time, file size.  
4. **Desktop / Mobile mode**—default Desktop; Mobile shows QR for real-device SDK testing (events still log on desktop).  
5. **Scaling Tests**—bottom bar: test dimensions/devices.  
6. **Warnings** (bottom-right)—common types:  
   - **External Resources** — unapproved third-party calls  
   - **Image Optimization** — compress images for load/size  
   - **Unexpected Behavior Detected** — logic that may break SDK metrics tracking

**中文：** 上线前首选 QA 工具：检查 SDK、运行行为与 Requirements。

**入口：** 独立上传游戏文件夹；或 P4D **Versions → Actions → Inspector**。

**要点：** 左侧 QA 模块逐项勾选；**Event Log** 对照 SDK 文档；查看加载时间与包体；Desktop/Mobile（QR 真机测 SDK）；底部 Scaling Tests；右下角 **Warnings**（外部资源、图片优化、可能影响 SDK 的异常行为）。
