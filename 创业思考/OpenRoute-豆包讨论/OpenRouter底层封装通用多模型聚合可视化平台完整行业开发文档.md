## 1\. 项目整体概述

### 1\.1 产品定义

基于 OpenRouter 统一 API 网关二次封装，打造可视化多模态 AI 操作台 SaaS 工具。单一界面统一调用 GPT、Claude、Gemini、Perplexity、DeepSeek 等海内外上百款大模型，覆盖文本对话、识图解析、文生图、语音转写、文字配音全模态能力。配套额度统计、多模型并行对比、行业专属提示词模板、对话历史、团队分权管控等通用功能。

### 1\.2 两条落地路线

1. **独立通用操作台**：独立站点，面向泛创作者、独立开发者、跨境中小商家；缺点：市场同质化严重，获客成本高。

2. **嵌入式增值模块（推荐）**：无独立网站，直接内置至现有 AEO 监测工具后台，仅服务平台付费跨境卖家；依托垂直精准流量，差异化优势明显，避开通用 AI 工具内卷。

### 1\.3 核心差异化优势

1. 统一 OpenRouter 网关，一套代码兼容全部模型，无需单独对接各家厂商异构 API；

2. 零代码可视化操作面板，普通商家无需掌握 API 开发知识；

3. 双计费体系：平台统一额度池 \+ 用户自有密钥 BYOK，对冲高额 token 消耗亏损；

4. 内置跨境 AEO 垂直场景专属提示词模板，通用聚合工具无同类行业功能；

5. 独有业务闭环：AI 生成文案一键同步 AEO 检测接口，复测 AI 收录适配效果。

## 2\. 目标用户分层与真实业务场景需求

### 2\.1 核心付费人群：Shopify 跨境独立站商家（付费意愿最高、高频使用）

1. **多 AI 渠道流量全覆盖需求**
ChatGPT、Perplexity、Gemini 网页抓取逻辑、内容收录偏好完全独立，单一模型产出文案仅适配单一流量渠道。商家通过多模型并行生成内容，融合多版本共性内容，产出一套可全平台收录的页面文案，避免流量丢失。

2. **分场景模型分工刚需**

- Perplexity：联网检索，挖掘海外用户搜索词、竞品 FAQ、行业热点，用于 AEO 调研素材收集；

- Claude：超大上下文窗口，批量改写长产品详情、专业设备技术参数；

- GPT4o：短营销文案、广告话术、消费向问答润色；

- Gemini：多语种翻译、适配谷歌 AI 摘要收录规则；

- DeepSeek 国产模型：低成本生成中文初稿，大幅降低 token 消耗。

3. **一站式提效需求**
无需同时打开多个 AI 官网来回复制粘贴，竞品调研、文案撰写、多版本对比、收录复测全流程在同一页面完成。

4. **综合成本管控需求**
单独订阅 GPT Plus、Claude Pro、Perplexity Pro 每月合计 60 美金固定支出；本平台按量计费无强制月费，商家可按需切换高低成本模型，压缩运营开支。

### 2\.2 次要人群 1：海外自媒体、短视频创作者

1. 同一选题多模型输出多风格文案，筛选适配 Facebook、TikTok 社媒内容；

2. 批量短视频脚本生成、多语种润色，配套文生图生成社交宣传配图。

### 2\.3 次要人群 2：独立开发者、程序员

1. API 可视化调试，快速测试各模型输出格式、参数效果，省去本地重复编码调试；

2. 多模型 A/B 效果测试，横向对比输出准确率、响应延迟、调用成本，为自有产品选型；

3. 多项目拆分额度，统一查看全项目 token 消耗报表。

### 2\.4 次要人群 3：跨境运营工作室（企业级付费客户）

1. 团队额度分权：管理员统一充值总额度，给运营、文案分配独立子账户，限制单人单日调用上限；

2. 多模型对比结果生成分享链接，团队协同筛选最优商品页面文案；

3. 批量导入商品关键词，批量自动化生成 FAQ、商品详情描述。

## 3\. OpenRouter 底层全模态能力支持范围

单一 OpenRouter 密钥统一支持文本、图像、音频三大模态，共用一套计费规则、统一请求规范，前端通过 Tab 标签隔离不同功能。

### 3\.1 文本模态（核心必做功能，高频使用）

接口地址：`/api/v1/chat/completions`
支持能力：多轮对话、长文档解析、联网检索、结构化 JSON 输出、网页引用溯源；
兼容全部文字类大模型：Perplexity、GPT 全系、Claude 全系、Gemini、DeepSeek 等。

### 3\.2 图像相关能力（中度需求，二期开发）

1. 识图理解：支持图片 URL、Base64 批量上传，自动 OCR 提取文字，解析页面卖点、参数；适配 GPT4o、Claude3、Gemini 多模态模型；
商家场景：竞品网页截图分析、自有网站页面缺陷提取、商品实拍参数识别。

2. 文生图：独立接口`/api/v1/images`，统一分辨率、画布比例、垫图参数；支持 Grok、Seedream 等图像生成模型；
商家场景：Pinterest 跨境商品封面、FAQ 配套宣传图。
限制：纯文本模型不支持图片上传与生成，前端自动过滤不兼容模型，防止调用报错。

### 3\.3 音频语音能力（低频需求，项目后期迭代）

1. STT 语音转文字：接口`/api/v1/audio/transcriptions`，音频文件转文本；用于短视频、直播录音挖掘用户搜索关键词；

2. TTS 文字转语音：接口`/api/v1/audio/speech`，文本生成多语种配音；用于独立站商品语音介绍；
技术限制：仅支持 16kHz 单声道 wav 音频，前端内置自动格式转换逻辑。

### 3\.4 配套扩展能力

1. PDF 文档解析：上传产品白皮书、竞品网页 PDF，AI 提取核心内容用于 AEO 优化；

2. 多模型合议：后端并行调用多款模型，自动提取输出共性答案，减少人工对比操作；

3. 内置插件：联网搜索、长文本压缩、JSON 格式化修复。

## 4\. 全球同类竞品产品形态拆解（2026 市场四类方案）

### 类型 1：通用 C 端对话聚合工具（不建议单独开发）

代表产品：\[Aurixel\.ai\]\(Aurixel\.ai\)、\[Eye2\.ai\]\(Eye2\.ai\)
产品特点：仅提供通用对话功能，无垂直行业场景；支持多模型切换、并行对比；同质化严重，泛流量付费转化极低，获客成本极高。

### 类型 2：开发者 API 调试平台

代表产品：OpenRouter 官方控制台、Poe Quora
产品特点：侧重底层参数调试、请求日志、用量监控，支持导出 SDK 代码；面向程序员，普通跨境商家使用率极低。

### 类型 3：垂直 SaaS 内置附属工具（最优落地路线）

代表产品：Promptwatch、Peec AI（海外 AEO 工具）
产品特点：无独立站点，嵌入主 SaaS 后台；内置行业专属提示词模板；与核心业务打通，文案一键复测；依托垂直精准流量，付费转化能力强。

### 类型 4：BYOK 密钥管理工具（面向重度专业用户）

代表产品：\[Conduity\.ai\]\(Conduity\.ai\)、SheetMagic
产品特点：不提供平台自有额度池，仅提供可视化操作台；用户填入自有 OpenRouter 密钥；平台仅收取月度工具订阅费，无 token 成本风险。

## 5\. 四大成熟盈利商业模式（可单独或组合使用）

### 模式 1：平台统一额度池，token 差价抽成（基础入门方案）

1. 逻辑：开发者在 OpenRouter 后台统一充值余额，用户在平台充值购买 token 额度，每笔 AI 调用抽取 5%\-10% 服务费盈利；

2. 优势：用户小额充值门槛低（5/10 美金起充），现金流稳定；

3. 短板：存在资金垫资成本，免费用户高频调用会产生亏损；

4. 适配人群：无自有 API 密钥的新手跨境小卖家。

### 模式 2：分层订阅会员 \+ 月度赠送额度（主推，适配 AEO 捆绑）

三层套餐，与 AEO 监测套餐账号、权限完全打通：

1. 免费版：每日 3 次基础对话，仅开放低价国产模型，无多模型并行对比；

2. 基础月套餐（$9\.9 / 月）：每月赠送固定 tokens，解锁 GPT/Perplexity/Claude，支持双模型并排对比；

3. 高级月套餐（$19\.9 / 月）：大额月度额度、批量文案生成、BYOK 密钥、团队子账户、全套 AEO 提示词模板、完整页面仿真复测。
优势：稳定月度复购收入，可严格管控单日调用成本。

### 模式 3：BYOK 自带密钥模式（零 token 成本，必做对冲功能）

1. 逻辑：用户自主持有 OpenRouter 私人 API 密钥，平台后端 AES 加密存储；所有 AI 调用费用直接从用户自身 OpenRouter 账户扣除，开发者不承担任何 token 支出；

2. 盈利点：仅收取平台工具月度订阅使用费，token 无任何加价；

3. 价值：对冲免费用户大额额度消耗亏损，适配重度运营工作室、自有 API 账号开发者。

### 模式 4：增值功能单独付费（补充收入渠道）

1. 单次批量文案生成包、PDF 图文导出、多模型共识内容合成；

2. 团队多人协作子账户单独加价；

3. 对外开放聚合 API 接口授权，给其他小型站点接入。

## 6\. 整体技术架构设计（Vercel Next\.js 轻量化架构）

### 6\.1 完整技术栈清单

1. 前端：Next\.js15 App Router、React19、TypeScript、Tailwind CSS4、shadcn/ui、react\-markdown、Vercel AI SDK useChat 流式对话；

2. OpenRouter 对接方案（优先选择）：openai 官方标准 SDK，一套代码兼容 OpenRouter 与各大厂商 API，迁移成本极低；备选：@openrouter/sdk 官方专用包；

3. 后端：Next\.js Edge API Routes，无自建服务器，Vercel 全托管；

4. 存储：Vercel Blob（图片、音频临时存储）、Vercel KV 数据库（用户额度、对话历史、加密密钥存储）；

5. 部署：Vercel 一键云端部署，无需自建代理 IP 池、无需服务器运维。

### 6\.2 三层分层架构

1. **前端可视化客户端（Client 组件）**
承载全部操作台 UI、参数滑块、模型选择、对话窗口、文件上传组件；密钥全程不暴露前端，所有 AI 请求中转后端执行。

2. **后端中转 API 层（Server Edge Runtime）**
核心能力：隐藏 OpenRouter 主密钥、账号鉴权、额度扣减、参数校验、SSE 流式内容转发、多模态文件预处理、BYOK 密钥解密调用。

3. **OpenRouter 底层网关层**
统一转发文本 / 图像 / 音频请求，统一返回 usage 用量统计数据，后端用于计费、扣减额度。

页面采用 Tab 标签分区隔离，「文本对话」设为首页核心，识图、文生图、语音工具作为次级页面，降低新手操作复杂度。

## 8\. 后端核心接口开发规范（Vercel Edge Route）

### 8\.1 环境变量统一配置（Vercel 后台配置，前端不可访问）

1. `OPENROUTER_API_KEY`：平台自有额度池密钥；

2. `APP_DOMAIN`：项目域名（OpenRouter 请求头必填参数）；

3. `AES_SECRET`：用户 BYOK 密钥加密存储密钥；

4. `AEO_DETECT_API`：联动 AEO 监测工具复测接口地址。

### 8\.2 核心接口 1：/api/models

1. 功能：拉取 OpenRouter 全量模型列表，后端分类格式化后返回前端下拉数据源；

2. 逻辑：缓存模型列表，减少重复网络请求，按模态（文本 / 图像 / 音频）标记兼容能力。

### 8\.3 核心接口 2：/api/chat（文本对话主接口）

```typescript
import OpenAI from "openai";
const openai = new OpenAI({
  baseURL: "https://openrouter.ai/api/v1",
  apiKey: process.env.OPENROUTER_API_KEY,
  defaultHeaders: {
    "HTTP-Referer": process.env.APP_DOMAIN,
    "X-Title": "跨境AEO多模型操作台",
  },
});
```

核心业务逻辑：

1. 校验用户账号剩余额度，额度不足直接拦截请求；

2. 区分两种调用模式：平台公共额度 / 用户 BYOK 自有密钥；

3. 透传前端全部可视化参数，Perplexity 网页引用通过 extra\_body 透传；

4. SSE 流式实时推送 AI 输出内容；

5. 读取接口返回 usage 用量数据，实时扣减个人账户 token 额度；

6. 统一捕获异常：模型失效、调用超限、格式错误，返回前端友好提示。

### 8\.4 核心接口 3：/api/image（识图 \+ 文生图）

1. 图片预处理：后端自动压缩大图至 1024\*1024 标准尺寸，降低调用失败概率；

2. 统一标准化文生图参数，厂商独有参数通过 provider\.options 兼容适配。

### 8\.5 核心接口 4：/api/audio（TTS/STT 语音接口）

自动转换音频格式为 OpenRouter 支持的 wav 单声道格式，返回音频二进制流，前端实现播放、下载功能。

### 8\.6 配套业务接口

1. /api/user/quota：用户额度查询、充值、扣减；

2. /api/user/byok：用户自有密钥 AES 加密存储、读取解密；

3. /api/history：对话历史持久化存储、分页查询；

4. /api/aeo/retest：文案同步至 AEO 检测接口，返回 AI 收录适配分数。

## 9\. 额度管理、计费、BYOK 密钥安全完整方案

### 9\.1 两套额度隔离体系

1. **平台公共额度池**：开发者统一充值 OpenRouter，用户消耗平台余额；后端根据 usage 字段实时扣减个人账户 tokens；文本 tokens、图片生成次数、音频时长三类消耗分开统计展示。

2. **BYOK 私有额度**：用户填入自有 OpenRouter 密钥，全部调用费用从用户自身账户扣除；平台仅统计调用次数，收取订阅费，无 token 成本。
