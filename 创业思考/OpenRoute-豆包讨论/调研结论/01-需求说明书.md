> 状态：**已确认**（2026-07-12）

---

## 1. 一句话定位

面向海外（欧美/全球）用户的 **纯 BYOK 多模型 AI 可视化操作台**：用户在 **OpenRouter 或同类统一聚合平台** 自备 API Key、充值一次即可调用多模型；我方提供英文站托管、对话界面、云端存储与权限体系，按月/年收取 **技术服务 / 订阅费**。不囤账号、不垫资 Token、不做 API 倒卖；品牌与站点完全独立，不绑定 AEO。

---

## 2. 约束与原则（硬边界）

### 2.1 做（In Scope）

| 项 | 说明 |
|----|------|
| 可视化对话 UI | 流式聊天、多模型切换、基础参数调节、对话历史 |
| 统一聚合上游 | 默认 OpenRouter；允许同类「一次充值、全模型计费」平台；用户无需给每个模型厂商分别充值 |
| BYOK | 用户填入自有上游 Key；模型费用由用户与上游结算 |
| 服务端转发 | Key **加密存服务端**，请求经我方后端转发至上游（体验优先；安全说明必做） |
| 云端能力 | 账号体系、对话云同步、基础权限 |
| 商业化二开 | Stripe（或同等）订阅；**免费 + 2～3 档付费**；免费层**无广告**，仅功能限制 |
| 新手引导 + 联盟 | 英文教程引导注册/绑 Key；官网含 OpenRouter 联盟/教程深链 |
| 品牌 | 完全独立品牌与站点（与 AEO / Levelmere 其他产品分离） |
| 合规底线 | 隐私政策、Key 与数据安全说明、用户可清空对话数据 |

### 2.2 不做（Out of Scope / Won’t）

| 项 | 原因 |
|----|------|
| 平台统一充值 / Token 差价 / 积分池 | 垫资与亏损风险，个人开发者不可控 |
| 批量采购 ChatGPT 等账号池 | 封号与合规风险 |
| 对外提供 API 中转 / 倒卖接口 | 合规与售后风险 |
| 绑定 AEO / GEO 监测业务 | 本需求为独立工具线；AEO 另案 |
| 免费层广告变现 | 已拍板：靠功能限制转化订阅 |
| MVP 支持「逐家官方 Key」 | 与「充值一次、全模型」叙事冲突；复杂度高 |
| 重做「第二个 OpenRouter」 | 上游聚合交给 OpenRouter 类平台；我方卖体验与托管 |

### 2.3 关于「依赖 OpenRouter」的产品叙事

鼓励用户去 OpenRouter（或同类聚合平台）注册 **不是缺陷，而是分工**：

- 上游 = 模型接入与计费层（用户充值一次，切换 GPT/Claude/Gemini 等无需分别开通）  
- 本产品 = 好用的聊天/工作台 + 托管运维 + 订阅制工具费  

官网明确展示 OpenRouter 教程与联盟深链。价值主张：用户付的是 **界面、存储、多模型一站式体验**，不是「更便宜的 Token」。

## 3. 目标用户与痛点

### 3.1 首发主人群：海外个人用户（创作者 / 自由职业 / 学生 / 办公）

- **痛点**：不想同时订阅多个 AI 官网会员；希望一个界面切换多模型；需要云端历史与基础工具。  
- **付费点**：为「省事 + 多模型 + 云端」付订阅费；Token 在上游自理。

### 3.2 首发辅人群：隐私敏感用户 / 独立开发者

- **痛点**：不信任平台代充额度；希望自备 Key、成本透明；需要稳定的多模型客户端。  
- **付费点**：为「BYOK + 可控成本 + 好用 UI」付费；可接受自学绑 Key。

### 3.3 明确非首发（后续再评估）

- 跨境卖家 / AEO 文案闭环用户  
- 大团队协作、审计、私有模型深度定制（可作为远期 Business 档功能，非 MVP 必做）

---

## 4. 商业模式

### 4.1 主模式

**Freemium + 2～3 档阶梯订阅**（MVP 即具备）：

- **免费层**：无广告；限制存储时长 / 高级功能 / 用量或能力上限，驱动升级  
- **付费 2～3 档**：按存储、文件、工具、支持等解锁（具体美元价后定，对标海外同类工具订阅）  

**唯一主营收**：会员订阅（工具服务费）。模型调用费 **100% 用户自付（经上游账单）**。

### 4.2 MVP 应具备的上游变现配合

- 官网 OpenRouter **联盟 / 教程深链**（注册与充值引导；佣金若官方有则作为增量）  

### 4.3 可选项（非阻塞）

- 小额增值：绑 Key 一对一指导、提示词/智能体模板包  

### 4.4 明确不做的变现

- Token 加价、站内充值余额、对外卖 API 额度、免费层广告  

### 4.5 定价原则

对标海外同类 BYOK 聊天工具的「工具订阅」区间；**档位结构已定（免费 + 2～3）**，具体金额在竞品调研后回填。

## 5. MVP 功能范围

### 5.1 Must（上线前必须有）

1. 用户注册/登录（英文站）  
2. 配置并保存用户自有 **OpenRouter（或已选定的同类聚合上游）Key**（BYOK）  
3. Key **服务端加密存储**；聊天请求经后端转发至上游  
4. 流式多轮对话、模型选择、基础参数  
5. 对话历史持久化（按档位区分保留策略）  
6. Stripe（或同等）订阅 + **免费 + 2～3 档**权限隔离（无广告）  
7. 英文新手引导：注册上游、充值一次、绑定 Key  
8. 官网：OpenRouter 联盟/教程深链 + Key/数据安全说明  
9. 基础隐私说明与清空对话 / 删除 Key 入口  

### 5.2 Should（尽量有，可紧随 MVP）

1. 文件上传 / PDF 等基础解析（若开源底座已有则保留）  
2. 提示词模板或简易智能体  
3. 用量展示（调用次数或上游 usage，**不做平台扣费**）  
4. 第二聚合上游预留（仅当符合「一次充值、全模型」且实现成本低）  

### 5.3 Won’t（本阶段明确不做）

1. 平台公共 Token 池与站内充值  
2. AEO 检测、文案复测、跨境提示词业务闭环  
3. 对外 OpenAPI 中转服务  
4. 免费层广告  
5. MVP 内「分别填写 OpenAI / Anthropic / Google 官方 Key」作为主路径  
6. 完整企业审计 / 多集群私有化交付（除非底座自带且零维护）

## 6. 开源选型

**当前约束更新**：你希望 **Vercel 部署 + 低成本**，同时尽量 **少二开、核心功能够用**。

这与「只选 LibreChat」不完全同向——LibreChat **不能原生跑在 Vercel**，通常要小 VPS（约 \$5～12+/月）。

| 路径 | 底座 | 部署 | 何时选 |
|------|------|------|--------|
| **路径 V（低成本优先）** | [LobeHub/LobeChat](https://github.com/lobehub/lobehub) | Vercel + Neon 等 | **更贴你现在的话**；功能全 + 固定成本极低 |
| 路径 A | vercel/chatbot | Vercel | 部署最快，但聊天能力要自己补 |
| 路径 C | HiveChat | Vercel + Postgres | Next 单体，社区较弱 |
| 路径 B | LibreChat | Docker/VPS | 功能最全；接受月费与运维时 |

**你做的事（各路径共性）**：托管品牌站、OpenRouter 默认化、Stripe 多档、绑 Key 引导与安全页。  
**你不做的事**：重写聊天前端、Token 池、AEO。

详情与更长候选表见 [`04-市场与开源调研.md`](./04-市场与开源调研.md) §4。

## 7. 个人开发者落地约束

| 约束 | 要求 |
|------|------|
| 时间精力 | Fork 成熟开源，核心功能不重做；仅少量商业化二开 |
| 风险 | 零垫资、零账号池、零 API 倒卖 |
| 部署 | 海外可访问；**优先 Vercel**；独立域名与品牌 |
| 部署成本 | **优先 Vercel + 托管 Postgres 免费档**；LibreChat 则小 VPS |
| 支付 | 优先 Stripe 订阅（免费 + 2～3 档） |
| 语言 | MVP 主打英文 |
| 成本 | 固定成本尽量压低；不以烧 Token 获客 |
| 运维 | 单人可维护；无复杂扣费纠纷与封号售后 |

分阶段建议（方向性，非排期承诺）：

1. 在路径 V（LobeChat/Vercel）与路径 B（LibreChat/VPS）中拍板 → Spike  
2. 品牌白标 + Stripe 多档 + 新手引导 + OpenRouter 深链  
3. 小流量验证；非必要不追上游新功能
