## 调研目标（对应你的原始问题）

1. **是否有需求**：谁在为「BYOK + 多模型聊天 UI」付费？付的是工具费还是 Token？  
2. **是否有解决方案**：市场已有哪些形态？缺口在哪？  
3. **是否有对应开源项目**：哪些仓库适合个人开发者少量二开后上线？  
4. **这类项目的特点**：共性能力、商业模式、技术栈偏好。  
5. **用户一般怎么在 Google 搜索**：真实搜索词与意图分层，便于后续 SEO/广告。

## 解决方案分类框架（沿用竞品讨论，待核实）

```text
1. 纯 API 网关     — 如 OpenRouter：卖接入与路由，弱/无聊天 UI
2. 托管聊天 SaaS   — 如 Multio / Poe / ChatHub：完整 UI，BYOK 多为附加
3. 开源自托管      — 如 LibreChat / HiveChat：自托管 + BYOK 彻底
4. 浏览器插件/客户端 — 轻量多模型切换，偏个人效率
```

你的目标落点：**2 的商业形态（托管 SaaS 收订阅）+ 3 的成本结构（纯 BYOK 不垫资）**，用开源底座二开实现。

## 品类特点（假设，调研时验证/修正）

预期共性（验证用）：

- BYOK 或「自备 Key」作为卖点或高级档权益  
- OpenAI 兼容协议 + 多模型切换 + 流式输出  
- 卖的是 **客户端/工作台体验**，不是更便宜的 Token  
- 常见变现：订阅、一次性买断、托管费；纯开源则靠捐赠/企业支持  
- 个人开发者友好路径：Next.js / Docker、英文文档、Stripe 易接

## Google 搜索词调研框架

### 意图分层（填词时归类）

| 意图 | 用户在找什么 | 示例方向（待证实） |
|------|----------------|-------------------|
| 问题认知 | 不想多开会员、要隐私、要自备 Key | `bring your own api key ai chat` |
| 品类词 | 多模型客户端 / 聚合聊天 | `multi model ai chat`、`all in one ai chatbot` |
| 上游关联 | 已有 OpenRouter，要 UI | `openrouter chat ui`、`openrouter frontend` |
| 自托管 | 要自己部署 | `self-hosted chatgpt`、`librechat alternative` |
| 比价/替代 | 逃离 Poe/ChatGPT Plus | `poe alternative byok`、`chatgpt alternative own api key` |
| 商业/开发 | 建站、SaaS | `byok ai saas`、`ai chatbot stripe subscription` |

### 每个词记录

- 是否有明显结果页（工具站 / 开源 / 文章）  
- 竞争大致强弱（观感即可，不做付费 Keyword Planner 也可先定性）  
- 与你定位的匹配度：高 / 中 / 低  
- 建议用途：产品名副标题 / 落地页 / Google Ads / 社区发帖  

豆包稿已出现的种子词（调研时验证）：

- `bring your own api key ai chat`  
- `multi model ai client`  
- `privacy ai chat`
