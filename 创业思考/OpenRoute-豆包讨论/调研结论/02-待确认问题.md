## 已拍板

| ID | 问题 | 结论 |
|----|------|------|
| D1 | 产品形态？ | **独立海外 SaaS 聊天站**，不嵌入 AEO；**完全独立品牌与站点** |
| D2 | 首发用户？ | **海外个人用户 + 隐私/开发者**；不做跨境卖家/AEO 场景为主 |
| P1 | 开源底座策略？ | 曾选路径 B（LibreChat）→ 现增加 **低成本/Vercel** 约束；**待你在路径 V（LobeChat）与路径 B 间拍板** |
| P2 | 免费层是否含广告？ | **不要广告**；免费仅功能限制，靠转化订阅 |
| P3 | 订阅档位？ | MVP 就要 **免费 + 2～3 档付费**；具体美元价仍可后定 |
| P4 | Key 支持范围？ | **统一聚合上游优先**：默认 OpenRouter Key；可扩展同类「充值一次、全模型可用」的平台；**不做**逐家官方 Key（OpenAI/Anthropic/… 分别充值） |
| P5 | 密钥与调用路径？ | **Key 服务端加密存储**，请求经我方后端转发至上游；安全说明必做；实现以开源底座能力为准 |
| P6 | 市场与语言？ | **主打英文站**，面向欧美/全球 |
| P7 | OpenRouter 对外说法？ | **官网明确写 OpenRouter 联盟/教程深链**（上游分工叙事保留） |
| P8 | 与 AEO 等关系？ | **完全独立品牌与站点** |
