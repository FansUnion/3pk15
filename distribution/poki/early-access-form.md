# Poki 早期准入表单 · 照抄填写

> 表单地址：[Request early access](https://developers.poki.com/share)  
> 说明：这是申请 **开发者后台准入**，不是上传游戏包、不是签约、不是上架。  
> 概念说明见 [`docs/商业成功/海外平台接入小白指南.md`](../../docs/商业成功/海外平台接入小白指南.md) §E2.1。  
> **姓名 / 邮箱 / 国家** 必须用你本人真实信息；下面只写仓库已定稿的产品口径。

## 提交前检查

- [ ] 已用真实姓名、长期邮箱、所在国家填写
- [ ] Studio 填 `Fangrush Studio`（勿写内部代号「狼与绵羊」）
- [ ] 作品链接含可玩 Demo：`https://fangrush.com`（若暂不可达，改用 itch / 录屏并说明）
- [ ] 引擎写 HTML5 / Web / Custom，**不要**写 Unity
- [ ] 第一次发游戏：平台履历选 None / Not yet，勿编造
- [ ] 已在本页表单提交（Submit）

## 字段对照（定稿口径）

| 字段 | 填什么 |
|------|--------|
| What's your name? | （你本人英文名或拼音全名） |
| What's your email? | （长期商务邮箱） |
| What's the name of your studio or team? | `Fangrush Studio` |
| In which country are you based? | `China`（或实际常驻国英文名） |
| How would you describe your studio or team? | Indie / Solo developer / Small indie studio；有 First-time 类也勾 |
| If you've previously released games, which platform… | None / Not yet（首次则勿勾假平台） |
| Can you provide us with links to those games? | 见下方「作品链接栏」粘贴块 |
| Which genres do you typically work on? | `Strategy, Puzzle, Board game, HTML5 casual` |
| Which engines do you typically work with? | `HTML5 / TypeScript / Next.js (custom board-game engine)`；下拉优先 HTML5 / Custom / Web |
| What are you looking to gain from working with Poki? | Distribution / Reach players；Monetization / Ads revenue share；Publishing support（有 Analytics / Growth 也可勾） |

## 提交后

1. 等待 Poki 邮件 / 后台邀请；可能较久、也可能无回复。  
2. 获准后进入 SDK、portal 包与正式提交，见同目录 [README.md](./README.md) 与任务 `T-017`。  
3. 通过准入 ≠ 游戏已上架。
